# PitGuard BIM Designer V3.11.0

面向基坑围护结构方案设计、分阶段计算、规范审查、配筋与构造深化、三维复核以及 IFC/CAD/PDF 交付的工程设计辅助系统。

## 启动

- Windows：运行 `start-windows.bat`
- Linux/macOS：运行 `bash start-linux.sh`
- 前端默认地址：`http://127.0.0.1:5173`
- API 文档：`http://127.0.0.1:8002/docs`
- 后端默认端口：`8002`；可通过 `PITGUARD_BACKEND_PORT` 覆盖

系统使用当前 Python 环境，不自动创建额外虚拟环境。启动脚本会检查缺失或损坏的 Python 包；安装失败退出时会打印可直接复制的安装命令。前端依赖已锁定，建议使用 `npm ci`。

## 文档入口

全部说明集中在 [`docs/README.md`](docs/README.md)。根目录只保留启动脚本、源码目录和本文件。

## V3.11.0 关键变化

- 钢筋加工深化下载改为 ZIP 成套包，包含 XLSX、CSV、完整 JSON、检查表和使用说明；JSON 保留为机器交换文件。
- 八个设计流程节点显著显示主控规范、强制等级、关键计算、条文关注点、规则 ID、项目状态和输出成果。
- 在线文档扩展为操作流程、计算原理、流程—规范矩阵和成果文件使用四部分。
- DOCX 计算书自动插入流程—规范—条文—规则矩阵和关键公式/复核点。
- 完整交付与 CAD 清单移除完成度 100% 硬编码，改用项目实时 Assurance 指标。
- 全局墙—围檩—支撑矩阵增加 K·u=F 残差、矩阵对称误差、条件数和正则化质量门禁，并在计算书中显式列出。
- 大型钢筋包按“XLSX 人工主表 + 完整 CSV/JSON”分层交付，示例 12,000 根逐根钢筋可在约 10 秒量级生成。
- 构造协调方案可真实写回逐根钢筋中心线、局部附加筋及预埋件位置/开孔，写回后自动重跑加工、净距和碰撞检查。
- 吊装场地支持边界、道路走廊、建筑/高压线禁入区和吊机占地包络，并用 A* 生成可检查的场内路径。
- A/B/C 整体方案完整计算拆分为三个独立后台任务，支持并行进度、单方案失败/重试和稳定输入哈希缓存。
- 候选预览按实际几何边界自动居中和放大；选中方案可滚轮缩放、拖动平移和一键适配。
- “闭环审查”继续保留，并新增十二模块完成度、证据、缺口、阻断状态和下一步动作。
- 工作台增加方案设计、结构计算、配筋深化、施工策划、校核审核和图纸发行六种角色视图；角色视图只调整优先级，不隐藏其他模块。
- 工程单位统一由 `packages/units/engineering-units.json` 生成后端/前端注册表。
- 三维查看器改为懒加载、折叠区延迟挂载和运行时 LOD；项目工作台生产块由约 244.84 kB 降至约 161.02 kB。
- 暂不接入外部有限元求解器，既有节点子模型输入能力继续保留。

## 验证

- 后端快速门禁：`bash scripts/test-backend.sh fast`
- 后端完整隔离门禁：`bash scripts/test-backend.sh full-isolated`
- 前端测试与构建：`bash scripts/test-frontend.sh`

Windows 后端门禁可运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/test-backend.ps1 -Mode fast
```

## 工程边界

系统输出用于方案设计、工程复核、深化协调和成果组织。施工图正式签发仍需注册岩土/结构工程师复核参数来源、模型边界、规范适用性、节点构造、吊装方案、施工工序和现场条件。
