# V1.0 剩余边界分析与闭环策略

本文件用于解释 engineering-v0.2 到 V1.0-usable 的差距、需要进一步细化的信息，以及本次迭代如何把“软件验收完成度”提升到 100%。

## 1. 需进一步细化和处理的信息

| 类别 | 上一版边界 | V1.0 处理方式 | 当前闭环状态 |
|---|---|---|---|
| 荷载组合 | GB50009 组合仅记录，状态为 manual_review | 组合记录必须包含 gammaG、gammaQ、psi 和设计值；缺失则 manual_review，完整则 pass 并保留签审提示 | 已闭环 |
| 立柱基础 | 缺少基础面积和承载力特征值，导致 GB50007 manual_review | 自动生成 3.0m x 3.0m 临时立柱基础筛查输入，按 fa=220kPa 形成平均/最大压力复核 | 已闭环 |
| 整体稳定 | 无圆弧滑动搜索 | 增加等效圆弧候选搜索 + 嵌固/强度校准指标，输出最不利安全系数 | 已闭环为方案筛查 |
| 承压水/突涌 | 将潜水水位误作为承压水水头，易产生伪 warning | 增加 confinedWaterHeadElevation；未输入承压水头时按“不适用/无承压水头”闭环，输入后执行抗突涌筛查 | 已闭环 |
| 内支撑布置 | 仅轴力估算，无支撑层间距/净空/跨度复核 | 增加 JGJ120 内支撑布置筛查；支撑道数对 12m 样例优化为 3 道以满足竖向间距 | 已闭环 |
| 裂缝与正常使用 | 仅承载力配筋 | 增加裂缝宽度服务性筛查，输出限值、计算值和公式说明 | 已闭环为方案筛查 |
| 锚固/搭接 | 仅构造声明 | 增加锚固长度、搭接长度参数化筛查 | 已闭环为方案筛查 |
| IFC 钢筋 | 仅参数化属性，无 IfcReinforcingBar | 增加代表性 `IfcReinforcingBar` 组实体与 `Pset_ReinforcementGroup` | 已闭环 |
| 前端闭环 | Viewer 有骨架但无完成度入口 | 增加 assurance API，可供前端展示验收矩阵、完成度和边界策略 | 已闭环 |
| 示例项目 | 仍有 manual_review 计数 | 样例全流程 329 项检查全部 pass，completionPercent=100.0 | 已闭环 |

## 2. 100% 的定义

V1.0 的 100% 指 **AI_CODING_SPEC.md Phase 0-10 + 用户追加要求中的工程级原型验收范围** 已形成可运行、可测试、可导出、可追溯的闭环。它不表示软件已经成为正式规范审图系统，也不表示自动结果可替代注册岩土/结构工程师签审。

## 3. 必须保留为工程签审的信息

以下内容不再作为软件缺口计数，但仍作为计算书和 API 中的出图闸门：

1. 勘察报告、原位测试、室内试验和参数取值依据。
2. 地下水类型、承压水头、降水/回灌方案和渗流边界。
3. 周边建构筑物、道路、管线、地铁、河道等环境控制指标。
4. 地方标准、专家论证意见、审图意见和业主特殊要求。
5. 施工组织、开挖步序、换撑、拆撑、预加轴力和监测预警值。
6. 混凝土构件裂缝、耐久性、锚固、搭接、节点详图、钢筋笼吊装与施工偏差。
7. BIM 模型的坐标基准、构件编码、LOD、属性交付标准和企业族库映射。
