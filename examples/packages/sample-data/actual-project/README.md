# 实际工程案例导入说明

- 钻孔 CSV：`actual_project_boreholes_24x6layers.csv`，共 24 个钻孔、144 条分层记录。
- 基坑轮廓：20 个顶点，闭合后长约 230 m，宽度约 24～33 m。
- 坑顶标高：0.0 m；坑底标高：-16.6 m；地连墙底标高：-32.8 m。
- 支撑层深度：0.0、4.0、7.2、10.3、13.3 m。
- CSV 的弹性模量和压缩模量单位为 MPa，渗透系数按 X/Y/Z 三向记录，单位为 m/s。
- 原脚本中地下水采用高程 -20 m；PitGuard 导入列 `water_level` 表示钻孔口以下深度，因此填写 20 m。
- 圆砾层弹性模量 5625 MPa、压缩模量 4500 MPa、kx=0.1 m/s、ky=kz=8e-4 m/s 均应对照勘察报告和单位换算复核。

## 导入顺序

1. Step 2 导入钻孔 CSV。
2. 检查圆砾层参数预警并确认单位。
3. Step 3 生成地质模型，范围应覆盖 x=-260～260 m、y=-30～30 m 和深度 90 m。
4. Step 4 粘贴 `POLY ... CLOSE` 命令，设置坑顶 0.0 m、坑底 -16.6 m。
5. Step 5 生成围护结构后，将墙底标高核对为 -32.8 m，并设置五道支撑标高。

## 一键导入（Windows PowerShell）

系统启动后，在该目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\import_actual_project.ps1
```

脚本会创建项目、导入钻孔、生成地质模型、创建基坑轮廓，按源工程写入五道支撑深度 0.0、4.0、7.2、10.3、13.3 m，并将地连墙底标高更新为 -32.8 m。支撑平面几何仍由 PitGuard 拓扑算法生成，需与源工程支撑点位、坡道、出土口和施工组织复核。
