# 深化设计与启动环境诊断

## 1. 目标与适用边界

V3.8.0 将施工图深化从钢筋加工表继续扩展到节点硬件、钢筋笼吊装、机械连接、预埋件碰撞和施工停检点，并把 Python 依赖诊断纳入 Windows/Linux 一键启动流程。

本模块用于设计深化筛查、图纸组织和质量门禁。高利用率节点仍需项目专项有限元分析；焊接节点仍需焊接工艺评定；钢筋笼吊装仍需结合实际吊机、吊具、场地和施工组织编制专项方案。

## 2. 节点硬件参数化模型

每个支撑—围檩节点生成以下对象：

- 承压板：宽、高、厚度、材质、承压应力和利用率；
- 加劲板：数量、厚度、高度、长度和对称布置；
- 焊缝：焊缝类型、焊脚/有效厚度、有效长度、质量等级和检测要求；
- 锚筋：直径、数量、锚固长度和筛查承载力；
- 预埋件实体包络：用于钢筋和施工净空碰撞检查。

钢筋混凝土支撑节点与钢支撑节点采用不同的荷载路径：

- 钢筋混凝土支撑的主轴力通过混凝土承压和纵筋锚固传递，围焊按预埋板组装与次生作用筛查；
- 钢支撑的完整轴力通过端板焊缝组传递。角焊缝不满足时，自动升级为全熔透坡口焊并提高无损检测要求。

图纸及台账：

- `D-10 节点承压板、加劲板、焊缝与锚筋深化大样`；
- `embedded_item_schedule.csv`；
- `weld_schedule.csv`；
- `stiffener_schedule.csv`。

## 3. 钢筋笼分节、吊装和运输

钢筋笼首先按墙幅水平分板，再按最大运输/吊装长度竖向分节，避免将整面长墙误认为单个钢筋笼。每个笼段输出：

- 墙幅和水平分板范围；
- 竖向分节范围；
- 长度、宽度和估算重量；
- 吊点数量和沿长度方向的比例位置；
- 动载系数、单索拉力和吊筋直径；
- 吊筋利用率和弹性变形筛查；
- 运输长度、宽度和重量包络；
- 临时加强和专项复核建议。

图纸及台账：

- `R-10 钢筋笼吊装、运输与临时加强分析图`；
- `cage_segment_schedule.csv`；
- `cage_hoisting_analysis.csv`。

## 4. 机械连接和接头错开

加工模型将超过定尺或运输长度的钢筋分段。D25及以上钢筋及墙体纵向主筋优先采用机械连接，其余按项目规则采用搭接。系统输出：

- 套筒规格；
- 正丝/反丝配置提示；
- 丝头加工和现场检验要求；
- 接头错开组；
- 检验批和见证抽检要求；
- 钢筋编号、宿主构件和接头位置。

图纸及台账：

- `R-11 机械连接套筒、丝头和接头错开详图`；
- `coupler_schedule.csv`；
- `fabrication_bbs.csv`；
- `fabrication_segments.csv`。

## 5. 预埋件—钢筋碰撞协调

预埋件按三维包络建立空间索引，钢筋按空间中心线建立索引。检查区分：

- 设计连接：节点锚固钢筋与预埋件按节点大样连接；
- 构造协调：常规墙筋、梁筋或支撑筋进入预埋件净空，需要局部绕筋、截断锚固或开孔；
- 硬碰撞：非关联钢筋穿越预埋件实体或施工净空。

为控制大型工程计算量，碰撞结果按预埋件、宿主、钢筋组和钢筋类型归并。图纸及台账：

- `Q-04 预埋件、钢筋和施工净空碰撞检查图`；
- `embedded_item_collision_checks.csv`；
- `deep_detailing_package.json`。

## 6. 施工顺序与停检点

系统根据计算工况和深化对象生成施工顺序，包括：

1. 测量放线和地下障碍复核；
2. 导墙、成槽、钢筋笼吊装和水下混凝土；
3. 冠梁和首道围檩；
4. 各开挖、支撑和换撑阶段；
5. 节点承压板、焊缝、预加轴力和支撑轴线复测；
6. 楼板形成后的分区换撑和拆撑；
7. 监测复核、变更归档和竣工模型交付。

墙体施工、节点检查和换撑被标记为停检点，输出至 `construction_sequence.csv`。

## 7. 施工图发行门禁

施工版新增 `DEEP_DETAILING_NO_FAIL` 门禁。以下任一情况会阻断施工版：

- 承压板、焊缝或锚筋出现硬失败；
- 钢筋笼吊装筛查出现硬失败；
- 非关联钢筋与预埋件发生实体硬碰撞；
- 必需深化图或深化台账缺失。

协调警告允许生成带水印审查版，不能被配置静默清除。

## 8. 当前 Python 环境依赖诊断

启动脚本使用当前已激活的 Python 环境，不创建额外虚拟环境。依赖检查器：

```bash
python scripts/check-python-env.py --format text
```

检查器读取 `services/api/pyproject.toml`，逐项验证可导入模块。二进制包导入异常同样视为不可用。

可输出：

```bash
python scripts/check-python-env.py --format json
python scripts/check-python-env.py --format missing
python scripts/check-python-env.py --format install-command
python scripts/check-python-env.py --format editable-command
```

缺少依赖时，Linux和Windows启动脚本会：

1. 列出缺失或损坏的包；
2. 默认尝试在当前Python环境执行 `pip install -e services/api`；
3. 安装失败或关闭自动安装时退出；
4. 退出前打印只包含缺失包的、可直接复制的安装命令。

关闭自动安装：

```bash
PITGUARD_INSTALL_DEPS=0 bash start-linux.sh
```

Windows PowerShell：

```powershell
$env:PITGUARD_INSTALL_DEPS="0"
.\start-windows.ps1
```

后端默认端口为 `8002`，可通过 `PITGUARD_BACKEND_PORT` 修改。
