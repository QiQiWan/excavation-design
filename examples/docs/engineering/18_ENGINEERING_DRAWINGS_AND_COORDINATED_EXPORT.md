# 工程施工图体系与协同成果导出

## 1. 目标

V3.13 将图纸生成和成果导出拆分为两个相互校验的工程子系统。图纸子系统负责表达设计意图、构件定位、配筋构造、施工阶段和节点做法；成果导出子系统负责组织 CAD、PDF、IFC、DOCX、钢筋加工数据、项目快照、审签证据和文件哈希。

图纸生成成功只说明文件已经产生。施工发行还需要通过工程计算、逐图表达质量、图种完整性、钢筋深化、审签、修订和现场条件门禁。

## 2. 图纸体系

### 2.1 图纸族

| 图纸族 | 核心内容 | 主要使用者 |
|---|---|---|
| G 总说明与追溯 | 图纸目录、设计说明、图纸—模型—计算—规范矩阵 | 设计、校核、审查 |
| S 结构与施工阶段 | 围护总平面、墙幅分幅、支撑分层平面、横纵剖面、施工阶段 | 设计、施工、测量 |
| R 配筋深化 | 墙体、支撑、围檩配筋，钢筋笼、套筒、吊装和下料索引 | 结构设计、翻样、加工 |
| D 节点详图 | 支撑—围檩、角撑、立柱、墙幅接头、止水、锚固、施工缝 | 设计、施工、质检 |
| Q 质量证据 | 逐图质量、完整性、碰撞、长期效应和发行门禁 | 校核、审核、批准 |

### 2.2 V3.13 新增或重构的关键图纸

- G-01：图纸—模型—计算—规范追溯索引。
- S-01：地下连续墙分幅、轴线与控制坐标图。
- S-03：横向控制剖面与围护嵌固图。
- S-04：纵向控制剖面、地下水与支撑分层图。
- S-05：开挖、支撑安装、换撑与拆撑施工阶段图。
- R-01：墙体配筋总图，明确双面主筋、水平分布筋和控制配筋表。
- R-02：墙体分区配筋立面，增加实际钢筋线、分区尺寸和支撑区索引。
- R-04：支撑配筋总图，增加端部加密区、跨中区、箍筋和接头分区。
- R-05：围檩配筋总图，增加典型截面、节点附加筋和支撑—围檩节点索引。
- D-03：支撑—立柱节点、立面、基础尺寸和设计信息表。
- D-06：墙幅接头、止水和钢筋笼连接详图。
- D-07：支撑端部锚固、施工缝与错开接头详图。
- Q-05：图纸完整性、表达深度和发行质量检查表。

### 2.3 逐图质量门禁

每张 DXF 独立检查：

1. 文件存在且可重新解析；
2. 图号唯一；
3. 必需图层存在；
4. 模型空间实体和文字达到对应图种的最低表达深度；
5. 原生尺寸数量满足要求；
6. 纸空间、锁定视口、图签、图号和比例齐全；
7. 施工模式下任一失败阻止发行。

输出文件：

- `drawing_sheet_quality.json`
- `drawing_sheet_quality.csv`
- `drawing_completeness.json`
- `construction_issue_gate.json`

逐图质量检查只覆盖表达、可解析性和发行完整性。结构安全由计算结果、规范规则和专业审签控制。

## 3. 图纸追溯

每张图通过以下链路绑定设计证据：

`图号 → renderer → 模型对象 → 控制计算 → 直接规范 → 规则 ID → 当前检查状态 → 输出文件`

机器可读文件为：

- `drawing_model_calculation_standard_matrix.csv`
- `drawing_set_manifest.json`
- `drawing_register.csv`

旧项目保存的 V3.12 规则会在打开时合并 V3.13 新增核心图纸。原 S-01 兼容支撑平面规则自动迁移为墙幅分幅和控制坐标图，各层支撑平面继续由 S-02-Lxx 表达。

## 4. 协同成果交付包

新增接口：

`GET/POST /api/projects/{project_id}/export/coordinated-delivery-package`

目录结构：

```text
00_release/  发行清单、移交单、验收矩阵、网页索引
10_drawings/ 正式图纸包和快速审查文件
20_bim/      IFC 多配置模型及兼容性检查
30_reports/  DOCX 计算书
40_rebar/    钢筋加工深化 ZIP
50_data/     项目快照、方案台账和墙长优化报告
90_audit/    关系矩阵、SHA256 和离线校验脚本
```

### 4.1 快速审查文件

正式图纸 ZIP 中的高频文件会同步提取到 `10_drawings/quick_review`，包括：

- 批量 PDF；
- 图纸目录；
- 逐图质量报告；
- 图种完整性报告；
- 施工发行门禁；
- 图纸—模型—计算—规范矩阵。

### 4.2 发行等级

| 等级 | 含义 |
|---|---|
| development_only | 存在硬失败、缺项或无效审签，只能继续设计 |
| review_complete | 成果已形成且无生成硬失败，可用于专业审查 |
| construction_issued | 施工模式门禁通过、当前快照审批有效，可进入正式发行流程 |

### 4.3 完整性和防篡改

交付包包含：

- `release_manifest.json`：版本、快照、成果、质量和发行状态；
- `issue_transmittal.csv`：移交批次和状态；
- `deliverable_register.csv`：文件用途、受众、大小和 SHA256；
- `acceptance_matrix.csv`：计算、图纸、钢筋、BIM、审签和现场条件；
- `deliverable_relationship_matrix.csv`：成果间追溯关系；
- `SHA256SUMS.txt`：全部文件哈希；
- `verify_delivery_package.py`：无需第三方依赖的离线校验脚本。

## 5. 使用建议

设计人员先查看 `00_release/index.html`，再按验收矩阵处理失败和人工复核项。校核人员使用 G-01 和追溯矩阵抽查控制构件。施工和翻样人员从正式图纸包与钢筋深化包共同获取信息。BIM 人员按用途选择 IFC profile。归档人员在移交前后分别运行哈希校验。

## 6. 仍需项目化输入

自动出图无法替代以下项目资料：企业图框和签审栏、实际墙幅施工顺序、接头工法、防水等级、支撑预加力、换撑拆撑方案、坡道和出土口、邻近建构筑物与管线、吊装设备、监测方案以及地方标准。缺失时系统保留人工复核或发行阻断项。
