# 一般多边形围护设计、计算状态一致性与地质设计域

## 1. 适用范围

V3.15 将围护结构平面设计从“轴对齐矩形的短边对撑”扩展为单一闭合、无孔多边形的通用流程。当前自动回归覆盖旋转矩形、梯形、L 形和 U 形，并保留圆形/竖井型方案的独立判别。曲线边界应先离散为足够密集的折线。

多基坑、带内岛或孔洞、自交轮廓、分离式围护体系及复杂曲面墙仍需专用几何模型，当前版本不得将其自动简化为普通单环多边形后直接发行。

## 2. 丰收湖项目异常的根因

丰收湖截图中的 503.95 mm 位移和 20～46 个 Fail 并非同一最新拓扑的稳定计算结论。主要原因包括：

1. 采用或自动修复支撑方案后，旧计算结果和候选方案完整计算仍保留在活动结果区；
2. 候选方案评分基于未经过凹角回墙和围檩支点间距预检的原始拓扑，卡片 Fail 数与最终采用方案不一致；
3. 施工阶段按列表顺序复制支撑 ID，拓扑变化后可能继续引用已删除构件；
4. 原平面算法依赖全局 X/Y 包围盒，旋转或凹形基坑会出现错误长短边、跨越凹口或错误环撑判定；
5. 点在多边形判断的分母符号错误，使下降边附近的内外判定不稳定；
6. 大型候选同时并行计算时，NumPy/BLAS 线程争用导致长时间停顿，前端容易继续显示旧结果。

## 3. 计算快照和状态门禁

每个计算结果写入当前支撑拓扑哈希：

```text
H = hash(基坑几何, 墙体, 支撑, 立柱, 地质, 施工工况)
```

发生以下任一变化时，活动结果立即失效：

- 基坑轮廓、标高或墙体变化；
- 支撑、围檩、节点或立柱变化；
- 采用 A/B/C 方案；
- 钻孔、地层或地质面变化；
- 施工工况中的支撑激活关系变化。

失效处理包括：

1. 将旧结果的摘要移入 `invalidatedCalculationArchive`；
2. 清空活动 `calculationResults`；
3. 清空候选方案 `fullCalculation` 和完整比选表；
4. 按阶段类型和最接近标高重建施工工况；
5. 设置 `requiresRecalculation=true`；
6. 阻止界面、计算书和出图闸门继续使用旧位移、内力和 Fail 数。

V3.14 及更早结果没有拓扑哈希，加载后自动归档并要求重算。

## 4. 一般多边形平面识别

### 4.1 局部主轴

采用边界长度加权的协方差矩阵识别局部主轴：

```text
C = Σ l_i (p_i-p̄)(p_i-p̄)^T / Σ l_i
C v = λ v
```

支撑方向、分仓站位和对称性均在局部主轴坐标系中计算，避免旋转基坑被误判为超长 X/Y 跨。

### 4.2 形状诊断

诊断量包括：

- 凹角数量和凹凸性；
- 局部长跨、短跨和长宽比；
- 平面圆形度；
- 边界点相对中心的径向变异系数；
- 是否属于竖井/近圆形闭合体系。

近方形大基坑不会只因长宽接近就自动采用环撑。环撑需要同时满足近圆形、低径向离散和无明显凹角等条件，或由用户明确选择中心岛/环撑体系。

### 4.3 支撑线坑内包含门禁

每条候选支撑中心线沿全段采样：

```text
segment(t) ∈ Ω, 0 < t < 1
```

只要中心线穿过 U 形凹口、L 形外部空区或其他坑外区域，即判定 `support_outside_excavation=fail`。该门禁同时进入候选方案硬约束、自动修复和正式发行检查。

### 4.4 凹角和短回墙

短回墙没有直接支撑端点时，允许的分析路径必须满足：

- 两侧相邻主墙具有有效支撑；
- 闭合围檩和转角节点可提供明确传力；
- 节点构造和局部加强进入专项复核。

满足时标记为 `indirect_corner_transfer`，不再错误计为“整面墙无支撑”；无法形成上述路径时继续为 Fail。

### 4.5 围檩补撑的可施工性门禁

一键围护设计在 Step 5 内执行围檩支点间距审计。对超限跨优先从目标墙面沿局部法向搜索另一有效墙面，方向偏转角仅作为次级方案。此前按最短射线选取时，靠近凸角的 60°～75°近切向支撑会在两端各退让 1 m 后坍缩为 1～2 m 的伪构件。

当前补撑候选必须同时满足：

- 两端中心线净距达到目标值的 90% 以上；
- 退让后的有效长度不小于最小角撑腿长；
- 支撑中心线全段位于基坑多边形内部；
- 与坡道、出土口、保护区和既有构件不发生硬冲突；
- 补撑后围檩最大无直接支点跨不超过硬上限。

无法满足上述条件的短角撑不进入模型，对应短墙通过闭合围檩和角部刚性节点形成间接传力，并保留节点专项复核要求。

## 5. A/B/C 候选设计

候选评分前统一执行：

1. 凹角回墙支撑增补；
2. 围檩支点间距修复；
3. 支撑线坑内包含检查；
4. 非环形支撑交叉消除、T/Y 节点和临时立柱重建；
5. 障碍、坡道和保护区检查；
6. 强度拓扑预检。

因此，卡片的 Fail/Warning、支撑数量和代理评分对应实际将被完整计算的拓扑。候选完整计算继续单独显示最大轴力、最大位移、围檩内力和正式发行闸门。大型模型自动采用串行候选计算，避免底层数值库过度并发；小型模型保留并行执行。

同一强度拓扑预检也已前移到“一键生成围护体系”。因此，Step 5 当前方案、A/B/C 代理评分和 Step 6 完整计算使用相同的支撑覆盖、围檩支点间距和坑内包含约束，不再等待计算失败后才增补构件。

## 6. 施工工况同步

施工阶段不再按原列表位置复制支撑。同步规则按以下顺序匹配：

1. 阶段类型；
2. 开挖或安装标高；
3. 当前支撑层级；
4. 当前有效构件 ID。

旧 ID 被移除，新增支撑按语义加入对应安装阶段。同步证据写入计算诊断；如仍存在无法映射的人工工况，保留 Warning 并要求复核。

## 7. 地质设计域自动扩展

### 7.1 设计域

地质模型必须覆盖：

```text
Ω_geo ⊇ buffer(Ω_excavation ∪ Ω_retaining, max(b_min, ηH))
```

其中围护范围包括实际墙段、支撑端点、墙连接点和立柱，缓冲区默认取 10 m 与 0.5 倍坑深中的较大值，可在项目设置中调整。

### 7.2 自动外扩

当地质面平面范围小于设计域时，系统自动重新生成地质网格，保留已导入的 VTU 网格引用，并输出：

- `boreholeTrustBounds`；
- `requiredBounds`；
- `modelBounds`；
- `maximumExtrapolationDistanceM`；
- `maximumAllowedExtrapolationDistanceM`；
- `designDomainCovered`；
- `autoExtended`。

钻孔可信范围内使用 IDW；可信矩形外将查询点夹持到最近可信边界，再计算地层标高。该方式用于避免无界线性趋势和极端外推，外扩区统一标记为低置信度。

### 7.3 工程边界

自动外扩解决“围护结构落在地质模型外部”的几何和软件错误，不能证明外扩区域勘察资料充分。最大外推距离超过项目限值时，覆盖状态为 Fail，必须补充钻孔、人工地质面或专项勘察解释后方可正式设计。

## 8. 回归矩阵

| 场景 | 核心检查 |
|---|---|
| 旋转矩形 | 支撑沿局部短向，跨度不受全局坐标旋转影响 |
| 梯形 | 不依赖平行边假定，支撑端点落在真实边界 |
| L 形 | 不穿越缺口，短回墙形成可追溯传力 |
| U 形 | 横跨开口的支撑被硬门禁拦截 |
| 大型近方形 | 不误判为环撑/竖井 |
| 小钻孔范围、大基坑 | 地质设计域自动外扩并标记低置信度 |
| 拓扑修改后加载旧项目 | 旧结果和候选完整计算自动归档，要求重算 |

V3.15 当前回归结果：旋转矩形、梯形、L 形、U 形和旋转方形均为 `Fail=0`、`supportOutsideExcavationCount=0`，强度拓扑预检状态为 Pass。质量状态保持 Warning，原因是通用样例未录入施工坡道、出土口和障碍区。

## 9. 正式发行条件

满足以下条件后，系统计算结果才可进入施工图发行流程：

- 当前结果拓扑哈希与项目一致；
- 施工工况无失效支撑 ID；
- 支撑中心线全部位于有效基坑域；
- 地质模型覆盖设计域；
- 超限外推、强度、刚度和稳定性无 Fail；
- 障碍、坡道、管线、邻建和施工顺序已录入或完成专业复核；
- 四级审签绑定当前项目快照。
