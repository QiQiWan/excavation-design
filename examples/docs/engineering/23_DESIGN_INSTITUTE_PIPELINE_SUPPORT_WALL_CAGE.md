# V3.20 设计院式围护设计、钢筋笼与成果闭环

## 1. 本轮重构目标

V3.20 将围护结构模型拆分为三个相互关联、用途不同的层级：

1. **计算墙段**：用于土压力、弹性地基梁、围檩反力、墙体内力与稳定计算；边界来自基坑轮廓折点和设计控制面。
2. **施工槽段**：用于地下连续墙成槽、接头、钢筋笼、吊装和施工图表达；一个计算墙段可以分成多个 3～7 m 槽段。
3. **钢筋笼对象**：每个施工槽段对应一套两面主筋、水平分布筋、拉结筋、接头和吊点信息；浏览器采用网格 LOD，完整数量进入 BBS、CSV 和 JSON。

这三个层级不能合并。把 60～111 m 的长计算墙段直接作为一个钢筋笼，会导致施工不可实施；把浏览器中抽样的几根钢筋当成真实配筋数量，会造成严重误判。

## 2. 水平支撑方案逻辑

### 2.1 先选体系族，再生成构件

系统根据平面长宽比、凹凸性、跨度、开挖深度和施工空间，先选择支撑体系族：

- 狭长基坑：短跨直对撑为主，转角墙—墙角撑作为局部补强；
- 近方形宽大基坑：明确选择双向框架或环撑—径向撑；
- 凹形基坑：按凸分区或施工分区形成直接传力子体系；
- 普通非环形体系：禁止支撑端点随意落在另一根普通支撑跨中；
- 受控 T/Y 节点：仅用于明确的双向框架、边桁架或凹角回墙，并配置竖向承托和节点验算。

### 2.2 丰收湖的布置策略

丰收湖为狭长、阶梯状基坑，当前采用：

- 5 个支撑标高；
- 沿局部短跨方向连续布置主对撑；
- 平面台阶和转折影响区加密站位；
- 转角处设置直接墙—墙角撑；
- 围檩支点超限时优先增加直接墙—墙支撑，不采用普通支撑跨中补撑；
- 所有非环形支撑执行零贯穿交叉检查。

V3.20 丰收湖自动结果：260 根支撑、40 根临时立柱、最大围檩直接支点间距 7.445 m，普通支撑单墙端点数量为 0。上传的 PLAXIS 参考脚本实际包含 5 层共 191 根支撑，因此自动方案仍偏保守，应将参考方案作为独立候选进行同口径完整计算，而不应直接将 260 根方案视为最终设计。

### 2.3 实际支撑方案导入

系统支持将设计院或 PLAXIS 已有支撑坐标表作为 Reference 方案导入：

`POST /api/projects/{project_id}/design/import-support-layout?replace=true`

CSV 至少包含 `levelIndex、elevationM、startX、startY、endX、endY`。导入后重新识别墙面连接点、坑内包含、普通交叉、有效无侧向长度、围檩支点和临时立柱。导入不会把几何方案直接视为结构计算通过；当源文件只有 EA、缺少截面和节点参数时，系统标记 `placeholder_requires_member_design`，要求完成截面映射、施工阶段同步和完整计算。丰收湖参考方案导入为 191 根支撑，普通交叉和坑外构件均为 0，围檩最大支点间距 9.0 m，属于可继续深化的 warning 状态。

## 3. 围护墙长度的两种含义

### 3.1 平面长度

平面长度必须体现基坑轮廓差异。系统保留 20 个计算墙段，并分解为 98 个施工槽段。槽段长度按项目设置控制在约 3～7 m，短折返墙可以保留更短的异形槽段，并进入吊装与接头专项复核。

### 3.2 竖向长度

墙趾标高是否分区，必须由稳定、渗流、承压水、软弱层和环境变形控制结果提供证据。丰收湖源模型中所有墙体采用共同墙底 -32.8 m，24 个钻孔的主控层界面也采用一致标高。当前筛查得到的墙趾需求高差为 0.000 m，小于 0.75 m 的分区触发值，因此保持统一墙趾。

系统支持 uniform、zoned 和 local 三种墙趾设计模式，但导入或人工锁定的较深墙趾不会被自动缩短。

## 4. 完整钢筋笼表达

每个施工槽段生成一个钢筋笼描述对象，包含：

- 坑内侧、坑外侧竖向主筋直径、间距和数量；
- 两面水平分布筋；
- 拉结筋或连接筋；
- 保护层、墙厚、笼宽和笼高；
- 槽段起止里程和接头类型；
- 吊装复核状态；
- 关联的深度配筋区和平面配筋区。

三维浏览器同时提供两种表达：

- **钢筋笼网格 LOD**：默认显示两面纵横钢筋网和拉结筋，用于识别钢筋笼形态；
- **可选择钢筋样本**：用于构件属性、碰撞和定位，数量受性能预算限制。

真实加工数量必须从钢筋表、BBS、CSV 和完整 JSON 读取。V3.20 丰收湖项目生成 98 个钢筋笼对象，完整钢筋估算数量约 166488 根，浏览器样本数量不代表加工数量。

## 5. 八阶段设计与发行流程

1. **P1 设计依据与设计域**：勘察、地下水、轮廓、周边环境和地质覆盖。
2. **P2 支护体系与施工分幅**：体系族、计算墙段、施工槽段、支撑层和施工阶段。
3. **P3 候选方案完整计算与选型**：A/B/C 独立分阶段计算，明确采用方案；代理值不能进入选型。
4. **P4 分阶段计算与规范校核**：土水压力、墙、围檩、支撑、立柱、稳定和数值质量。 分层渗流筛查输出 `seepageRiskIndexMax`（越小风险越低），不得与嵌固、抗隆起安全系数混为同一最小安全系数。
5. **P5 构件与墙趾联合设计**：截面、配筋、墙趾、支点和节点联合迭代。
6. **P6 钢筋笼与节点深化**：钢筋笼、接头、套筒、吊点、施工缝和碰撞检查。
7. **P7 成果生成**：当前已选方案和当前计算快照生成 CAD、PDF、IFC、DOCX、XLSX 与 JSON。
8. **P8 校审与发行**：设计、校核、审核、批准、警告闭环和正式发行。

API：`GET /api/projects/{project_id}/expert-design/pipeline`。

## 6. 成果生成约束

- 几何代理结果不能作为计算内力；
- 三维 LOD 不能作为加工数量；
- 旧候选和旧计算不能进入图纸；
- 计算墙段不能直接等同于施工槽段；
- 图纸至少应包含总平面、控制剖面、钢筋笼、节点和立柱基础核心图种；
- 正式发行包必须绑定项目快照、方案 ID、计算结果 ID、图纸规则集和文件哈希。

## 7. 当前工程边界

V3.20 可完成方案和构件级深化，但以下事项仍需项目设计人员确认：

- 槽段接头工法、防水等级和成槽顺序；
- 钢筋笼吊装设备、分段、桁架筋和吊点；
- 支撑预加轴力、温度、偏心和节点构造；
- 坡道、出土口、邻近建筑和管线；
- 拆换撑时序与永久结构强度形成时间；
- 地方标准、企业标准图和审图意见。
