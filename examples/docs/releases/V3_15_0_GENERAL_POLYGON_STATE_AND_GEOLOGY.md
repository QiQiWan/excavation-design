# V3.15.0 一般多边形、计算状态与地质设计域迭代

## 版本目标

解决丰收湖项目在支撑拓扑更新后仍显示旧位移和旧 Fail 数的问题；将支撑布置从全局包围盒启发式升级为一般多边形局部主轴设计；当地质模型小于围护结构和施工影响区时自动扩展模型范围。

## P0

- 计算结果绑定支撑拓扑哈希；旧结果自动归档并阻断使用。
- A/B/C 方案采用、几何修改、地质修改后清空候选完整计算。
- 施工阶段按类型和标高同步支撑，消除失效构件 ID。
- 修复点在多边形判断的方向符号错误。
- 增加支撑中心线离开基坑域的硬门禁。
- 丰收湖当前设计复算：Fail 0，最大墙体位移 3.418 mm。

## P1

- 引入边界加权局部主轴和形状诊断。
- 支持旋转矩形、梯形、L 形、U 形和近圆形判别。
- 凹角回墙和围檩支点间距修复在候选评分前执行。
- 大型 A/B/C 完整计算自动串行，小型模型并行。
- 短回墙区分直接支撑、闭合围檩间接传力和真实缺失支撑。
- Step 5 一键生成围护体系即执行强度拓扑预检，原始方案不再带着可自动修复的围檩硬超限进入 Step 6。
- 局部围檩补撑优先选择接近墙面法向的传力路径；净距退让后长度不足或中心线离开坑内的候选自动剔除。

## P2

- 地质设计域包含基坑、实际围护结构和施工影响缓冲。
- 钻孔范围不足时自动扩展 IDW 网格，可信边界外采用夹持外推。
- 增加地质覆盖 API、界面证据和计算硬门禁。
- 在线文档增加一般多边形、状态一致性和地质设计域章节。
- GB 55017-2021 纳入勘察与地质流程矩阵。

## 工程边界

当前几何核支持单一闭合无孔多边形。多基坑、内岛、孔洞、自交轮廓和复杂曲线边界仍需专用表示。地质自动外扩仅维持模型连续性，低置信度外推不能替代补充勘察。

## 本轮回归结果

旋转矩形、梯形、L 形、U 形和旋转方形的一键围护设计均完成强度拓扑预检，支撑中心线坑外数量为 0，围檩硬超限数量为 0。当前统一保留的 Warning 来自障碍物、坡道、出土口等项目条件尚未录入，不能由纯几何算法自动消除。
