# PitGuard V3.25.0：平行角撑体系与应用登录入口修复

## 1. 施工图角撑识别

上传施工图中的端部角撑采用相邻两面围檩之间的独立平行斜撑。每根斜撑在两侧围檩上分别设置支承节点；同一角区的多根斜撑保持近似平行并沿墙面错开。角撑区与中部直对撑区分开，转角局部形成连续三角形传力体系。

不允许采用多根斜撑向同一墙点汇聚的 V 形或扇形布置。该形式会产生围檩局部集中反力、承压板和钢筋拥挤、斜撑夹角不一致，以及轴压构件传力路径不清晰等问题。

## 2. 原算法缺陷

- 围檩超限修复固定目标墙站位，再从相邻墙搜索端点，多次修复会向同一墙点发散。
- 初始角撑和围檩修复斜撑属于两套生成逻辑，可能在同一角区叠加。
- 原质量指标主要检查准确重合节点，无法识别距离很近但未完全重合的墙节点拥挤。
- 支撑中心线向坑内退让后，旧流程重新投影墙连接点，造成墙面站位漂移和虚假的围檩超限。
- 方案排序缺少角撑族平行度与墙节点最小间距硬约束。

## 3. V3.25.0 修复

- 采用共同转角等链距方法生成平行角撑族。
- 默认每个合格角区生成 2 根角撑，可配置 1–3 根。
- 每根角撑两端使用独立围檩节点，默认节点间距 3.0m。
- 围檩超限修复改为增加平行族成员，不再生成 V 形扇撑。
- 保留支撑中心线退让前的真实墙连接站位。
- 新增角撑平行度、节点拥挤和扇形异常检查。
- 新指标进入候选硬约束、词典序排序、联合墙长优化和施工版发行闸门。

## 4. 登录弹窗修复

生产脚本会在写入新 Nginx 配置前，扫描 `/etc/nginx/conf.d` 与 `/etc/nginx/sites-enabled`，删除其他文件中属于 `designer.eatrice.cn` 的旧 server 块并备份。新虚拟主机显式设置 `auth_basic off`。

启动完成后脚本同时检查本机虚拟主机和公网域名响应头。只要存在 `WWW-Authenticate`，部署立即失败并提示继续排查旧 Nginx、上游代理或 CDN。

未登录用户访问根域名时，前端会自动进入：

```text
/login?reason=required
```

不会再出现浏览器 Basic Auth 弹窗。

## 5. 部署

```bash
cd /opt/excavation-design
sudo bash start-linux.sh
```

验证：

```bash
curl -kI https://designer.eatrice.cn/ | grep -i WWW-Authenticate
curl -s https://designer.eatrice.cn/api/auth/status
```

第一条命令应无输出；第二条应包含 `"loginRequired":true`。
