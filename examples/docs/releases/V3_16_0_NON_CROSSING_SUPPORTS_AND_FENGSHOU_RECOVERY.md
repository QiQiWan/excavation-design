# V3.16.0 普通水平支撑零穿越与丰收湖计算恢复

## 1. 迭代目标

V3.16.0 解决丰收湖项目中“墙体强度和刚度显示通过，但总体计算仍有 20 项 Fail”的状态冲突，并将普通水平支撑的布置规则统一为同层零穿越。明确采用环梁—径向撑体系时，环撑节点按专用拓扑处理；普通主对撑、次对撑、角撑和局部补撑均不得以跨中连续穿越的方式相交。

## 2. 丰收湖 20 项 Fail 的根因

20 项硬失败主要来自 `QUALITY-SUPPORT_CROSSING`。旧算法允许在交叉位置设置临时立柱后，将两根连续杆件的跨中穿越判为可接受。该处理在结构计算中能够形成节点约束，但施工图和实际构造仍表现为两根支撑贯穿交叉，未满足普通内支撑体系的清晰传力、节点构造和施工可实施性要求。

同时存在以下实现缺陷：

1. 支撑内部 T/Y 节点可能被最近墙段算法误识别为围护墙端点，污染支点语义和分担宽度；
2. 围檩支点修复和凹角回墙修复完成后，后续净距调整可能重新引入穿越；
3. 被主支撑截断的次构件在过滤后可能留下失去墙端或节点承托的悬空端；
4. 大型近方形基坑仍可能退化为单向网格，局部围檩支点间距较大；
5. 旧计算结果只核对部分几何哈希，算法和规则集升级后仍可能被显示为当前结果；
6. 诊断界面没有把支撑穿越从强度、刚度问题中独立分类。

## 3. 非环形水平支撑的节点契约

### 3.1 普通支撑

普通主对撑、次对撑、角撑和局部补撑遵循以下规则：

- 同一支撑层内不得出现两根连续杆件的 proper crossing；
- 次对撑或角撑遇到主支撑时，在首个有效交点终止；
- 终止点形成 T/Y 节点，并设置临时立柱或经设计确认的竖向承托；
- 支撑端部必须连接围檩、环梁或有效 T/Y 节点；
- 净距调整后重新检查坑内包含、有效长度、节点连接和交叉关系；
- 任何普通支撑穿越均作为硬失败，设置交叉立柱不能将其降级为通过。

### 3.2 环梁—径向撑体系

只有项目明确选择环梁—径向撑、中心岛或等效环形传力体系时，才采用环撑节点规则。环形体系仍需验证：

- 环梁闭合；
- 径向构件与环梁节点一致；
- 节点局部承压、剪切和构造；
- 施工顺序、中心岛或竖向支承；
- 与坡道、出土通道和地下结构的冲突。

## 4. 代码实施

### 4.1 支撑几何生成

`support_layout.py` 增加非交叉几何核：

- 检测同层 proper crossing；
- 将分支支撑裁剪至首个既有主支撑节点；
- 按优先墙端方向生成 T/Y 支撑；
- 最终净距调整后再次执行交叉清理；
- 迭代删除失去墙端或 T/Y 节点的悬空构件；
- 大型近方形基坑采用非交叉双向 T/Y 网格候选。

### 4.2 端点语义

内部节点只有在墙面捕获容差内才可登记墙段编号。端点落在其他支撑内部时，登记为 T/Y 节点；最终保留构件集合形成后重新计算端点语义，清除已经失效的临时关系。

### 4.3 质量与碰撞门禁

`support_layout_quality.py` 和 `collision_service.py` 采用同一契约：

- 普通 proper crossing：Fail；
- 带临时立柱的 T/Y 端点连接：允许；
- 明确的环撑—径向撑节点：按环撑规则检查；
- 坑外穿越、障碍冲突和悬空端：Fail。

质量结果增加 `nonRingCrossingCount`，便于计算页、报告和发行门禁直接读取。

### 4.4 结果状态

项目加载时同时核对：

- 支撑拓扑哈希；
- 施工阶段构件 ID；
- 算法版本；
- 规则集版本。

任一不一致时，旧结果自动归档并标记为失效，必须按当前拓扑重新计算。

## 6. 剩余 Warning

剩余 5 项均属于工程输入和施工组织边界：

1. 第一道支撑接近坑顶，需确认冠梁、栈桥顶撑或独立首撑定位；
2. 支撑端点、节点偏心、温度和预加轴力需要深化；
3. 预加力、温度、间隙和安装偏差仍属于快速筛查；
4. 支撑安装、预加力、换撑和拆撑时序需与楼板强度形成时间及监测反馈对应；
5. 坡道、出土口、中心岛、保护区、管线和邻近建筑尚未完整录入。

因此计算硬失败已经消除，正式施工图发行仍需完成项目条件和审签闭环。

## 7. 一般形状适用性

本轮规则适用于单一闭合、无孔的直线或充分离散曲线多边形，包括旋转矩形、梯形、L 形、U 形和近方形基坑。以下类型继续要求专用拓扑：

- 多个分离基坑；
- 带内岛或孔洞的多环基坑；
- 中心岛、环撑、桁架和盖挖体系；
- 倾斜墙、退台墙或明显三维高差；
- 自交轮廓和离散精度不足的曲线墙。
