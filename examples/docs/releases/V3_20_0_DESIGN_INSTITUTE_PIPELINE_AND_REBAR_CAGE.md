# PitGuard V3.20.0

V3.20.0 将支撑体系选择、计算墙段、施工槽段、完整钢筋笼、墙趾证据和成果发行重构为统一的设计院式流程。

主要变化：

- 支撑生成由规则化铺线调整为“体系族选择—站位设计—转折区加密—直接传力复核—完整计算”；
- 狭长基坑保持短跨主对撑连续性，角撑作为局部补强，不再删除端部主对撑后形成大围檩跨；
- 计算墙段与施工槽段分离，丰收湖 20 个计算墙段形成 98 个施工槽段；
- 每个槽段生成完整钢筋笼网格描述，浏览器显示两面主筋、水平筋和拉结筋；
- 墙趾统一或分区由稳定与地质证据决定，禁止为视觉效果制造无依据高差；
- 新增八阶段设计流水线及 API；
- 新增实际支撑 CSV 导入 API，可将设计院或 PLAXIS 已有方案作为 Reference 候选，导入后重建端点、节点、立柱并执行拓扑和围檩审计；
- 修正分层渗流结果语义：风险指标改为 `seepageRiskIndexMax`，不再误标为安全系数或参与最小稳定安全系数统计；
- 候选方案未完成独立计算或未明确采用时，P3 保持 warning；
- 核心施工图图种不完整时，P7 保持 warning；
- 丰收湖复算 Fail 0，强度、刚度和稳定性通过，正式发行仍受候选选型、核心图纸和审签控制。
