# V3.2.0 配筋失败诊断、支撑拓扑与人机交互迭代

## 1. 本轮目标

V3.1.0 已具备分区配筋、钢筋三维显示和 CAD 成套输出能力，但公开基准算例仍出现配筋失败。V3.2.0 对失败链路逐层追踪，修复计算工况、支撑传力、截面设计、节点承压和操作界面之间的断点。

## 2. 配筋失败的真实原因

### 2.1 施工阶段引用了失效支撑 ID

自动修复或采用候选方案会重建支撑对象，支撑 ID 随之变化。旧计算工况仍保存原 ID 时，施工阶段无法激活有效支撑。墙体会被误算为近似无支撑深悬臂，内力和位移被显著放大。

V3.2.0 为支撑拓扑生成哈希，并在计算前审计：

- 当前支撑拓扑哈希；
- 工况保存的拓扑哈希；
- 失效支撑 ID 数量；
- 没有有效支撑的施工阶段数量；
- 支撑层级与换撑层级。

发现不一致时，系统重建默认激活路径，并按阶段顺序保留地下水、超载、分区和换撑操作参数。

### 2.2 长方形基坑端墙缺少直接传力路径

原布置以短跨主对撑为主，长边传力较充分，端墙主要依赖少量角撑。32 m 深、90 m × 45 m 的基准坑中，单根角撑参考分担墙宽约 22.5 m，角撑和端部节点轴力集中。

V3.2.0 对深大、长宽比较高的基坑增加正交次对撑：

- 主对撑负责短跨方向；
- 正交次对撑为端墙建立直接传力路径；
- 主次支撑交点自动设置临时立柱；
- 交叉点作为有支承网格节点处理，不再判定为非法交叉；
- 计算、质量评分、三维、CAD 和配筋表统一识别 `secondary_strut`。

基准算例中，正交次对撑增加 10 根，角撑最大参考分担墙宽由约 22.5 m 降至 9.75 m，最大支撑设计轴力由约 59.5 MN 降至约 25.7 MN。

### 2.3 极限状态轴力被配筋策略重复放大

支撑对象中的 `designAxialForce` 已包含荷载组合和施工效应。原配筋策略再次按保守/均衡/经济系数放大需求，形成重复分项。

V3.2.0 将策略作用改为：

- 保守：目标利用率 0.78；
- 均衡：目标利用率 0.88；
- 经济：目标利用率 0.95。

输入轴力保持为当前极限状态设计值，策略只控制截面和钢筋组合的目标余量。

### 2.4 高弯矩墙体计算存在配筋面积平台化

原受弯快速计算在单筋截面判别式失效后直接限制受压区高度，所需钢筋面积不再随弯矩增长，无法区分“可增加钢筋”和“截面体系需要升级”。

V3.2.0 显式计算单筋界限弯矩。超过界限后进入双筋估算分支，输出：

- 单筋界限弯矩；
- 附加受拉钢筋；
- 所需受压钢筋；
- 截面承载体系升级标志；
- 增厚墙体、双排钢筋或专项截面设计建议。

### 2.5 承压板向下取整造成临界假失败

节点承压板边长按毫米向下取整时，实际面积略小于理论面积，临界节点可能出现承压应力仅高出容量极小数值的假失败。

V3.2.0 按 0.90 目标利用率确定承压面积，并将边长向上规格化到 5 mm。接近截面边界的节点标记为警告，承载不足仍保留失败状态。

### 2.6 配筋目录上限没有给出合理升级路径

V3.1.0 的墙体配筋目录主要覆盖单层钢筋。需求超过目录后只能失败，用户难以判断应增加钢筋、增厚截面或调整支撑体系。

V3.2.0 增加：

- 单层/双层钢筋组合；
- D20～D40、100～225 mm 间距候选；
- 机械连接需求标识；
- 支撑截面与纵筋联合搜索；
- `UPDATE_REBAR_ONLY`、`UPSIZE_SECTION_AND_RECALCULATE`、`ADD_OR_REARRANGE_SUPPORTS` 三类处理动作。

## 3. 配筋诊断闭环

配筋方案现在包含 `diagnostics`：

1. 计算有效性；
2. 支撑拓扑状态；
3. 各配筋类别状态统计；
4. 失败原因分组；
5. 建议操作及优先级；
6. 是否允许应用；
7. 是否允许输出施工图复核包；
8. 是否必须加审查版标识；
9. 截面变更数量。

应用方案时，若支撑截面发生变化，系统自动重新计算构件刚度、支撑轴力和节点承压，再生成最终配筋方案。

## 4. 人机交互优化

配筋工作台收束为四步：

1. 校验计算；
2. 选择策略；
3. 处理问题；
4. 应用与出图。

界面默认采用“仅问题项”过滤，提供构件编号、原因和图号搜索。主要操作区只保留：

- 配筋策略；
- 应用配筋方案或应用截面优化并复算；
- 下载施工图包或审查版图纸。

总图、配筋图和节点大样的分专业下载折叠到次级区域。计算或拓扑未通过时禁用应用按钮；存在阻断项时只能输出审查版图纸。

## 5. CAD 输出优化

新增 D-08《主次支撑网格交叉节点与立柱连接大样》，表达：

- 主对撑和正交次对撑；
- 临时立柱；
- 节点核心区；
- 承压板；
- 主次方向钢筋；
- 预埋件、连接器和净距复核提示。

`90_schedules/` 新增：

- `design_diagnostic_summary.json`；
- `design_diagnostic_summary.csv`。

它们记录计算有效性、支撑拓扑、失败原因、推荐动作和施工图闸门，可供审查台账、计算书和外部系统读取。

## 6. 基准算例结果

| 指标 | 修复前记录 | V3.2.0 |
|---|---:|---:|
| 有效支撑阶段 | 可能因旧 ID 退化为无有效支撑 | 自动同步全部阶段 |
| 正交次对撑 | 0 | 10 |
| 角撑最大参考分担墙宽 | 约 22.5 m | 9.75 m |
| 最大支撑设计轴力 | 约 59.5 MN | 25.7 MN |
| 最大墙体设计弯矩 | 异常工况可达数万 kN·m/m | 基准算例约 1656 kN·m/m |
| 配筋阻断项 | 24 | 0 |
| 施工图闸门 | 关闭 | 开启，仍保留 32 项构造复核 |

上述结果用于软件回归和方案合理性检查，不替代项目级第三方有限元复核及正式施工图审签。
## 7. 整体工作台收束

V3.2.0 将项目工作台划分为两种显示层级：

- **精简模式**：默认启用，仅显示交付状态、下一步动作、当前工作步骤和关键阻断项；
- **专业模式**：展开完整项目树、设计参数、候选方案、指标驾驶舱和高级导出功能。

模式选择保存在浏览器本地。CAD 主入口改为“自动闸门”，后台根据配筋诊断自动生成施工图复核版或审查版，用户不再需要理解多个相似导出按钮之间的差异。
