# PitGuard V3.23.0：支撑交点、墙长联合变量、IFC/配筋/图纸与登录优化

## 1. 审查结论

V3.22.0 已具备“非法穿越最少、内部 T/Y/X 汇交较少”的洁净支撑框架，但围护墙上的支撑端点被排除在汇交统计之外；墙体平面设计段长度和竖向墙趾长度虽已有独立优化器，尚未与支撑拓扑进入同一个候选变量空间。因此，原系统对“墙上多杆共点”和“支撑—墙长联合设计”的落实程度属于部分完成。

V3.23.0 已把上述两项补成生成、评价、排序、采用、重新计算和成果导出的闭环：

- 所有带墙面编号的支撑端点均登记为墙上连接点；
- 单根支撑正常落墙点只做追溯，不施加汇交惩罚；
- 同层两根及以上支撑共用墙/围檩节点计为墙上汇交；
- 三根及以上支撑共用墙节点计为高分支墙节点；
- 支撑候选先比较硬约束，再依次比较非法穿越、墙上高分支汇交、墙上汇交、总汇交和平面复杂度；
- 基坑开挖边界保持固定，逐墙面的计算控制段长度、施工槽段长度与数量、局部加强范围、竖向墙趾长度和支撑拓扑进入联合候选；
- IFC、钢筋笼、施工图和正式发行闸门继承同一组墙段、槽段和连接节点编码。

## 2. 水平支撑交点目标的实现

### 2.1 节点分类

系统现在区分五类平面节点：

1. 非法同层跨中穿越：无合法结构节点的 X 型穿越，属于硬失败；
2. 内部合法汇交：带明确节点语义的 T/Y/X 节点；
3. 墙上单支撑点：一根支撑在墙/围檩上的正常支承点，只记录追溯；
4. 墙上汇交点：两根及以上支撑共用同一墙/围檩节点，计入主要优化目标；
5. 墙上高分支点：三根及以上支撑共用同一墙/围檩节点，采用更高惩罚并进入正式发行审查。

不同支撑层在平面投影上的交叠单独统计，不按同层非法穿越处理，但进入视觉复杂度诊断。

### 2.2 洁净度表达

平面复杂度采用：

\[
C_{plan}=100N_{cross}+N_{internal}+2N_{internal,high}+1.5N_{wall}+3N_{wall,high}+0.2N_{projection}
\]

其中，非法穿越的权重远高于合法汇交，墙上多杆共点的权重高于普通内部 T/Y 节点，体现围檩局部承压、节点板、局部配筋和安装空间的额外复杂度。

候选排序采用词典序：

1. 安全、稳定、坑内包含、端点有效性、围檩支点间距等硬约束；
2. 非法穿越数量；
3. 墙上高分支汇交数量；
4. 墙上汇交数量；
5. 总汇交数量；
6. 平面交叉复杂度；
7. 支撑轴力代理、跨度、对称性、施工通道、构件量和综合工程评分。

这种排序避免通过提高结构风险来换取平面整洁，也避免高综合分方案掩盖明显的交叉与节点拥挤。

### 2.3 前端可审查性

A/B/C 方案卡显示“非法穿越 / 墙上汇交 / 内部汇交”。方案总平面使用圆形节点标记墙上汇交，菱形节点标记内部汇交，并在悬停信息中列出相关支撑编号。联合优化面板同时显示墙长变量、墙趾分区和洁净度指标。

## 3. 围护墙长度作为设计变量

“围护墙长度”需要区分四种工程含义：

- 基坑边界上的墙面物理平面长度：由开挖轮廓决定，在当前优化中作为硬约束固定；
- 结构计算控制段长度：可按墙面、荷载和支撑分布优化；
- 施工槽段长度与槽段数量：可在成槽、接头、防水、吊装和设备能力边界内优化；
- 竖向墙长/墙趾标高：可按嵌固、整体稳定、抗隆起、渗流和软弱层控制优化。

V3.23.0 的联合候选包含：

- 保持现状方案；
- 单墙面控制段/槽段方案；
- 对称墙面对方案；
- 全墙面联合方案；
- 统一墙趾、分区墙趾和保持现状方案；
- 多种支撑拓扑、支撑间距和立柱控制跨度。

联合评分以支撑、平面墙长和竖向墙长为主要分量，并增加槽段过度分割、墙趾分区过多、墙上节点拥挤等施工复杂度惩罚。在同一洁净度等级内，候选列表保留墙面长度方案和支撑体系的多样性。

工程边界：系统不会自动改变开挖轮廓来“缩短围护墙”。开挖边界调整属于方案条件变更，必须由用户明确修改并重新进行地质、环境和稳定复核。

## 4. IFC 模型构建优化

### 4.1 已完成

详细 IFC 模式建立以下层级：

- 计算墙：`IfcElementAssembly`；
- 施工槽段：独立 `IfcWall`；
- 计算墙与施工槽段：`IfcRelAggregates`；
- 相邻施工槽段接头：`IfcRelConnectsElements`；
- 每个施工槽段的钢筋笼：`IfcGroup`；
- 代表钢筋与钢筋笼组：`IfcRelAssignsToGroup`；
- 支撑、围檩、冠梁、立柱、节点板和预埋件：独立构件及属性集；
- 项目属性：记录软件版本、导出模式、墙长变量状态和支撑洁净度指标。

每次导出同时生成 `*.ifc_manifest.json`，包含：

- 计算墙与施工槽段编码；
- 槽段里程、长度、接头类型和吊装复核状态；
- 钢筋笼 IFC Group 与代表钢筋实体；
- 支撑及全部墙上连接点；
- 墙上汇交节点；
- 优化目标和版本信息。

协调轻量版继续保留整段墙和参数化钢筋，控制文件体量；详细版用于语义审查；施工可视化版使用兼容性更好的代理实体；分析版保留轴线、弹簧、荷载和施工阶段信息。

### 4.2 仍可深化

- 使用 IfcOpenShell 或经认证 IFC SDK 替代手写 STEP 生成器；
- 增加工程坐标系、国家坐标系和 `IfcMapConversion`；
- 建立 IDS/MVD 校验规则，约束属性集、编码和对象类型；
- 增加 BCF 问题交换、构件 GUID 稳定策略和模型差异比较；
- 对支撑—围檩、围檩—墙、钢支撑端板等节点建立更精确的连接几何；
- 需要加工级 BIM 时，增加逐根钢筋、弯折形状、套筒和搭接实体，但应与轻量/详细模式分离，避免模型规模失控；
- 继续针对 Revit、Navisworks、Solibri、Bonsai 和 BIMVision 建立真实导入回归矩阵。

## 5. 配筋可视化与钢筋笼深化

### 5.1 已完成

- 每个施工槽段生成独立钢筋笼对象；
- 显示坑内侧、坑外侧竖向主筋、水平分布筋和拉结筋；
- 显示施工接头、吊点、竖向分段和搭接/套筒区；
- 钢筋笼编码继承施工槽段编码；
- 钢筋深化中的笼段、BBS、吊装计划和套筒计划保持同一宿主编码；
- 浏览器样本数量与加工数量分离，完整数量保存在 BBS、CSV、XLSX 和 JSON；
- IFC 代表钢筋按施工槽段归入钢筋笼组，避免钢筋实体与墙段失去关联。

### 5.2 仍可深化

- 基于真实钢筋中心线生成弯折形状和端部锚固；
- 增加接头错开、同截面接头率、机械连接等级和保护层垫块；
- 增加吊筋、加强桁架、声测管、注浆管、预埋件和导管避让；
- 将碰撞检查从包围盒升级为钢筋实体—预埋件—节点板精确距离检查；
- 根据吊装设备、运输长度、笼重和吊点反力生成可执行吊装分段；
- 为超长墙和局部加强区增加可切换的 LOD，避免浏览器一次渲染全部钢筋。

## 6. 施工图导出优化

### 6.1 已完成

施工图包新增：

- `support_junction_schedule.csv`：全部墙上支承点、墙上汇交和内部汇交节点；
- `wall_panel_cage_traceability.csv`：计算墙—墙面设计长度—施工槽段—钢筋笼—图纸索引—IFC 宿主键；
- `cross_artifact_traceability.json`：支撑拓扑、墙长变量、槽段/钢筋笼双向映射和图纸哈希；
- IFC 侧车清单随协同交付包复制到快速审查目录。

正式发行闸门新增：

- 水平支撑非法穿越必须为 0；
- 墙上高分支汇交在审查版形成警告，在施工版形成阻断；
- 施工槽段编码不得重复；
- 每个施工槽段必须存在钢筋笼映射；
- 钢筋笼不得引用不存在的施工槽段；
- 追溯表和跨成果一致性文件必须存在。

- 生成节点详图索引与平面/剖面/节点间双向跳转；
- 对长墙自动分页，并保持轴号、里程和接头编号连续；

- 施工版发行前建立 PDF、DWG/DXF、IFC、计算书、BBS 的统一版本锁定。

## 7. 应用内登录

生产访问已从 Nginx Basic Auth 弹窗改为应用内登录页：

- 后端校验 PBKDF2-SHA256 密码哈希；
- 登录后签发 HMAC 签名、HttpOnly、SameSite 会话 Cookie；
- 前端启动时检查登录状态，未登录显示完整登录界面；
- API 通过 viewer、designer、checker、reviewer、approver、admin 角色执行权限控制；
- 自动化集成仍可使用独立 API Key；
- 一键部署脚本生成管理员账号、密码、会话密钥和自动化 API Key；
- Nginx 不再包含 `auth_basic`，5173/5174 不进入生产链路。
- FastAPI Swagger、OpenAPI、系统诊断和详细就绪信息均要求登录；仅健康检查和认证接口公开；
- GET/POST 兼容导出路由拆分为唯一 OpenAPI Operation ID，消除接口文档冲突；
- 生产前端默认同域请求 `/api`，直接构建也不会残留 `127.0.0.1:8002`。

后续工业安全增强项包括：集中用户数据库、密码修改/重置、失败次数锁定、MFA、OIDC/企业 SSO、会话撤销列表、CSRF 令牌、细粒度项目权限和安全审计告警。

## 9. 工程使用边界

V3.23.0 已落实软件内部的优化、追溯和成果一致性。正式工程签发仍需设计单位完成地质资料审查、结构与稳定计算复核、节点构造审查、第三方软件对标、真实项目验证、施工组织协调和专业审签。系统中的墙长变量不授权自动改变开挖边界，墙上汇交最少也不能覆盖安全、稳定和施工工序等硬约束。
