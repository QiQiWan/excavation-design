# V3.0.0 集成迭代说明

本版本针对 V2.9.0 的几何显示、计算拓扑、并发安全、数据载荷、测试执行和文档结构进行集中收束。

## 主要完成项

### 几何与云图

- 增加统一几何一致性检查和稳定几何哈希。
- 修正墙云图竖向坐标，统一使用实际高程。
- 增加单阶段、全阶段包络、有符号和绝对值显示模式。
- 包络在相同高程选择绝对值控制项并保留符号。
- Three.js 资源在组件卸载时显式释放。

### 计算与优化

- 墙体求解仅接收实际连接当前墙段的支撑。
- 支撑弹簧刚度由材料、截面、长度、方向和构件角色逐构件计算。
- 候选方案优化改用不可变配置对象，消除模块级参数并发覆盖。
- 阶段结果作为墙体采样和全局耦合结果的唯一完整数据源，报告图表区仅保留摘要。

### 数据与任务

- 项目列表改为 `ProjectSummary`，项目详情和结果历史按需加载。
- 后台任务持久化到 SQLite；服务重启后未完成任务标记为 `interrupted`。
- 同一项目任务使用项目级锁串行执行。
- 导出制品记录文件大小和 SHA-256。
- 数值内核线程数集中配置，默认单线程，降低嵌套并发和延迟波动。

### 前端与交付

- 工作台、文档页和 Three.js 三维模块按需加载。
- 前端依赖锁定到明确版本，统一使用 `npm ci`。
- CAD 钢筋深化数据一次生成、多图共享，显著降低导出耗时。
- 软件、算法、规则集和导出模式版本集中维护。
- 根目录阶段性说明全部迁移到 `docs/archive/root-notes/`，现行说明按架构、工程、运维和发布主题集中维护。

## 工程边界

当前空间耦合仍属于杆系/弹簧代理模型，土体本构与地下水计算保留工程简化。钢筋输出用于设计辅助和施工表达，尚未覆盖全部节点级放样规则。规范审查实现为可追溯条文子集，正式项目仍需专业复核、专项计算和审签。

详细验证结果见 [V3.0.0 验证记录](V3_0_0_VERIFICATION.md)。
