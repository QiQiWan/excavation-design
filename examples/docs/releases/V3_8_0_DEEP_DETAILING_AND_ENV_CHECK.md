# V3.8.0 深化设计与运行环境诊断迭代

## 1. 本次迭代范围

V3.8.0 在V3.7.0专业施工图生产链上补齐以下能力：

- 节点承压板、加劲板、焊缝、锚筋和预埋件参数化设计；
- 钢筋混凝土支撑与钢支撑采用独立节点荷载路径；
- 钢筋笼水平分板、竖向分节、吊点、索力、吊筋和运输包络筛查；
- 机械连接套筒、丝头、接头错开和检验台账；
- 预埋件—钢筋三维包络碰撞和构造协调；
- 施工顺序与停检点；
- D-10、R-10、R-11和Q-04四类深化图；
- 深化设计施工版发行门禁；
- Windows/Linux当前Python环境依赖检查和退出安装命令提示。

## 2. 关键修复

### 2.1 钢筋笼重量计算

旧逻辑可能按整面长墙估算一个钢筋笼，导致重量和吊装需求失真。现按最大6m水平墙幅和最大12m竖向长度分节，重量、吊点和运输检查均基于真实笼段范围。

### 2.2 节点焊缝荷载路径

旧筛查对钢筋混凝土支撑节点也使用完整轴力校核围焊，造成不符合实际传力机制的焊缝失败。现区分：

- RC支撑：混凝土承压与纵筋锚固承担主轴力，预埋板围焊按次生作用筛查；
- 钢支撑：完整轴力进入端板焊缝，角焊缝不足时升级为全熔透坡口焊。

基准算例深化设计硬失败由12项降为0项，保留1054项预埋件—常规钢筋构造协调警告。

### 2.3 大规模碰撞性能

预埋件与钢筋碰撞由全量两两循环改为Shapely空间索引，并按预埋件、宿主、钢筋组和钢筋类型归并。基准算例深化模型可在约2秒内完成。

### 2.4 启动依赖诊断

启动脚本不再等到Uvicorn导入失败后才提示。启动前读取锁定依赖并验证当前环境；缺包退出时显示精确命令，例如：

```bash
/path/to/python -m pip install fastapi>=0.115,<1 uvicorn[standard]>=0.34,<1
```

实际命令只列出当前环境缺少或导入损坏的依赖。

`URBAN-TOPDOWN-32M-WALL-5SUPPORT`结果：

- 节点硬件：230组；
- 钢筋笼吊装工况：138个，全部通过筛查；
- 机械连接套筒：21,886个；
- 预埋件构造协调检查：1,054项；
- 深化设计硬失败：0；
- 深化设计状态：warning；
- CAD图纸：41张；
- DXF独立校验：41/41通过。

协调警告表示钢筋进入节点预埋件净空，需要在D-10中完成绕筋、截断锚固、预埋件开孔或局部配筋调整。它们不会被自动降级为通过。

## 5. 工程边界

当前节点硬件、吊装和碰撞属于工程深化筛查。正式施工前仍需：

- 高利用率/复杂节点专项实体或壳单元有限元；
- 焊接工艺评定和项目焊缝无损检测方案；
- 结合实际吊机、场地、吊具和天气的吊装专项方案；
- 企业DWG/CTB出版环境；
- 设计、校核、审核、批准和注册工程师签审。
