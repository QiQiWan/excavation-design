# V2.0.10 功能性与人机交互优化说明

## 1. 当前仍有优化空间

当前系统已经打通地勘导入、地质建模、基坑轮廓、围护体系、支撑候选优化、完整计算、IFC 和 DOCX 输出，但人机交互还存在三类问题：

1. 流程状态分散。用户需要在流程栏、结果页、候选方案页和导出页之间切换，才能判断当前是否可以正式出图。
2. 人工决策沉淀不足。局部锁定、候选方案选择、A/B/C 完整计算结果虽然已经存在，但缺少统一的操作员摘要。
3. 初次部署门槛偏高。后端虚拟环境、前端依赖、API 地址和数据库路径需要用户分别配置，Windows 用户尤其容易在命令行路径和执行策略上出错。

## 2. 本轮优化内容

### 2.1 设计决策驾驶舱

新增 `EngineeringDecisionBoard`，在工作台主区顶部集中展示：

- 地勘资料完成状态；
- 地质模型完成状态；
- 基坑轮廓完成状态；
- 围护体系完成状态；
- 最新计算结果状态；
- 成果闸门状态；
- 局部锁定数量；
- 候选方案数量；
- A/B/C 完整计算比选数量；
- 出图阻断项和警告项。

每个状态卡片都可以跳转到对应步骤。

### 2.2 启动脚本

根目录新增：

- `start-linux.sh`：适用于 Linux/macOS Bash 环境；
- `start-windows.ps1`：适用于 Windows PowerShell；
- `start-windows.bat`：双击入口，内部调用 PowerShell 脚本。

脚本负责：

- 创建 `services/api/.venv`；
- 安装/更新后端依赖；
- 检查并安装前端依赖；
- 设置默认 SQLite 数据库到 `runtime/pitguard.sqlite3`；
- 启动 FastAPI 服务；
- 启动 Vite 前端；
- 输出访问地址。

## 3. 后续建议

下一阶段可继续推进：

- 项目设置表单：地下水位、超载、安全等级、规范版本可视化编辑；
- 长任务进度 API：对候选完整计算、IFC 导出、计算书生成提供后端实时进度；
- 决策历史时间线：记录每次锁定、优化、采纳候选、复算和出图操作；
- 计算失败可解释化：将 fail/warning 与平面图、构件表和计算书条目联动高亮。
