# V3.0.0 验证记录

## 已完成验证

- Python 应用与测试文件编译检查通过。
- 后端快速发布门禁 13 项通过，耗时约 22.1 秒；其中 V3.0.0 新增集成测试 5 项覆盖轻量项目列表、结果历史限流、闭合几何对账、局部支撑配置、构件刚度和任务持久化。
- 核心 MVP 路径通过：健康检查、项目创建、轮廓更新和计算流程。
- 既有任务、问题中心、追溯链、CAD、设计长度冗余和方案看板测试通过。
- 公开基准与钢筋深化相关测试支持隔离执行；V2.5 完成度闭环组 4 项通过，耗时约 16.2 秒；V2.4 CAD 逐根钢筋几何导出单项通过，耗时约 19.3 秒。
- CAD 导出将钢筋深化数据由多次重复生成改为一次生成并复用，相关导出测试由超过 15 分钟降至约 17—20 秒。
- 前端 3 项测试通过，生产构建通过；入口包约 11.21 KB，工作台约 190.32 KB，Three.js 独立包约 523.86 KB。
- 项目列表摘要化验证通过：82 个项目从约 136.02 MB 降至约 30.76 KB。
- 单项目连续 6 次完整计算验证通过，总耗时约 24.3 秒，进程常驻内存约 747 MB。

## 仍需持续验证

- 全部历史重型测试在单一 pytest 进程中连续运行时，原生数值/绘图库存在内存驻留与耗时累积。发布脚本使用 `full-isolated` 模式逐节点执行，规避进程级状态污染；后续应继续拆分公共大样例并降低测试夹具体量。
- Three.js 独立资源包约 524 KB，仍高于 Vite 默认 500 KB 提示线。
- 计算精度仍需与 PLAXIS、MIDAS GTS NX、ABAQUS 或经审定的内部程序进行多工况定量对标。
- 复杂异形基坑、局部支撑缺失、分区开挖、换撑、承压水和空间效应需要增加专项基准算例。
- 钢筋模型仍以规则化施工表达为主，节点碰撞、锚固、搭接错开与施工放样需要深化。

## 发布判定

V3.0.0 满足工程设计辅助原型的软件交付条件。正式项目出图和盖章条件仍受岩土参数、施工方案、规范适用性、专项分析和专业审签约束。
