# PitGuard V3.87.10 围护墙钢筋笼平面路径一致性修复

## 问题现象

配筋深化页面能够显示全部围护墙钢筋笼，但钢筋笼的平面轮廓与工程输入阶段绘制的围护墙轮廓不一致，局部转折被拉成直线，历史槽段端点还可能继续沿用旧轮廓坐标。

## 根因

1. 钢筋笼可视化只读取墙轴线首尾点，将多段折线压缩为一条弦线。
2. 槽段 `construction_panels.start/end` 在基坑轮廓或围护墙重新生成后未按当前轴线重投影，历史坐标被误当作权威几何。
3. 前端每个钢筋笼只按 `start -> end` 生成直线网格，忽略墙面转折。
4. 旧完整性检查只核对墙体宿主数量，无法识别“20 面墙都存在但平面路径错误”。
5. IFC、CAD 与配筋深化模块对槽段长度和端点使用的几何来源不完全一致。

## V3.87.10 修复

### 单一权威平面路径

新增 `app/geometry/wall_path.py`。围护墙与钢筋笼统一按以下优先级解析平面路径：

1. 当前基坑分段；
2. 当前设计面包含的连续分段；
3. 当前围护墙轴线；
4. 基坑闭合轮廓对应边。

历史槽段端点仅用于偏差审计，不再覆盖当前工程几何。

### 按里程重建槽段

槽段按当前墙轴线里程重新定位：

- 保存槽段编号、接头类型和人工复核属性；
- 重算起点、终点、长度和完整 `planPath`；
- 发现间隙、重叠、长度异常或端点偏离时自动重建连续槽段计划；
- 将修复信息写入配筋方案审计字段。

### 折线路径钢筋笼

围护墙钢筋笼现在包含完整 `planPath`。前端三维视图沿折线逐段生成：

- 双面纵筋；
- 水平分布筋；
- 拉结筋；
- 接头、吊点和搭接区；
- 围护墙宿主轮廓。

局部钢筋方向根据当前路径切向和法向计算，不再使用全墙首尾点方向。

### 全链路一致性

以下模块共享同一条规范化墙路径与槽段计划：

- 配筋三维预览；
- 配筋深化与钢筋工程量；
- 配筋方案应用；
- IFC 导出；
- CAD 图纸与槽段追溯表。

### 新增几何审计

配筋可视化摘要新增：

- `repairedPanelGeometryCount`；
- `wallPanelGeometryMismatchCount`；
- `maximumPanelGeometryDeviationM`；
- `wallPlanGeometryStatus`；
- 单槽段 `geometrySource`、`geometryStatus` 和 `storedGeometryDeviationM`。

界面明确显示“匹配”“已自动修复”或“仍有未解析墙面”。

## 兼容性

旧项目不需要删除数据库。重新应用当前配筋方案时，系统会把历史槽段坐标重新绑定到当前围护墙路径，并保存修复后的槽段几何。旧槽段编号和接头属性尽量保留。
