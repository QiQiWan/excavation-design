# PitGuard V3.87.8 配筋 Worker、完整宿主与渐进列表修复

## 1. 修复范围

本版本处理以下故障：

1. 独立配筋任务正常完成后，下一轮 worker 启动会把运行中任务误标记为中断。
2. 配筋方案在 API 进程同步构建，叠加完整计算结果与截面反馈复算时可能造成内存峰值过高。
3. 支撑拓扑更新后，旧配筋方案仍可能被界面复用，导致箍筋、侧面构造筋、拉结筋和附加筋未显示。
4. 个别地下连续墙轴线退化或槽段映射缺失，三维钢筋模型少显示一面墙。
5. 配筋复核、墙体分区、支撑配筋、围檩节点和图纸目录一次性展开，页面过长。

## 2. Worker 修复

- 新 worker 只恢复其心跳文件中明确归属且进程已死亡的任务。
- 正常的一任务一进程退出状态 `completed` 不再触发全表中断。
- 仅对心跳超过阈值的孤立 running 任务执行保守清理。
- 配筋任务设置独立硬超时，默认 3600 秒。
- 前端遇到明确的 worker 中断时，自动使用新进程重试一次。

## 3. 配筋内存修复

- 外部 worker 模式下，API 只排队，不在 API 进程构建数千条配筋检查记录。
- 截面反馈复算前，将完整配筋方案替换为紧凑审计记录。
- 复算前释放旧工程对象和旧计算包络，避免同一进程同时保留两份完整项目。
- 复算完成后重新读取权威计算结果，再生成并应用最终配筋方案。

## 4. 配筋方案时效性

每个方案新增：

- `hostSignature`
- `sourceCalculation.resultId`
- `sourceCalculation.inputSnapshotHash`
- `sourceCalculation.supportTopologyHash`

若墙体或支撑的 ID、轴线、标高、截面或拓扑发生变化，旧方案自动失效并重新生成。

## 5. 水平支撑钢筋完整性

当前支撑配筋合同包含五类钢筋：

- 纵向主筋
- 端部加密区与跨中普通区箍筋
- 侧面构造分布筋
- 拉结与架立筋
- 搭接与锚固区附加筋

三维可视化发现旧方案没有覆盖当前支撑时，会按当前拓扑轻量重建支撑配筋行，并输出重建数量和缺失钢筋族。

## 6. 地下连续墙完整显示

墙轴线恢复顺序：

1. 当前墙轴线；
2. 对应基坑分段轴线；
3. 对应设计面或槽段；
4. 基坑闭合轮廓边；
5. 前端从已生成钢筋笼几何恢复宿主显示。

可视化摘要新增应显示墙数、实际显示墙数、修复轴线墙数和仍缺失墙编号。

## 7. 渐进列表

以下列表默认显示前 12 项并支持展开/收起：

- 配筋深化入口阻断；
- 配筋深化复核；
- 失败原因；
- 配筋对象复核；
- 墙体分区；
- 水平支撑；
- 围檩与节点；
- 图纸目录。
