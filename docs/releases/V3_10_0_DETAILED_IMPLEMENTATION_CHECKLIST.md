# PitGuard V3.10.0 P0–P2 详细迭代实施清单

## 1. 范围与边界

本轮在 V3.9.0 基础上推进 P0–P2，但按项目决策暂不接入 CalculiX、Abaqus 等外部有限元求解器。既有节点子模型输入文件能力继续保留，不在本轮启动外部求解任务。

本轮目标包括：

1. 将构造协调候选真实写回钢筋和预埋件几何；
2. 写回后重新执行加工、净距、碰撞和深化检查；
3. 将吊装路径由直线筛选升级为场地网格 A* 路径规划；
4. 将 A/B/C 完整计算拆分为可并行、可缓存、可独立重试的任务；
5. 将单位注册表改为后端和前端共用的单一数据源；
6. 按角色收束工作台，但保留全部模块和闭环审查入口；
7. 增加十二模块完成度审查，保留原完整验收矩阵；
8. 重构候选方案预览，使模型自动居中、放大，并支持缩放和平移；
9. 通过懒加载、延迟挂载和运行时 LOD 降低三维模块首屏成本。

## 2. P0：构造协调真实写回

| 编号 | 实施内容 | 验收依据 |
|---|---|---|
| P0-01 | 建立 `detailGeometryPatches` 参数化补丁模型 | 补丁包含目标对象、几何增量、来源候选和版本 |
| P0-02 | 钢筋绕行写回逐根钢筋中心线 | 原始中心线被替换，长度和重量同步更新 |
| P0-03 | 局部附加筋生成真实 U 形钢筋 | 新增钢筋具有唯一编号、宿主、直径和中心线 |
| P0-04 | 预埋件移位写回中心坐标和包络 | 后续碰撞检查使用新坐标 |
| P0-05 | 预埋件开孔写回孔洞参数和影响半径 | 穿越设计孔洞的关联钢筋可按设计连接识别 |
| P0-06 | 几何写回后重新执行 BBS、分段、净距和碰撞 | 不再只依据预测净距将问题改为通过 |
| P0-07 | 稳定问题组 ID | 同一问题复算后仍可追踪，不因列表顺序变化而漂移 |
| P0-08 | 图纸派生几何失效标识 | 几何改变后要求重新生成图纸和修订差异 |

### 2.1 写回链路

```text
碰撞问题组
→ 生成绕筋/移位/开孔/局部加筋候选
→ 选择候选
→ 形成 detailGeometryPatches
→ 写回逐根钢筋或预埋件几何
→ 重算长度、重量和加工分段
→ 重跑净距与碰撞
→ 更新深化状态
→ 标记图纸派生几何过期
```

### 2.2 当前工程边界

- 直线钢筋的局部绕行和 U 形附加筋已形成真实中心线；
- 复杂多段弯折、空间套筒避让和孔边板壳应力仍需专项构造复核；
- 候选写回不能绕过结构承载、锚固、裂缝和施工净距门禁。

## 3. P0：吊装场地数字化与路径规划

| 编号 | 实施内容 | 验收依据 |
|---|---|---|
| P0-09 | 增加场地边界 | 路径节点不能越出场地 |
| P0-10 | 增加道路走廊 | 输出道路覆盖率和偏离距离 |
| P0-11 | 增加建筑、高压线和地下设施禁入区 | A* 网格阻止路径穿越 |
| P0-12 | 考虑吊机履带/支腿包络 | 禁入区按吊机占地半径膨胀 |
| P0-13 | A* 路径规划 | 输出路径点、长度、转弯次数和搜索节点数 |
| P0-14 | 场地路线进入吊装可行性评分 | 路线冲突、转弯过多和道路覆盖不足形成惩罚 |
| P0-15 | 保留吊重、臂长、地基、风载校核 | 路径可行不代表吊装工况整体通过 |

当前路径模型用于方案阶段和施工策划前期筛选。正式专项方案仍需结合实测道路宽度、坡度、地基处理、吊机说明书和现场交通组织复核。

## 4. P1：A/B/C 并行计算与缓存

| 编号 | 实施内容 | 验收依据 |
|---|---|---|
| P1-01 | 新增单候选完整计算公开入口 | 每个候选可独立调用完整计算链路 |
| P1-02 | 新增候选输入哈希 | 哈希覆盖基坑、地层、设计设置、校准参数、候选几何和算法版本 |
| P1-03 | 建立文件缓存 | 相同输入直接读取候选计算结果 |
| P1-04 | 增加候选批处理任务 API | 一次提交最多三个候选，返回独立任务 ID |
| P1-05 | 候选任务并行执行 | 候选计算不再被同一项目全程串行锁住 |
| P1-06 | 写回阶段使用短项目锁 | 防止多个任务同时覆盖项目结果 |
| P1-07 | 前端显示逐候选进度 | 已完成方案可以先展示 |
| P1-08 | 支持单候选失败和重试 | 其他候选结果不受影响 |
| P1-09 | 缓存命中可视化 | 前端显示命中状态和输入哈希摘要 |

样例首轮三个候选完整计算用时约 17.55 s；相同输入第二轮全部命中缓存，用时约 2.24 s。

## 6. P1：闭环审查与模块完成度

保留既有 `acceptanceMatrix`，并新增十二模块完成度审查：

1. 项目与地勘输入；
2. 三维地质模型；
3. 基坑几何与一致性；
4. 围护与支撑设计；
5. A/B/C 方案优化；
6. 施工阶段与结构计算；
7. 配筋与加工深化；
8. 节点、碰撞与吊装深化；
9. 出图规则与 CAD/PDF；
10. BIM 与模型交付；
11. 监测、校准与审签；
12. 正式发行与归档。

每个模块输出：

- 完成度；
- 完成项/总项；
- 状态；
- 是否阻断；
- 缺口；
- 已形成证据；
- 下一步动作；
- 负责角色。

界面中该审查始终可见。按角色聚焦只调整优先级和默认展开状态，不删除任何模块。

| 编号 | 实施内容 | 验收依据 |
|---|---|---|

| P1-18 | 滚轮缩放 | 以鼠标位置为中心缩放 |
| P1-19 | 指针拖动平移 | 支持检查局部转角和节点 |
| P1-20 | 放大、缩小、适配按钮 | 可随时恢复自动适配视图 |

## 8. P1：按角色聚焦的人机交互

新增六种角色视图：

- 方案设计；
- 结构计算；
- 配筋深化；
- 施工策划；
- 校核审核；
- 图纸发行。

每种视图只改变：

- 主流程步骤优先级；
- 下一步动作；
- 默认展开区域；
- 次要区域的视觉弱化。

所有工程数据、模块完成度、闭环审查和专业详情仍可访问。

## 9. P2：前端三维性能

| 编号 | 实施内容 | 验收依据 |
|---|---|---|
| P2-01 | 五个三维查看器懒加载 | 未进入对应页面时不下载和执行 |
| P2-02 | 折叠详情延迟挂载 | 关闭的高级区域不创建三维场景 |
| P2-03 | 三维渲染质量档位 | 自动、性能、均衡、高质量 |
| P2-04 | 根据场景复杂度选择 LOD | 大模型降低圆柱分段和 VTU 采样密度 |
| P2-05 | 像素比按质量档位限制 | 避免高 DPI 屏幕无约束增加 GPU 负荷 |
| P2-06 | 前端主工作台分包 | `ProjectWorkspace` 由约 244.84 kB 降至约 161.02 kB |

Three.js 公共包仍约 523.86 kB，但已从首屏移出并仅在需要三维功能时加载。后续可进一步拆分具体 Three.js addon 和查看器运行时代码。

## 10. 本轮未实施项目

- 未调用外部有限元求解器；
- 未自动批准复杂钢筋空间避让；
- 未将 A* 路径替代正式吊装专项方案；
- 未隐藏闭环审查和专业模块；
- 未通过单位或显示格式改变底层工程量纲。
