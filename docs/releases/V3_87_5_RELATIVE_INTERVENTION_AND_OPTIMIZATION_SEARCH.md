# PitGuard V3.87.5 相对加固、可审计复算与一键多目标优化

## 1. 问题定位

V3.87.4 的“增厚并复算”按钮携带了计算结果生成时的绝对建议值。例如建议墙厚为 0.90 m，首次点击后墙厚已经达到 0.90 m，第二次点击仍把 0.90 m 写回当前墙段。后端按绝对值赋值，设计快照没有继续变化，因此连续复算可能得到相同结果。

同时，原任务只生成新的计算结果编号，没有完整记录构件修改前后值、设计输入哈希、结果哈希和配筋刷新状态。界面难以区分“真实修改后复算”和“重复提交同一设计”。

## 2. 相对加固语义

V3.87.5 将可自动执行的结构措施统一为基于当前持久化设计值的单调增量：

- 墙厚：每次至少增加 0.10 m，并按 0.05 m 模数上取整，自动上限 2.40 m；
- 墙趾：每次至少加深 0.50 m；
- 冠梁、围檩和环梁：宽度、高度每次至少增加 0.10 m，并受工程上限约束；
- 支撑：调用支撑深化优化器，仅接受产生真实构件变化的结果；
- 锁定墙趾、荷载、土层、水位和锁定施工阶段不会被自动修改。

每次动作记录 `before`、`after`、`delta`、`objectId`、`objectCode` 和 `valuePolicy=relative_from_current`。达到自动上限后停止连续放大，并提示调整体系、材料或进行专项设计。

## 3. 可审计复算合同

单项措施执行后，任务按以下顺序运行：

1. 从数据库读取当前最新工程；
2. 按当前值应用相对加固；
3. 重新运行验算—优化—再验算闭环；
4. 按更新后的内力重新生成配筋方案；
5. 配筋引起截面变化时再复算，最多两轮；
6. 保存新的工程版本、计算合同和配筋合同。

计算结果增加以下证据：

- 前后输入快照哈希；
- 前后采用方案快照哈希；
- 前后结果哈希；
- 前后控制结果值；
- `inputChanged`、`numericalResultChanged` 和 `resultChanged`；
- `lastAppliedAction`、`appliedInterventionCount` 和 `rebarRegenerated`。

前端显示“最近一次有效修改”，明确构件由哪个值调整到哪个值。

## 4. 一键多目标优化并复算

新增任务 `calculation_optimize_search`。系统在隔离工程副本中评估最多四组有界结构优化组合：

1. 当前方案闭环；
2. 经济分区；
3. 刚度优先；
4. 截面优先。

每个候选分别运行完整计算闭环，按以下工程优先级排序：

1. 计算闭合；
2. 结构闭合；
3. 硬失败数量；
4. 结构失败数量；
5. 定量安全储备缺口；
6. 安全系数总缺口；
7. 最大位移；
8. 墙、梁和支撑材料代理增量。

系统只把排名第一的候选围护体系写回正式工程，随后在正式工程上重新计算并更新配筋。该结果是当前参数边界、候选族和迭代次数内的最优可行解，不代表无限设计空间中的数学全局最优解。

## 5. 前端变化

计算阻断解决中心新增“一键优化并复算”。完成后显示：

- 已评估候选数；
- 完全闭合候选数；
- 采用策略；
- 材料代理增量；
- 各候选排名、硬失败、定量缺口和最大位移；
- 单项措施按钮改为“按当前值递增并复算”。

## 6. 验收重点

1. 对同一墙段连续点击两次，墙厚应连续增加，界面显示两次不同的前后值；
2. 新结果的输入快照哈希或设计快照哈希应变化；
3. 一键优化任务应生成至少两个独立候选，并保存候选排名；
4. 采用候选后应生成新的计算结果和配筋方案；
5. 达到构件自动上限时应停止放大并给出专业处理提示。
