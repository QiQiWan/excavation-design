# PitGuard V3.87.3 单一设计主流程与质量追溯中心

## 1. 问题背景

V3.87.2 项目工作区同时展示两套具有流程语义的界面：

1. 顶部六阶段主导航：设计基准、工程输入、围护方案、计算验算、配筋深化、成果交付；
2. 常驻的“V3.87 设计主流程”九阶段面板及七个页签。

两套界面均显示进度、阻断、下一步和阶段编号，导致以下问题：

- 用户无法判断哪套流程是权威入口；
- 六阶段与九阶段的完成率和阻断语义容易被理解为互相冲突；
- 质量证据面板占据首屏较大区域，真正的当前设计操作被推到页面下方；
- 项目打开时额外请求设计核心聚合数据，增加首屏负载；
- 九个内部质量域被误用为第二套导航，而其本质是参数、规则、方案、计算、配筋和成果证据。

## 2. 设计原则

V3.87.3 采用单一权威流程：

1. 设计基准；
2. 工程输入；
3. 围护方案；
4. 计算验算；
5. 配筋深化；
6. 成果交付。

九个原设计核心阶段保留为内部证据域，通过“质量与追溯”按需打开，不承担流程导航。

## 3. 证据域映射

| 六阶段主流程 | 内部证据域 |
|---|---|
| 设计基准 | D1 规范与参数确认 |
| 工程输入 | D2 工程输入与设计域 |
| 围护方案 | D3 方案搜索与比选、D4 围护结构联合设计 |
| 计算验算 | D5 计算核验与结果包络 |
| 配筋深化 | D6 配筋与节点深化 |
| 成果交付 | D7 施工图生成、D8 计算书一致性、D9 校审快照与发行 |

## 4. 前端调整

- 删除主页面常驻的第二套流程面板；
- 在项目标题区新增“质量与追溯”按钮；
- 质量中心采用右侧抽屉，默认关闭；
- 抽屉打开时才调用 `/design-core/bundle`；
- 抽屉内六组质量卡与主流程一一对应；
- 当前步骤高亮；
- 点击“进入该步骤”后关闭抽屉并返回主流程；
- 支持遮罩关闭、关闭按钮、Escape 键和背景滚动锁定；
- 页签名称由“设计主流程”改为“质量总览”，其他页签统一使用“证据”语义。

## 5. 后端与文档调整

设计核心聚合结果新增：

- `presentationRole = quality_assurance`；
- `primaryWorkflowStageCount = 6`；
- `primaryWorkflow`；
- `evidenceDomainCount = 9`；
- `evidenceGrouping`；
- `evidenceDomains` 兼容字段。

在线文档统一使用“六阶段单一设计主流程”。九个细分阶段仅作为内部质量证据域说明。

## 6. 不变边界

本补丁不修改：

- 围护结构生成算法；
- 水平支撑拓扑算法；
- 结构计算内核；
- 配筋公式；
- 施工图和计算书生成算法；
- 参数来源和正式发行门禁。

## 7. 验收标准

- 项目首屏只出现一套六阶段流程导航；
- “设计质量与追溯中心”默认不可见；
- 打开质量中心后只显示六组质量卡，不出现第二套九阶段导航；
- 质量卡可返回对应主步骤；
- 关闭抽屉后当前步骤不改变；
- 项目首次打开时不请求设计核心 bundle；
- 在线文档不再将九证据域称为主流程；
- 前后端版本显示 3.87.3。
