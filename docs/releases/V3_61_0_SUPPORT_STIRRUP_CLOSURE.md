# PitGuard V3.61.0：水平支撑箍筋设计与三维显示闭环

## 问题结论

此前界面“显示了箍筋”并不等于“箍筋设计和显示正确”。旧链路存在四个断点：

1. 只要构件上存在一条 `stirrup` 记录，就可能被判为箍筋完整，无法区分端部加密区和跨中普通区。
2. 端部、跨中箍筋的采样范围都可能落在支撑全长，导致重叠或位置失真。
3. 全局三维抽样优先消耗纵筋额度，A 端箍筋也可能先耗尽箍筋额度，跨中和 B 端因此不显示。
4. “支撑构造近景”仍复用全基坑模型和边界，工程师无法真正定位到单根支撑核查。

V3.61.0 将箍筋设计记录、持久化、三维生成、接口抽样和人机核对改为同一套分区合同。

## 工程计算改进

- 为钢筋混凝土水平支撑增加可追溯斜截面受剪设计：支撑自重、施工临时线荷载和偏心弯矩等效剪力共同形成设计剪力。
- 同时控制斜截面受剪承载力、最小箍筋率、最大构造间距和端部轴压约束。
- 设计结果明确输出 A/B 两端加密区和跨中普通区的直径、间距、区段长度、有效箍肢和闭合几何箍肢。
- 没有正式施工阶段轴力时，方案可按几何、自重和构造下限先行生成，但状态为“待工程师复核”，计算完成后自动更新。
- 短支撑由两端加密区控制时，不再因小数舍入生成毫米级“伪跨中区”。

当前简化计算采用：

`Vd ≤ 0.7·ft·b·h0 + fyv·Asv·h0/s`，且 `Asv/(b·s) ≥ 0.24·ft/fyv`。

该计算用于方案设计与软件闭环，正式施工图仍需按项目采用规范、节点受力、抗震等级、锚固和施工条件进行专业复核。

## 数据合同改进

- `ReinforcementGroup` 增加区段类型、区段起止、区段长度、箍肢数和设计来源。
- 正式配筋方案对每根钢筋混凝土支撑持久化六组记录：纵筋、侧面构造分布筋、端部箍筋、跨中箍筋、拉结/架立筋和搭接附加筋。
- 旧版通长箍筋不会再冒充“端部+跨中”分区完整。
- 后端完整性结果同时校验钢筋族、设计分区和实际三维抽样分区。

## 三维核查改进

- 新增单根宿主聚焦接口参数 `focus_host_id`，可按支撑 ID 或编号只加载一根构件。
- 三维箍筋按 A 端、跨中、B 端的真实里程分别生成，不再全长重叠。
- 全局抽样先按钢筋族分层，再对箍筋按三个区段分层，防止跨中和 B 端被抽样额度吞掉。
- 新增“查看该支撑全部钢筋”和“仅看该支撑箍筋”，聚焦后自动调整相机、模型边界、宿主过滤和钢筋显示倍率。
- 工程师可直接查看端部/跨中配置、设计剪力、承载力、利用率、计算依据和三段显示状态；界面不再暴露内部变量名。
