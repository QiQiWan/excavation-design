# PitGuard V3.47.0 P3-1～P3-3 工程深化闭环

## 1. 版本目标

本版本在 V3.46 六阶段核心流程上推进三项能力：

- P3-1：将不利工况筛查转化为独立施工阶段正式复算；
- P3-2：建立版本化企业规范、节点、钢筋组合和出图资源库；
- P3-3：形成节点、预埋件、锚固、套筒、钢筋碰撞和局部子模型深化闭环。

同时完成两个交互修正：任务完成后停留在当前阶段；设计基准首页明确说明各选项对后续设计的影响。

## 2. P3-1 正式不利工况复算

支持降水失效、超挖、局部渗流、承压水头抬升、预加轴力/温度/安装偏差以及长期效应。每个场景采用独立工程种子，重新生成施工阶段并运行当前计算内核。历史候选、旧计算、报告图和逐根钢筋不会进入场景副本。

完整结果写入外部压缩成果，项目工作区仅保存场景摘要、控制安全系数、控制工况和成果引用。场景顺序执行，避免多套施工阶段模型同时驻留内存。

## 3. P3-2 企业工程资源库

新增 `packages/enterprise` 资源目录。当前资源类型包括：

- 国家/地方/企业标准模板；
- 项目安全目标和荷载组合策略；
- 钢筋混凝土支撑—围檩节点模板；
- 钢管支撑—围檩端板节点模板；
- 地连墙和混凝土支撑钢筋组合；
- 节点图、钢筋图和场景图编号规则。

资源库加载时检查缺失 ID、重复 ID、安全目标范围、节点轴力区间和标准图引用。设计基准页面可选择企业库和标准模板，并显示其对安全目标、节点选型、钢筋组合和图纸表达的影响。

内置库用于软件验证。正式项目应导入企业批准版本并记录版本号。

## 4. P3-3 节点与钢筋深化闭环

独立后台任务 `p3_detailing_closure` 执行：

1. 逐根钢筋、套筒、锚固、搭接和预埋件深化；
2. 拥挤度、净距、机械连接和可施工性校核；
3. 支撑—围檩节点局部受力分析；
4. 企业节点模板匹配；
5. 高风险节点局部子模型及 CalculiX 输入文件生成；
6. 构件、预埋件和钢筋空间碰撞；
7. 空间协调优化与施工图草案状态汇总。

完整逐根数据、局部模型和碰撞记录外部化。工作区只保留控制检查、数量统计、模板匹配和成果引用。

## 5. 交互修正

- 设计基准、地质模型、候选方案、计算和配筋任务完成后保持当前页面；
- 页面显示“已完成，请检查结果”，由用户决定是否进入下一阶段；
- 初次打开项目时仍可定位到第一个未完成步骤，后续刷新不自动切换；
- 设计基准增加六类影响说明：工程等级、场地环境、荷载组合、分析模型、材料与安全储备、企业资源库。

## 6. 计算书和审计

计算书增加：

- 企业工程资源库及标准模板；
- 正式不利工况复算表；
- 最大墙体位移、支撑轴力和最小安全系数；
- 企业节点、预埋件与钢筋空间深化摘要；
- 未匹配节点、碰撞、局部子模型和控制检查。

新增：

```bash
python scripts/audit-p3-engineering.py --project-id <PROJECT_ID>
python scripts/smoke-p3-engineering.py
```

## 8. 当前边界

- 正式不利工况复算沿用当前一维/空间耦合内核；复杂三维渗流和非线性土体仍需专业软件复核；
- CalculiX 输入已生成，求解器执行与结果回写仍属于后续增强；
- 企业节点模板覆盖范围有限，未匹配节点保持人工专项设计；
- 钢筋碰撞为几何和构造筛查，施工可穿入性仍需结合钢筋笼分段、吊装和现场工艺；
- 全量 DXF 压力测试依赖 `ezdxf` 和项目正式图框环境。
