# PitGuard V3.12.0 实际工程案例迭代报告

## 1. 案例数据识别结果

本轮从用户提供的 PLAXIS Python 工程脚本中提取并转换了以下数据：

- 24 个钻孔，平面范围 X=-260～260 m、Y=-30～30 m；
- 每孔 6 个主控层位，共 144 条分层记录，实际使用 12 个地层代码；
- 地面标高 0.0 m，模型底标高 -90.0 m；
- 地下水水头标高 -20.0 m，转换为钻孔口以下水位深度 20.0 m；
- 基坑坑底标高 -16.6 m，地连墙底标高 -32.8 m；
- 20 点异形闭合轮廓，面积约 6125 m²，周长约 540 m；
- 项目指定支撑深度 0.0、4.0、7.2、10.3、13.3 m。

导入 CSV 保留饱和重度、压缩模量以及 X/Y/Z 三向渗透系数。圆砾层 E=5625 MPa、Eoed=4500 MPa、kx=0.1 m/s、ky=kz=0.0008 m/s 按源脚本原值保存，同时合并为参数级风险提示，避免在 24 个钻孔中重复显示同一异常。

## 3. 预警闭环调整

### 3.1 转为算法证据的项目

以下信息不再占用 Warning 数量：

- 支撑端点吸附到围檩或墙面节点；
- 候选支撑因无节点交叉而被删除；
- 主支撑按短跨方向布置；
- 支撑中心线向坑内偏移以避免与围护墙重合；
- 凹角短回墙通过相邻支撑节点形成传力区；
- 自动增加临时立柱并缩短有效无侧向支承长度。

这些内容进入 `retainingSystem.layoutSummary.designNotes`，前端以“算法已执行动作与设计证据”单独展示。

### 3.2 继续保留的风险

- 有效无侧向支承长度仍超过限值；
- 最下一道支撑距坑底小于硬下限；
- 坡道、出土口、保护区、管线和邻近建构筑物尚未导入；
- 构件承载力、变形、稳定或地下水验算未通过；
- 墙厚、墙深、节点与正式施工图仍需要专业复核；
- 实际支撑平面点位尚未按源 PLAXIS 支撑端点逐根导入。

### 3.3 已修复的算法问题

- 坑底净空：目标值改为 2.5 m，硬下限为 2.0 m，消除恰好等于阈值时的伪预警；
- 长支撑：由构件总长度改为有效无侧向支承长度；
- 凹角短回墙：存在相邻有效支撑节点时识别为角部传力区；
- 项目指定支撑深度：新增 `supportLevelDepthsM`，实际工程输入不会再被企业默认分层覆盖；
- 地连墙初选边界：进入 `manual_review`，不再与算法失效混合显示。

## 5. 规范矩阵重构

界面由“左侧流程说明 + 右侧独立规范长列表”改为按具体计算组织：

`流程步骤 → 具体计算 → 方法 → 直接适用规范 → 条文关注点 → 已实现规则 → 当前状态 → 输出成果`

例如：

- 土水压力卡片只显示 JGJ 120、GB 55003、GB 50009 及对应规则；
- 混凝土承载力和配筋卡片只显示 GB 55008、GB 50010 及对应规则；
- 墙幅接头和止水卡片显示 GB 55030、GB 55032、GB 50202 及施工质量证据；
- 数值求解残差和条件数单列为软件数值质量证据，不伪装成规范条文。

工作台和在线文档共用同一后端矩阵。条文号只从规则注册表的 `clauseReference` 输出；未建立精确条文映射时明确显示“适用条件需项目复核”。

## 7. 重要边界

用户提供的源脚本内包含硬编码的外部软件连接口令。该口令未写入 PitGuard 导入成果；若仍为有效凭据，应立即轮换，并改为环境变量或密钥管理服务。

当前导入包恢复了地质、基坑轮廓、墙底标高和支撑层标高。源脚本中的逐根支撑端点、不同层材料、施工阶段挖土块选择和 PLAXIS 结果尚未全部转换为 PitGuard 原生对象。正式计算前应完成支撑平面复核、周边环境导入、荷载确认和施工阶段对照。
