# PitGuard V3.72.0 事务化流程、稳定性治理与丰富结果

## 1. 本轮结论

V3.71.0 已具备异形支撑二维框架、墙—围檩—转接框架反力迭代、节点空间效应、数值病态门禁和专业交付证据。进一步审计后，系统仍存在四类全局问题：

1. **整体流程缺少事务边界。** 计算函数会在项目对象上逐步写回施工阶段、构件设计和工程证据。中途异常可能留下部分新结果与部分旧结果混合的状态。
2. **计算过程可观测性不足。** 用户只能看到任务整体完成或失败，难以判断时间消耗在墙体求解、反力迭代、构件深化还是交付闸门。
3. **结果“有数据”与“可用于工程决策”没有充分区分。** 结果域平均完整度可能掩盖地质、强度、稳定或深化等关键域的失败。
4. **稳定性指标语义存在混用风险。** 安全系数、越小越优的风险比值和质量指数曾被放入同一个最小值集合；支撑构件稳定检查也可能被宽泛的 `STABILITY` 字符串误归入岩土稳定。

V3.72.0 针对上述问题完成流程、稳定性和结果表达三条链路的闭环。当前版本能够明确回答：计算执行到哪一步、哪一步最耗时、工程计算是否通过、交付证据是否通过、数值解是否健康、哪些构件和工况控制设计、哪些关键域仍在阻断。

## 2. 整体流程优化

### 2.1 选择性事务工作集

计算入口改为“隔离试算—成功提交—失败回滚”的事务流程。试算副本只深复制计算期间允许修改的字段：

- 地质模型；
- 基坑与围护体系；
- 施工工况；
- 高级工程证据；
- 项目消息和时间戳。

历史计算结果、导出记录和大体量只读对象不再整体深复制，从而避免事务保护造成额外内存峰值。计算成功后只提交允许修改的工作集字段；失败时记录输入哈希、异常类型、异常信息和 `projectMutationCommitted=false`，原项目设计状态保持不变。

事务状态写入：

- `lastCalculationTransaction`；
- `lastCalculationFailure`；
- `calculation-execution/transaction_started`；
- `calculation-execution/transaction_committed`；
- `calculation-execution/transaction_rolled_back`。

### 2.2 六阶段执行台账

每次完整计算记录六个阶段：

1. 输入合同与预检；
2. 施工阶段墙—围檩求解；
3. 反力迭代、转接框架与构件深化；
4. 岩土稳定与不利情景；
5. 质量和正式交付闸门；
6. 结果证据冻结。

每阶段保存：

- 状态；
- 耗时；
- 有界指标；
- 阻断说明；
- 运行时事件。

最终状态分开表达：

- `engineeringStatus`：工程计算状态；
- `deliveryStatus`：正式交付状态；
- `status`：综合状态；
- `bottleneckPhase`：最大耗时阶段及占比。

构件验算失败会直接使“反力迭代、转接框架与构件深化”阶段失败。数值框架成功求解不会覆盖支撑稳定、承载力或节点深化失败。

### 2.3 计算中断后的恢复边界

V3.72 已将各阶段结果写入 JSONL 运行时事件，可用于崩溃定位和进度恢复判断。当前尚未保存刚度矩阵、分解因子和阶段内部求解快照，因此进程崩溃后可以识别最后完成阶段，但仍需从完整计算入口重新执行。后续可在阶段边界增加不可变检查点和矩阵结构缓存。

## 3. 稳定性与数值可靠性优化

### 3.1 自适应反力迭代

墙—围檩—转接框架反力迭代在 V3.71 固定点算法基础上增加：

- 反力相对残差；
- 反力绝对残差；
- 位移相对残差；
- 位移绝对残差；
- 松弛系数历史；
- 两周期振荡检测；
- 停滞检测；
- 收敛质量等级；
- 提前失败。

迭代结果采用 `pitguard-wall-wale-transfer-iteration-v2`。若残差持续停滞或出现振荡，系统降低更新幅度；无法在限定轮数内收敛时阻断正式计算。

### 3.2 数值健康包

统一生成 `pitguard-numerical-health-v1`，汇总：

- 原始和尺度化条件数；
- 条件数等级；
- 最大平衡残差；
- 回退求解次数；
- 病态系统阻断数；
- 反力迭代状态；
- 转接框架节点刚度比；
- 节点位置与构件刚度敏感性；
- 空间节点最大转角、扭矩和条件数。

数值健康结果单独显示，避免“模型求解成功”被解释为“构件设计通过”。

### 3.3 外部与独立基准验证

本版本采用两层验证：

1. **外部成熟内核证书。** 保留 V3.71 实际生成的 OpenSees 线弹性框架、桁架和局部空间节点证书。证书与 `STRUCTURAL_KERNEL_VERSION` 绑定，工作流和结果管线升级不会无效化未改变的结构单元内核；单元刚度、自由度或坐标变换发生变化时必须重新验证。
2. **独立参考模型。** 新增三项 SciPy 稠密对称方程参考模型，单独实现框架、桁架和空间转动刚度，不调用生产单元组装代码。当前三项全部通过，最大相对位移误差约为 `3.90e-14`。

当前运行环境未安装 OpenSeesPy，因此实时外部基准状态明确返回 `unavailable`，不会使用生产求解器结果伪装外部验证。

## 4. 稳定指标语义修复

### 4.1 三类指标分离

稳定专项结果现分为：

| 指标类型 | 判定方向 | 典型指标 |
|---|---|---|
| 安全系数 | 越大越安全 | 嵌固、抗隆起、承压水突涌、渗流稳定、整体稳定 |
| 风险比值 | 越小越安全 | 分层渗透风险、降水阶段水位差控制 |
| 质量指数 | 越大越好 | 软弱下卧层筛查指数 |

系统分别记录 `value`、`limit`、`direction`、`utilization` 和 `reserveRatio`。`minSafetyFactor` 只从真实安全系数中选取；风险比值以最大利用率控制；零风险不再显示虚假的超大储备比。

### 4.2 构件稳定与岩土稳定分域

`PITGUARD-SUPPORT-DEEP-DESIGN-STABILITY` 属于支撑构件强度与稳定验算，现归入 `strength_verification`。岩土稳定域只统计有明确工程语义的嵌固、抗隆起、整体稳定、承压水和渗流规则。

这一修复消除了以下错误：

- 将分层渗透风险值 `0.0` 报告为最小安全系数；
- 将支撑组合稳定利用率大于 1.0 误报成岩土整体稳定失败；
- 稳定阶段显示通过，但结果完整度中的稳定域显示失败。

## 5. 计算结果丰富度

### 5.1 施工阶段矩阵与关键阶段

每个施工阶段输出：

- 开挖标高和阶段类型；
- 激活支撑数量；
- 墙体最大压力、弯矩、剪力和位移；
- 支撑最大轴力；
- 围檩最大弯矩、剪力和挠度；
- 最大尺度化条件数；
- 最大平衡残差；
- 最小岩土稳定安全系数；
- 最大稳定风险利用率；
- 检查状态统计。

系统按墙位移、墙弯矩、支撑轴力、围檩弯矩、数值条件和检查惩罚计算关键阶段排序。

### 5.2 构件包络与热点

结果目录包括：

- 逐墙内力与位移包络；
- 逐支撑轴力包络；
- 逐围檩弯矩、剪力和挠度包络；
- 逐转接梁轴力、剪力和弯矩包络；
- 立柱和基础包络；
- 支撑节点承压、转角、扭矩和偏心弯矩热点；
- 稳定专项指标；
- 钢筋组和缺失配筋统计。

### 5.3 阻断根因台账

新增：

- `blockingCheckLedger`；
- `warningCheckLedger`；
- `manualReviewLedger`；
- `ruleStatusCounts`。

重复发生于多个构件和阶段的检查同时保留实例统计和去重后的规则根因。界面优先显示根因数量，避免数百个重复 Warning 淹没真正控制问题。

### 5.4 完整度与就绪度分离

结果完整度描述“是否生成了该结果域”；工程就绪度描述“关键结果域是否允许继续设计”。工程就绪度采用加权结果域和关键域上限：

- 地质缺失时最高 65%；
- 强度失败时最高 55%；
- 岩土稳定失败时最高 60%；
- 数值健康失败时最高 50%；
- 正式发行闸门未通过时，正式交付就绪度最高 49%。

因此，丰富的计算输出不会冲抵地质资料、构件承载力、稳定性或专业审签的关键阻断。

## 6. 量化评估

评估模型为约 60 m × 45 m、开挖深度 16 m、三道支撑的合成 L 形基坑。未提供真实勘察资料和真实专业签署。

| 指标 | 闭合内环 | 中心汇交框架 | 环内弦杆框架 |
|---|---:|---:|---:|
| 单方案完整计算耗时 | 约 6.6 s | 约 7.1 s | 约 6.3 s |
| 阶段—墙段结果 | 42 | 42 | 42 |
| 支撑包络 | 72 | 72 | 72 |
| 围檩包络 | 18 | 18 | 18 |
| 转接梁包络 | 18 | 30 | 24 |
| 立柱/基础包络 | 19 | 22 | 22 |
| 节点热点 | 50 | 50 | 50 |
| 数值最大相对残差 | 约 `3.8e-14` | 约 `5.4e-14` | 约 `2.5e-14` |
| 反力迭代 | 3 轮，强收敛 | 3 轮，强收敛 | 3 轮，强收敛 |
| 计算结果完整度 | 约 86% | 约 79% | 约 78% |
| 工程就绪度 | 65% | 55% | 55% |
| 正式交付就绪度 | 49% | 49% | 49% |

三个方案的控制岩土安全系数均为嵌固稳定 `1.476`，限值 `1.20`。分层渗流风险和降水控制比均为 `0.0`，作为越小越优的风险指标单独显示，不参与最小安全系数。

本算例的正式发行全部阻断，原因包括：

- 无真实钻孔和水位资料；
- 部分方案存在支撑构件稳定或转接深化失败；
- 无真实注册结构工程师审签；
- 正式计算保证和交付证据未闭合。

## 8. 后续优先级

1. 在阶段边界保存可恢复检查点，支持进程崩溃后的阶段级续算；
2. 将二维墙—围檩—转接模型扩展为全局六自由度空间梁—杆—立柱模型；
3. 增加土体非线性、接触、开裂刚度退化、二阶效应和预应力损失；
4. 将结果目录接入计算书章节、CAD 图层和 IFC 属性集，形成跨成果一致性校验；
5. 建立真实工程基准库，覆盖矩形、长条形、L/U/T/Z/H 形和多凹角轮廓；
6. 接入企业 PKI 或可信电子签章服务，替换本地受信资格目录；
7. 在可用网络环境完成前端构建、交互回归和 Three.js 按需拆包。
