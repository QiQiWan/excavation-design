# V3.87.9 配筋计算合同自动恢复

## 现象

结构计算已经闭合，配筋方案也已写入，但配筋深化入口仍显示：

- 当前计算合同与当前设计快照不一致；
- 计算结果缺失或已过期；
- P3 尚未运行。

诊断日志显示同一工程随后能够完成配筋任务，说明该状态可能来自压缩工程快照、外置施工阶段结果和前后端保存时序之间的短暂不一致。

## 根因

1. P3 入口使用紧凑工程对象，未装载外置的最新施工阶段结果，导致有效计算被误判为缺少证据。
2. 配筋 Worker 返回的是保存前诊断，保存后计算合同已经有效，但页面继续显示旧阻断。
3. 部分配筋入口提交 `recalculate=false`，设计快照变化后不会主动恢复计算合同。
4. 前端任务完成后只读取一次配筋方案，可能命中外置结果与配筋合同分批保存的中间状态。

## 修复

- P3 入口统一使用 `require_with_latest_calculation` 装载权威计算证据。
- `CALCULATION_NOT_CURRENT` 标记为可自动恢复。
- P3 遇到仅含可恢复阻断时，自动执行“补算—保存—重新配筋—重建入口合同”。
- 配筋保存后重新读取权威工程和外置阶段结果，生成最终配筋合同。
- 计算合同返回具体差异字段，便于审计。
- 前端配筋动作默认允许补算，并在任务完成后有限轮询权威合同。
- 自动恢复期间显示黄色处理中间状态，不展示误导性的红色最终阻断。

## 状态语义

- `autoResolvable=true`：可由系统通过补算和重建合同恢复。
- `current=true`：计算合同与当前设计快照、支撑拓扑、工况和版本完全匹配。
- 真实硬失败、用户冻结工况、专项设计输入缺失仍需人工处理，不会被自动掩盖。
