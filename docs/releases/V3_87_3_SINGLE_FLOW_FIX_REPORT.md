# PitGuard V3.87.3 单一流程优化与修复报告

## 一、问题结论

截图中的混乱来自两套具有同等视觉权重的流程同时出现在项目首页：

- 上方六阶段核心流程；
- 下方常驻的九阶段设计核心面板。

二者都包含序号、完成率、阻断、下一步和页签，用户难以识别权威流程。九阶段面板实际承担质量证据汇总，却以“设计主流程”命名并长期占据首屏，导致当前设计步骤被下推。

## 二、潜在影响

1. 设计人员可能按九阶段面板操作，同时忽略六阶段主导航；
2. 六阶段与九阶段就绪度算法不同，容易被理解为状态冲突；
3. “交付质检”页签可能在设计基准阶段提前出现，打断顺序认知；
4. 项目打开时立即请求设计核心聚合数据，增加首屏加载；
5. 小屏幕下两套流程叠加，页面滚动距离显著增加；
6. 后续新增功能可能继续同时接入两套入口，形成维护分叉。

## 三、最终界面结构

### 主页面

只保留：

1. 设计基准；
2. 工程输入；
3. 围护方案；
4. 计算验算；
5. 配筋深化；
6. 成果交付。

主页面展示当前步骤的实际编辑、计算和导出功能。

### 质量与追溯中心

通过项目标题区右侧按钮按需打开，用于：

- 参数来源与正式设计资格；
- 条文执行证据；
- 方案完整计算证据；
- 配筋闭合证据；
- 施工图、计算书和发行证据；
- 外部资料及设计复核请求。

该中心默认关闭，不承担流程导航。

## 四、九证据域与六阶段映射

| 主阶段 | 证据域 |
|---|---|
| 设计基准 | 规范与参数确认 |
| 工程输入 | 工程输入与设计域 |
| 围护方案 | 方案搜索与比选、围护结构联合设计 |
| 计算验算 | 计算核验与结果包络 |
| 配筋深化 | 配筋与节点深化 |
| 成果交付 | 施工图生成、计算书一致性、校审快照与发行 |

## 六、性能与稳定性收益

- 项目首次打开时不再加载设计核心 bundle；
- 首屏减少一个聚合请求和一组大型卡片渲染；

- 抽屉内请求仍使用序号防止旧响应覆盖新项目；
- 工程计算、配筋和成果数据完全不变。
