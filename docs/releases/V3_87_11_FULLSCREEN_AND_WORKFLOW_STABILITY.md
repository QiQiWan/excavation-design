# PitGuard V3.87.11 全屏与流程稳定性

## 三维模型全屏

四类 WebGL 模型统一接入全屏容器：工程三维模型、项目综合模型、围护墙受力云图、钢筋 IFC 模型。支持浏览器原生全屏和页面内全屏降级，进入或退出全屏时自动触发尺寸重算。

## WebGL 稳定性

- 统一创建、上下文丢失处理和资源释放。
- 页面不可见或模型离开视口时暂停实际绘制。
- 根据设备内存限制像素比。
- 销毁面板时主动释放 GPU 上下文。
- WebGL 创建或恢复失败只影响当前模型面板，可单独重建。
- 开发环境默认不启用 React StrictMode 双挂载；可用 `VITE_REACT_STRICT_MODE=true` 单独开启。

## 流程稳定性

- 前端每 30 秒读取系统 readiness，异常时缩短到 12 秒，并显示数据库、worker、内存、磁盘和任务队列的降级/阻断原因。
- GET 请求可去重；写请求不再错误共享同一在途响应，文件上传和不同工程版本不会串包。
- 临时 HTTP 状态使用指数退避和 `Retry-After`。
- 任一写操作成功后自动清理前端读缓存。
- 项目刷新采用世代号，后返回的旧响应不能覆盖新工程快照。

## 边界

运行时监测用于提前暴露资源和 worker 问题，不会自行修改结构设计参数。系统处于 not_ready 时应先恢复数据库、worker 或资源条件。
