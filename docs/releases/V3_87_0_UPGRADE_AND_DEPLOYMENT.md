# PitGuard V3.87.0 升级与部署说明

## 1. 升级范围

V3.87.0 从 V3.81.0 升级时保留旧版施工计划、现场快照和偏差事件字段，旧数据可以继续读取。新版主流程使用：

- `parameter_provenance`；
- `external_collaboration_records`；
- `design_review_requests`；
- `design_snapshots`；
- `/api/projects/{project_id}/design-core/*` 接口。

旧版施工和现场对象不参与首次设计发行门禁，也不计入设计完成度。

## 2. 数据迁移

1. 升级前备份数据库、对象存储和运行时证据目录；
2. 使用现有项目仓储加载历史项目，Pydantic 默认值会补充新增字段；
3. 不自动删除旧版 `construction_plan_stages`、`field_execution_snapshots` 和 `deviation_events`；
4. 由项目负责人重新确认正式设计参数及其来源；
5. 重新生成候选方案、完整计算、配筋闭合、图纸和设计快照；
6. 旧成果在新快照生成前保持只读，不自动继承正式发行资格。

## 3. 后端启动

后端默认端口为 8002：

```bash
cd services/api
python -m pip install -e .
uvicorn app.main:app --host 0.0.0.0 --port 8002
```

生产环境应将矩阵求解、候选方案完整计算、图纸生成和计算书生成放入短生命周期 worker。API 进程只负责请求、状态和成果查询。

## 4. 前端启动

```bash
cd apps/web
npm ci
npm run test
npm run build
```

当前交付环境中的 `node_modules` 不完整，打包时已移除。部署环境必须重新执行 `npm ci`。只有 Vitest 和 Vite 正式构建通过后，才允许部署新版前端。

## 5. Nginx 建议

- `/api/` 反向代理至 `127.0.0.1:8002`；
- 前端静态文件由独立目录提供；
- 计算、导出和候选完整计算接口配置较长超时；
- 上传和导出目录禁止执行脚本；
- 保留请求 ID、项目 ID 和任务 ID 日志。

## 6. 运行目录

建议将以下目录映射为持久卷：

- 项目数据库；
- 上传资料；
- 计算结果对象；
- 图纸、计算书和 IFC；
- 设计快照和审计日志；
- worker 临时目录。

临时情景项目、渲染文件和矩阵缓存应设置生命周期并定期清理。

## 7. 发布门禁

生产部署前至少完成：

- 全部后端回归；
- 前端 Vitest；
- Vite 生产构建；
- 数据迁移和回滚测试；
- 100 次耐久计算；
- 多项目并发；
- 外部软件基准；
- 真实项目试点；
- 安全扫描和备份恢复演练。

未满足上述条件时，系统界面和导出文件应显示 `engineering_preview`。

## 8. 兼容性注意事项

- 正式方案采用前必须完成候选完整计算；
- 软件建议参数不能直接取得正式设计资格；
- 图纸、计算书、配筋和 IFC 必须绑定同一 DesignSnapshotId；
- 旧版施工计划、现场快照和偏差事件不得反向覆盖设计参数；
- 外部现场信息仅通过协同资料和设计复核请求进入设计流程。
