# V3.9.0 深化优化、单位体系、整案比选与聚焦交互

## 1. 迭代目标

V3.9.0 同时推进六项工作：

1. 将预埋件—钢筋构造冲突从“问题清单”升级为可比较、可应用、可复核的几何化候选方案；
2. 为高轴力节点生成快速筛查、构造变体和 CalculiX/Abaqus 非线性实体接触输入文件；
3. 将钢筋笼分节、吊机能力曲线、站位、作业半径、地基承载、风载、禁入区和运输路径纳入联合优化；
4. 建立统一 SI 工程单位注册表，消除刚度、标高、轴力、位移等数据的无单位显示；
5. 恢复并强化 A/B/C 整体方案比选入口；
6. 将工作台收束为“当前成果—关键问题—下一步动作”，把专业中间量放入按需展开区域。

## 2. 构造协调优化

### 2.1 问题归并

碰撞检查按以下键归并：

```text
预埋件 ID + 宿主构件 + 钢筋组
```

每个问题组保留：

- 当前实际净距（m）；
- 要求净距（m）；
- 估算侵入量（m）；
- 受影响钢筋组；
- 原始碰撞检查 ID；
- 关联 D-10、Q-04 图纸。

### 2.2 四类候选

每个问题组生成四类参数化候选：

| 候选 | 参数化几何增量 | 必须复核 |
|---|---|---|
| 钢筋局部绕行 | 偏移向量、过渡长度、最小弯曲半径 | 锚固、净距、弯曲半径 |
| 预埋件小范围移位 | 平移向量、偏心增量、最大允许移位 | 承压、焊缝、偏心 |
| 预埋件开孔穿筋 | 开孔直径、边距、圆角、孔边加劲 | 净截面、局部屈曲、孔边传力 |
| 截断重锚与局部加筋 | 截断数量、替代钢筋面积、U 形筋、套筒 | 等强替代、锚固、裂缝控制 |

候选评分综合净距增益、施工性和结构代价。应用候选后，系统根据“原净距 + 几何增量”重新计算预测净距，并检查候选自带的承压、锚固、净截面等验证条件。候选无法满足要求时保留 warning/fail，不会直接把状态改为 pass。

### 2.3 当前工程边界

当前候选已经具备可写回的参数化几何增量。正式施工图仍需把增量写入完整逐根钢筋和预埋件几何，重新运行承压、焊缝、净距、碰撞与节点计算，并形成修订云线和节点详图。

## 3. 高轴力节点子模型

### 3.1 两级分析

节点分析采用两级模式：

1. 六自由度凝聚刚度模型快速筛选接触压力、钢板等效应力、位移、转角和控制利用率；
2. 自动生成 CalculiX/Abaqus 兼容的非线性实体接触输入文件。

快速筛选用于排序和方案比较，输入文件用于后续正式求解器复核。

### 3.2 参数化构造变体

每个节点自动比较：

- 现状节点；
- 承压板平面尺寸增加 20%；
- 双侧增加两道加劲板；
- 扩大节点核心区并增加约束钢筋。

输出每个变体的预测利用率、预测位移、材料代价和综合评分。

### 3.3 求解输入文件

CAD/深化包新增：

```text
90_schedules/node_submodels/<节点编号>.inp
```

输入文件采用 N-mm-MPa 单位，包含：

- 混凝土实体；
- 承压板实体；
- 支撑端部实体；
- 钢材和混凝土弹性参数；
- 压缩接触与摩擦；
- 支撑端部荷载；
- 几何非线性静力步；
- 位移、反力、应力和接触输出。

该输入文件是可复核的起始模型，正式分析仍需细化网格、材料非线性、钢筋锚固、焊缝、初始间隙及施工偏差。

## 4. 钢筋笼—吊机—场地联合优化

### 4.1 输入能力

系统支持默认吊机库，也允许项目在 `advancedEngineering.craneLibrary` 中覆盖或增加实际设备能力曲线。

项目可通过 `advancedEngineering.craneSitePlan` 配置：

- 实际吊机站位；
- 站位地基容许承载力（kPa）；
- 进场通道宽度（m）；
- 工地大门；
- 吊装禁入区；
- 设计风速（m/s）。

### 4.2 联合约束

每个钢筋笼段对吊机—站位组合检查：

- 动载和安全系数后的需求吨位；
- 实际作业半径与能力曲线插值；
- 所需臂长；
- 接地压力/地基承载利用率；
- 建议路基箱或垫板面积；
- 钢筋笼迎风面积和风载利用率；
- 吊运路径穿越基坑长度；
- 吊装禁入区；
- 运输路径禁入区；
- 进场通道宽度。

钢筋笼抽样改为按墙幅轮询，避免工况上限只截取第一幅墙。

### 4.3 当前边界

默认能力曲线和自动站位只用于方案筛选。正式吊装必须导入实际设备工况表、吊索具、履带板/路基箱、地基检测结果、现场风速限值、回转禁入区和审批运输路线。

## 5. 工程单位体系

新增：

```text
GET /api/system/units
```

统一单位包括：

| 物理量 | 单位 |
|---|---|
| 长度、标高 | m |
| 位移 | mm |
| 力、轴力 | kN |
| 线荷载、剪力 | kN/m |
| 弯矩 | kN·m |
| 墙体单位宽度弯矩 | kN·m/m |
| 压力、地基接地压力 | kPa |
| 应力、强度 | MPa |
| 平动刚度 | kN/m |
| 转动刚度 | kN·m/rad |
| 转角 | mrad |
| 温度 | °C |
| 质量 | t |

显示规则：

- 表格优先在表头显示单位；
- 关键卡片在数字后显示单位；
- 未激活数据使用“—”，不使用 0 代替状态；
- 工程量禁止显示无单位裸数字。

CAD 新增 `engineering_units.json`，V3.9 新增 CSV 表头全部使用显式单位后缀。

## 6. A/B/C 整体方案比选恢复

### 6.1 V3.8 中为何难以找到

候选功能没有从后端删除。V3.8 的精简流程把“生成候选”放在围护结构高级操作，把“完整计算”放在计算高级操作，两个入口分散且默认折叠，因此产生功能消失的观感。

### 6.2 V3.9 新流程

“围护结构”和“计算”主步骤均直接显示 A/B/C 整案比选：

1. 点击“生成 A/B/C 候选”；
2. 查看斜撑混合、双向网格和传统直对撑等完整方案预览；
3. 点击“完整计算 A/B/C”；
4. 比较最大轴力、最大位移、围檩弯矩、支撑/立柱数量、Fail/Warning 和出图闸门；
5. 点击“采用整套方案”；
6. 系统整体替换支撑、立柱和节点，并要求重新计算。

逐墙微调保留在专业模式，不再作为默认决策流程。

## 7. 聚焦型工作台

默认精简模式固定为三层：

### 当前成果

显示当前方案、计算状态、配筋状态、图纸发行状态及关键控制值。

### 关键问题

只显示优先级最高的问题卡片，按构件/条文/阶段归并；完整问题表放在展开区。

### 下一步动作

根据项目状态给出一个主操作和少量次操作，如生成整案候选、完整比选、采用方案、复算、配筋或出图。

默认折叠：

- 原始计算追溯链；
- 全局矩阵；
- 完整验收矩阵；
- 逐墙微调；
- 算法中间量；
- 专业出图配置；
- 全量钢筋与碰撞台账。

## 9. 发布边界

V3.9 的三项优化均属于设计深化和施工策划辅助能力。构造协调候选、局部实体输入文件及吊机站位推荐不能替代注册工程师校审、正式有限元求解、实际设备工况表和经审批的专项施工方案。
