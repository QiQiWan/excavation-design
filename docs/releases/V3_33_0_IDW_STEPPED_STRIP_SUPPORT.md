# PitGuard V3.33.0：IDW 地质模型与长条形台阶基坑支撑闭环

## 1. 版本目标

V3.33.0 针对两个直接影响工程使用的问题进行闭环修复：

1. IDW 地质模型后台生成成功后，完整曲面被外部对象存储接管，网页工作区却没有保留可视化预览，导致三维查看器显示曲面数量为 0；
2. 长条形、局部变宽和台阶式基坑沿每个轮廓折点机械插入支撑站位，造成局部支撑过密，端墙仍缺少有效围檩支点，计算结果同时出现围檩支点间距、墙体弯矩、变形和配筋失败。

## 2. IDW 工作区预览

- `GeologicalModel` 新增 `surfacePreviews`；
- 完整 IDW 曲面继续写入 `runtime/artifacts`；
- 工作区保留默认不超过 36×36 的边界保持型预览网格；
- 旧版已经外部化的地质曲面在升级迁移时自动回填预览；
- 三个三维查看器统一通过 `effectiveGeologicalSurfaces()` 读取完整曲面或预览曲面；
- worker 重组完整项目时仍使用原始高分辨率曲面，预览不会进入正式计算。

## 3. 长条形台阶轮廓识别

新增 `elongated_stepped_strip` 原型。识别依据包括：

- 最小旋转外接矩形长宽比；
- 正交边比例和凹角数量；
- 沿局部长轴的截面是否长期保持单一连续区间；
- 截面中心线漂移率；
- 截面宽度的台阶变化。

局部向外扩宽不会再将连续长条基坑误判为一般凹形多分区基坑。

## 4. 自适应短跨支撑站位

设局部长轴坐标为 `x`，局部净宽为 `w(x)`，参考净宽为 `w_ref`，目标站距为 `s0`。支撑站距按下式生成：

```text
s(x) = clip[s0·sqrt(w_ref / w(x)), s_min, s_max]
```

该模型近似保持单个支撑分担平面面积 `w(x)·s(x)` 稳定。宽区适当加密，窄区适当放宽，同时受 3–6 m 工程分仓和项目最小净距约束。

轮廓宽度突变处只移动最近的一条站位，不再在折点两侧同时新增支撑。任何相邻站位小于项目最小净距均形成硬失败 `support_station_cluster`。

## 5. 端部平行角撑族

长条形基坑端部短墙的支承由两组镜像平行角撑闭合。第 `k` 根角撑在两条相邻围檩上的链距相同：

```text
d_k = d_0 + k·Δd
```

每根角撑具有独立的墙上承压节点，不允许形成 V 形扇撑、共节点角撑或支撑终止于另一根支撑。角撑数量由端墙长度和围檩硬上限反算，不再通过密集横向直撑替代端部斜向传力。

## 6. 支撑优化硬约束

V3.33.0 的候选硬约束增加：

- 支撑站位最小净距；
- 站位折点聚集数量必须为 0；
- 非环形支撑非法穿越数量必须为 0；
- 普通支撑终止于其他支撑数量必须为 0；
- 墙上高分支汇交和角撑端点拥挤必须为 0；
- 围檩直接支点间距必须满足硬上限；
- 所有自动补强构件必须形成墙—墙、墙—环梁或环梁—墙的明确传力路径。

## 7. 计算恢复顺序

计算前按照以下顺序执行：

1. 识别平面原型和局部主轴；
2. 检测站位聚集、非法穿越、悬空端点和围檩跨；
3. 对存在硬几何失败的旧方案重新运行支撑候选优化；
4. 对采用方案执行回墙和围檩支点硬门禁；
5. 同步施工阶段构件 ID；
6. 执行墙—围檩—支撑计算；
7. 独立完成墙体截面、配筋、变形和稳定性校核。

平面拓扑修复可以消除围檩跨和异常站位问题。墙体弯矩、位移或配筋仍不满足时，需要继续调整墙厚、墙趾、支撑层位、构件刚度或地基参数，系统不会通过增加无效密集支撑掩盖截面问题。

## 8. 升级与旧项目

升级后执行：

```bash
cd /opt/excavation-design
sudo bash start-linux.sh
```

旧项目应重新执行：

1. IDW 地质模型生成或工作区迁移；
2. 形状识别与支撑候选生成；
3. 采用新的候选方案；
4. 重建施工阶段；
5. 重新计算和导出。

原计算合同会因算法版本和支撑拓扑哈希变化而失效。
