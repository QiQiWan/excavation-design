# PitGuard V3.56.0 工程师引导与配筋入口自动恢复

## 本次解决的问题

过去界面把缺资料、结果过期、规则尚未生成记录和专项复核都显示成“待重算”。工程师无法判断重复计算是否有效；配筋入口发现梁缺少设计结果时只阻断，没有调用计算链补齐。

V3.56 建立两级恢复机制：验算矩阵先判断责任类型，配筋任务再执行可安全自动化的恢复序列。自动恢复不会绕过设计基准、支撑拓扑硬失败或专项责任。

## 工程师看到的四类状态

1. **系统可自动补算**：输入完整，但当前计算包缺记录或已经过期；点击补算当前方案。
2. **需补工程资料**：明确列出资料、责任方、录入阶段和位置；重复计算无效。
3. **工程师确认筛查**：通用筛查已具备条件，但项目专用构造、边界或条文适用性需确认。
4. **专项设计/校审**：必须形成并绑定专项验算或校审结论，软件不得自动宣告通过。

## 一键恢复配筋入口

执行顺序固定为：校验当前设计快照 → 补算完整施工阶段 → 回写冠梁/围檩/环梁设计结果 → 生成并写入配筋 → 重新判定深化入口。

若仍不能闭合，任务错误会按梁类别列出具体编号：冠梁重点检查墙段与墙顶剪力工况对应；围檩和环梁重点检查支撑是否在该标高工况激活，以及梁—支撑节点关联是否完整。

## 水平支撑配筋合同

支撑明细表固定展示纵向主筋、端部和跨中箍筋、侧面构造筋、拉结筋及搭接附加筋。问题筛选只控制问题中心，不再隐藏已经通过的完整支撑配筋。

## 安全边界

自动恢复只处理可追溯的计算和数据写回问题。设计基准缺失、拓扑硬失败、勘察资料缺失和专项复核仍由工程师处理；系统会给出原因和完成标准，但不会绕过安全门禁。
