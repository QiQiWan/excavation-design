# PitGuard V3.60.0：配筋入口自愈与梁设计证据补齐

## 问题结论

截图中的 25 个入口阻断来自两个不同状态：1 项为“配筋草案尚未写入构件”，24 项为“围檩只有几何对象，没有施工阶段设计结果”。后者不是继续录入地质资料即可解决的问题：全局计算器过去只为直接收到支撑节点反力的墙面围檩生成结果，同层其余连续围檩即使重复运行相同工况，也仍然没有设计记录。

同时，渐进式设计导航先判断地质设计域、后判断当前计算合同，导致已有有效计算的项目仍在配筋页面显示“自动建立地质模型”，把工程师带回错误阶段。

## 修复后的闭合动作

“一键关闭配筋入口”按以下顺序执行：

1. 校验当前计算合同和施工阶段结果是否有效；
2. 对缺失冠梁读取对应墙段的墙顶剪力包络；
3. 对无直接节点反力的围檩，仅在同一道支撑内选取已计算围檩的最不利弯矩、剪力和挠度包络；
4. 使用现有梁截面完成承载力与挠度设计，不在证据恢复过程中静默改变截面；
5. 写入梁主筋、箍筋、侧面构造筋、拉结筋和节点附加筋；
6. 写入墙和水平支撑配筋合同；如配筋优化确实改变支撑截面，则自动复算并再次配筋；
7. 重新检查 P3 入口。

## 工程边界

- 同层包络补齐只使用已经计算的施工阶段数据，不虚构地勘、荷载或支撑反力。
- 采用同层包络的梁标记为“专业复核”，可以进入节点和空间深化，但正式出图前必须确认围檩连续传力和施工阶段激活关系。
- 若现有截面对保守包络仍不满足，系统保留为硬失败并给出补强/复算入口，不会把失败改写成通过。
- 找不到同层计算证据时继续阻断，并明确列出对象，不跨支撑层借用结果。

## 交互更新

- 当前计算有效时，页面首要动作改为“自动关闭配筋入口”，不再被延期地质任务抢占。
- 配筋策略按钮改为“采用保守 / 采用均衡（推荐） / 采用经济”，点击即执行证据补齐、配筋写入和必要复算。
- P3 诊断按原因分组显示“为什么、怎么处理、涉及对象、系统能否自动处理”。
- 自动可解问题提供统一按钮；工程师不再需要根据 25 条对象逐一寻找入口。
