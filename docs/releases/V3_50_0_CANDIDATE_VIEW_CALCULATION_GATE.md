# PitGuard V3.50.0 候选视图与计算门禁恢复

## 1. 版本目标

V3.50 针对丰收湖项目仍存在的三个直接问题进行收口：

1. A/B/C 候选卡片中的平面预览越出选中边框；
2. “支撑布置评分平面高亮”只有轮廓，支撑、立柱和评分为空；
3. 升级后旧项目仍保留历史诊断候选，计算资格门禁持续阻断。

本版本不降低工程门禁，也不把受控阻断方案强制标记为正式方案。修复重点是让界面读取正确候选，让旧候选按当前算法重新生成，并在全部非拓扑门禁通过时恢复正式计算资格。

## 2. 候选预览越界的根因与修复

### 2.1 根因

样式表末尾存在一条历史兼容规则：

```css
.schemeOverviewSvg,
.schemeDetailSvg,
.coreCandidateSvg {
  width: 100%;
  height: 100%;
}
```

该规则位于候选卡片固定高度规则之后，CSS 级联将候选 SVG 高度覆盖为父级的 100%。卡片内容高度继续增长时，SVG 穿出卡片边界，而蓝色选中边框仍停留在原卡片盒模型范围内。

### 2.2 修复

- 候选主卡 SVG 固定为 180 px，并设置最大高度；
- 详细平面图使用独立的响应式高度；
- 候选卡设置 `min-width: 0`、`overflow: hidden` 和稳定的伸展规则；
- SVG 使用 `preserveAspectRatio="xMidYMid meet"`，不再通过 CSS 拉伸几何；
- 选中边框、预览图和指标区域统一包含在同一个卡片盒模型内。

## 3. 支撑评分平面未显示的根因与修复

### 3.1 根因

评分平面组件固定读取：

```text
project.retainingSystem.supports
project.calculationResults[-1].supportLayoutQuality
```

当项目处于以下状态时，当前体系可能为空：

- 只生成了 A/B/C 候选，尚未正式采用；
- 候选属于诊断方案；
- 旧项目的 selectedCandidateId 已失效；
- 计算结果因几何变化已归档。

此时 A/B/C 卡片已有完整 `planGeometry`，但评分平面仍读取空的当前体系，所以只显示基坑轮廓，支撑数、评分和最大跨均为零或空值。

### 3.2 修复

- A/B/C 卡片选中状态向下游查看器广播；
- 评分平面优先显示当前正在检查的候选 `planGeometry`；
- 候选数据包含轮廓、全部支撑、立柱、体系角色和候选指标；
- 正式采用方案仍优先使用当前计算质量结果；
- 页面明确标识“正式体系”或“诊断候选”；
- 点击 A/B/C 任意卡片后，评分平面、支撑数、立柱数、最大跨和评分同步切换。

## 4. 丰收湖计算持续阻断的根因

### 4.1 当前输入复核

用户上传的钻孔 CSV 与软件中的丰收湖 24 孔样例逐字节一致；上传的 20 点台阶形轮廓、顶标高 0.0 m 和坑底标高 -16.6 m 也与样例完全一致。因此本次阻断并非输入文件发生差异。

### 4.2 持久化状态问题

V3.48/V3.49 之前生成的候选没有当前候选契约标识，并可能保存为：

```text
候选卡片存在
selectedCandidateId = null
formalSchemeEligible = false
当前 supports / columns 为空或与候选不同步
旧计算结果因几何变化归档
```

升级程序后，这些数据库中的候选不会自动重新计算。计算资格检查仍看到旧的受控阻断证据，因而拒绝启动正式计算。单纯修复支撑端点承压面不足以替换整套历史候选。

## 5. 当前候选契约与有界自动恢复

### 5.1 候选契约

每个新候选记录：

```text
candidateContractVersion = 3.50-bearing-closure-v2
```

计算门禁可以区分：

- 当前算法生成的候选；
- 历史候选；
- 缺失当前采用方案；
- 当前正式方案已经有效。

### 5.2 计算前恢复

仅当计算阻断类别为 `Q-TOPOLOGY` 时，隔离 worker 执行一次有界恢复：

```text
识别旧候选契约或缺失当前方案
→ 确认地质覆盖
→ 保留围护墙设计
→ 按当前端点承压面规则重新生成最多 3 套候选
→ 采用硬约束通过的推荐方案
→ 失效旧计算结果
→ 重新进行完整设计资格检查
→ 资格通过后继续正式计算
```

限制条件：

- 默认最多 12 次试算；
- 候选池最多 6 套；
- 支撑构件上限 800；
- 不并行保留完整项目副本；
- 几何、坐标、地质或设计基准门禁不会被自动绕过。

若当前算法仍无法生成正式方案，系统保持阻断并给出真实控制类别。

## 8. 升级后的丰收湖项目处理

升级后可直接点击“计算当前方案”。若项目保存的是历史候选契约且其他设计资格均通过，worker 会先生成当前契约候选并采用正式方案，然后重新检查资格。

为了人工检查 A/B/C 几何差异，推荐流程仍为：

```text
打开围护方案
→ 重新生成 A/B/C
→ 点击各候选检查评分平面
→ 采用正式候选
→ 进入计算验算
```

旧计算结果已经因几何或标高变化归档，必须重新生成施工工况和正式计算结果，不能与新支撑拓扑混用。
