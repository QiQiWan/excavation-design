# PitGuard V3.45.0 设计基准、校核闭环与配筋深化专业版

## 1. 本轮目标

V3.45.0 在 V3.44 有界候选搜索和隔离 worker 的基础上，集中解决以下工程使用问题：

1. 将工程等级、基坑安全等级、场地复杂程度、周边环境等级、规范体系和荷载组合前置为设计必需条件；
2. 将 A/B/C 候选从几何截图提升为可解释、可居中、可完整计算的工程方案比选；
3. 对已有计算结果统一形成强度、刚度和稳定性安全系数矩阵；
4. 将配筋深化从同步接口和单一出图闸门中拆出，形成后台草案生成、构造校核和正式出图资格三个层次；
5. 补齐水平支撑纵筋、分布筋、箍筋、拉结筋和节点附加筋的三维可视化；
6. 为地质、围护结构、计算云图和配筋模型统一增加全屏、复位和图层显隐入口；
7. 保持核心界面精炼，详细条文、参数和构件明细按需展开。

## 2. 六阶段核心流程

默认工作流调整为：

1. **设计基准**：确认工程分级、规范体系、荷载组合、材料与安全储备；
2. **工程输入**：钻孔、地层、地下水、基坑轮廓、标高和周边约束；
3. **围护方案**：围护墙、围檩、支撑、角撑、立柱和 A/B/C 候选；
4. **计算验算**：施工阶段、内力、变形、强度、刚度和稳定性；
5. **配筋深化**：围护墙、围檩、支撑、节点与施工构造；
6. **成果交付**：计算书、施工图、IFC、审查清单和机器审计数据。

设计基准未确认时，候选生成、核心设计和完整计算均由后端任务门禁阻止，避免仅靠前端按钮状态控制。

## 3. 设计基准合同

### 3.1 项目级输入

新增项目级参数：

- 项目工程等级；
- 基坑安全等级；
- 场地复杂程度；
- 周边环境等级；
- 临时支护或支护兼永久阶段；
- 国家标准核心、国家加地方标准或专项审查体系；
- 标准、保守或项目自定义荷载组合；
- 永久作用分项系数 `γG`；
- 可变作用分项系数 `γQ`；
- 组合值系数 `ψ`；
- 重要性系数；
- 稳定安全储备比例；
- 地基承载力特征值；
- 默认混凝土、钢筋和保护层；
- 抗弯和抗剪设计路径。

### 3.2 荷载组合表达

系统显示并保存以下组合合同：

- 承载能力极限状态基本组合；
- 正常使用极限状态标准组合；
- 正常使用极限状态准永久组合。

界面明确区分“系数配置”和“工程作用项目”。系统不会因为给出通用组合表达式而推定项目已经完成全部荷载调查。

### 3.3 规范职责

核心规范映射包括：

- GB 55003-2021：地基基础强制性底线；
- JGJ 120-2012：基坑支护体系、变形和稳定性；
- GB 50009-2012：作用分类和荷载组合；
- GB 50007-2011：地基承载力与基础校核；
- GB 55008-2021：混凝土结构强制性要求；
- GB/T 50010-2010（2024 年局部修订）：混凝土抗弯、抗剪、配筋和构造。

地方标准和项目专项条款仍需项目设计人员确认。

## 4. A/B/C 方案比选重构

### 4.1 视图居中和统一缩放

候选预览按完整轮廓、支撑和立柱的几何包围盒计算中心，并根据固定目标宽高比扩展视窗。三个方案采用一致的留白规则，长条形、旋转形和局部台阶形平面均保持在画面中心。

### 4.2 比选层级

候选卡片显示：

- 结构体系；
- 支撑和立柱数量；
- 最大支撑跨度；
- 拓扑硬约束；
- 正式计算状态；
- 最大位移、支撑轴力、围檩内力和最小安全系数（已完整计算时）。

候选生成阶段的代理分和完整施工阶段计算结果分开展示。只有完成 A/B/C 完整计算的候选才进入正式工程排名。

### 4.3 体系差异

候选多样性继续以体系族、目标支撑间距、立柱服务跨、线位模式、支撑数量和立柱数量控制。若三套完整计算结果和传力路径高度一致，系统提示其属于线位微调，不作为正式方案差异的充分依据。

## 5. 强度、刚度和稳定性安全系数矩阵

### 5.1 统一计算表达

对“作用效应不大于抗力/允许值”的检查：

- 利用率：`u = Sd / Rd`；
- 安全系数：`SF = Rd / Sd = 1 / u`。

对“提供值不小于最低要求”的检查：

- 利用率：`u = required / provided`；
- 安全系数：`SF = provided / required`。

显示内容包括设计值、规范限值、单位、安全系数、利用率、控制工况、控制构件、规范和条款引用。

### 5.2 专业校核清单

系统形成以下完整检查目录：

**强度**

- 围护墙抗弯和抗剪；
- 围檩抗弯、抗剪和局部承压；
- 水平支撑轴压和节点承载力；
- 临时立柱轴压；
- 立柱基础地基承载力。

**刚度与使用性能**

- 围护墙最大水平位移；
- 围檩挠度；
- 混凝土裂缝控制。

**稳定性**

- 水平支撑稳定；
- 临时立柱稳定；
- 围护墙嵌固稳定；
- 坑底抗隆起；
- 渗流稳定；
- 承压水突涌；
- 支护体系整体稳定。

当已有计算模块提供数值时，页面输出真实安全系数。当前模型尚未形成数值或资料不足的项目保留为“人工复核”，不会生成虚构安全系数。

### 5.3 结果汇总

页面按强度、刚度、稳定性分组显示：

- 最小安全系数；
- 平均安全系数；
- 通过、警告、失败和人工复核数量；
- 控制校核项；
- 原始规范检查状态。

重复逐构件记录按规则合并，只保留最不利构件，同时记录涉及对象数量。

## 6. 配筋深化闭环

### 6.1 后台隔离执行

配筋应用改为独立 `rebar_design` 后台任务。API 只提交任务并查询进度，具体配筋组合、截面调整、节点校核和结果保存由一次性 worker 完成。

### 6.2 三层结果

配筋流程分为：

1. 配筋草案生成；
2. 构件和节点可施工性校核；
3. 施工图发行资格。

存在复核项时仍可生成和查看审查版配筋草案；只有正式计算、构造和出图闸门满足时才允许施工图发行。

### 6.3 适用条件

生成配筋草案至少需要：

- 当前围护体系；
- 当前施工阶段计算结果；
- 构件截面和材料参数。

前端不再因部分构造诊断为警告而完全禁用草案按钮；后端仍对缺少计算结果或围护体系给出明确错误。

## 7. 水平支撑完整钢筋三维显示

水平支撑三维预览新增并保留：

- 纵向主筋；
- 侧面分布筋；
- 端部和跨中封闭箍筋；
- 拉结筋；
- 节点及局部附加筋；
- 纵筋错开搭接示意。

每根支撑内部先为各钢筋族预留采样配额；全局模型再次按钢筋族和构件编号分层抽样，防止全部显示预算被纵筋占用。接口摘要增加 `supportBarTypesPresent`，便于前端和自动测试确认各类钢筋均已进入场景。

逐根钢筋仍属于浏览器抽样层；完整加工数量、接头、弯钩、套筒和吊装构造以 CAD 钢筋表和人工复核为准。

## 8. 统一三维交互

地质模型、围护结构模型、墙体云图和配筋模型统一使用全屏容器，提供：

- 全屏进入和退出；
- 视角复位；
- 图层显隐；
- 构件、标签、问题和钢筋类别筛选；
- 透明度或线框模式（由具体查看器支持）。

Three.js 仍按需动态加载，不进入项目列表和普通首屏。

## 9. 界面精炼原则

默认页面遵循：

- 结论优先；
- 一个阶段一个主图；
- 设计值、限值和状态同屏；
- 长说明收纳到“规范依据”“查看原因”和“处理建议”；
- 内部 UUID、JSON 和调试证据不进入默认视图；
- 机器审计数据保留在交付包数据与审计目录。

顶部状态只保留当前阶段、工程/安全等级、采用方案、控制工况、问题数量和成果状态。

## 10. 数据和性能边界

V3.45 延续 V3.44 的有界执行策略：

- 候选试算和完整候选池有上限；
- 完整计算和配筋通过隔离 worker 执行；
- 网页使用工作区和轻量预览；
- 完整施工阶段数组、逐根钢筋、地质网格和导出文件外部化；
- 三维钢筋采用分层抽样，避免浏览器加载全部物理钢筋。

本轮核心自检结果：

- 候选方案：3；
- 当前完整计算：1；
- 支撑/立柱：36/4；
- 配筋检查：240；
- 完整快照约 1.44 MB；
- 工作区约 1.12 MB；
- 外部对象约 0.70 MB；
- 核心链路约 4.24 s；
- 隔离 worker 链路约 4.83 s，任务期间 worker 有效内存约 234 MB 峰值量级。

不同项目规模、施工阶段数量和构件数量会导致实际耗时变化。

## 12. 当前边界

1. 安全系数矩阵已覆盖专业校核目录，但只有已有可靠数值模型的项目输出数值；缺失项目保持人工复核；
2. 荷载组合系数由项目确认，系统不自动推断周边建筑、车辆、堆载和特殊施工荷载；
3. 地方标准、审图要求和项目专项条件需在项目级标准体系中补充；
4. 钢筋三维模型用于设计检查和沟通，正式加工仍以经审签施工图和钢筋表为准；
5. A/B/C 正式排序需要对候选分别完成施工阶段计算。

## 13. 后续优化路线

### P0：工程可用性收口

- 在真实大型 Windows 工程上复测设计基准、候选、计算、配筋和导出全链路；
- 对每项强度、刚度和稳定性建立“已计算/缺资料/未实现/人工复核”状态矩阵；
- 对配筋报错增加构件、截面、工况和输入字段级定位；
- 将任务阶段、内存和结果摘要写入项目审计记录；
- 补齐 `ezdxf` 环境下的施工图压力测试。

### P1：规范参数和多方案深化

- 建立基于工程等级、安全等级和环境等级的项目参数模板；
- 对荷载组合增加施工阶段启停和作用分组；
- A/B/C 引入完整计算 Pareto 排序：位移、内力、安全系数、材料量和施工复杂度；
- 增加项目级构件目录、节点目录和真实型钢截面数据库；
- 配筋增加裂缝宽度、锚固、搭接、套筒和拥挤度定量检查。

### P2：计算模型深化

- 墙—围檩—支撑—立柱统一空间刚度模型；
- 节点半刚性、预加轴力、温度和安装偏差；
- 混凝土开裂刚度、徐变和收缩；
- 降水失效、承压水、超挖和局部渗流不利工况；
- 关键节点壳/实体局部子模型。

### P3：数据闭环和企业应用

- 监测数据反演土参数和支撑刚度；
- 施工阶段预测和风险预警；
- 多项目模板、权限、校审签名和版本冻结；
- 规范版本生命周期管理；
- 与设计院标准图集、构件库和审查规则库联动。
