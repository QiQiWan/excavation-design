# PitGuard V3.87.0：V3.82—V3.87 全面迭代实施与评估报告

## 1. 迭代目标

本轮将 V3.81 中偏重施工计划、现场快照和偏差事件管理的主流程收缩为设计院工程师能够实际控制的专业设计流程。主流程固定为九个阶段：

1. 设计依据与参数确认；
2. 工程输入与资料资格；
3. 围护方案搜索；
4. 围护墙—围檩—支撑—立柱联合设计；
5. 计算核验与逐构件包络；
6. 配筋和节点构造闭合；
7. 施工图生成与图面质检；
8. 计算书生成与条文追溯；
9. 校审、统一快照与发行。

施工单位来文、监测周报和现场异常通知降级为外部协同资料。只有可能改变原设计边界的内容才生成设计复核请求。旧版施工计划、现场快照和偏差事件结构保留兼容读取，不再参与首次设计发行完整度。

## 2. V3.82：设计核心收缩

### 已完成

- 新增九阶段设计核心工作流及统一就绪度；
- 新增 `ExternalCollaborationRecord`，用于接收施工单位来文、监测报告、现场反馈和设计联系单；
- 新增 `DesignReviewRequest`，仅处理是否超出原设计条件、是否复算、是否变更设计；
- 旧版施工计划、现场快照和偏差事件退出设计师主导航和首次设计发行门禁；
- 前端新增 `DesignCoreWorkflowPanel`，集中显示当前结论、阻断原因和下一步动作。

### 业务边界

外部现场数据缺失不会降低设计就绪度。系统不再要求设计人员维护每日开挖状态、现场预加轴力、阶段验收或完整偏差事件链。

## 3. V3.83：规范和参数治理

### 参数治理

新增 `ParameterProvenanceRecord`，每个控制参数记录：

- 数值和单位；
- 参数来源；
- 置信度；
- 是否经过确认；
- 是否允许用于正式设计；
- 修改后影响的方案、计算、配筋、图纸和报告；
- 人工确认人和时间。

来源类型包括勘察报告、项目批准、建设单位提供、规范规定、企业标准、推导值和软件建议。软件建议值与未确认参数可以用于草案，不具备正式设计资格。

### 条文执行证据

新增条文级规则证据视图，区分：

- `calculated`；
- `missing_input`；
- `not_applicable`；
- `not_calculated`；
- `not_implemented`；
- `manual_review`。

规则覆盖率只表示软件已执行证据，不能解释为规范全文自动审查完成。

## 4. V3.84：方案搜索与联合设计

### 五级搜索结构

方案搜索分为：

1. 围护体系；
2. 支撑体系族；
3. 支撑拓扑与转接体系；
4. 墙厚、嵌固、支撑标高、构件截面；
5. 完整计算和真实结果排序。

### 候选多样性和正式采用门禁

- A/B/C 必须来自不同体系族或具有实质性拓扑差异；
- 正式采用方案必须完成当前版本完整计算；
- 代理评分只能用于快速筛选；
- 采用方案、完整计算结果、设计基准和拓扑哈希必须一致；
- 候选方案支持从布局修复区和正式候选区读取，避免候选结果在后续报告中丢失。

本次合成 L 形算例形成三类候选：闭合内环、中心汇交框架和环内弦杆框架，三类候选均执行了隔离完整计算。

## 5. V3.85：计算、逐构件包络和配筋闭合

### 逐构件包络

统一输出：

- 对象 ID；
- 工况 ID；
- 响应类型；
- 显式单位；
- 最小值、最大值和包络值；
- 控制工况与结果哈希。

当前已覆盖围护墙位移、弯矩和剪力，支撑轴力，围檩挠度、弯矩和剪力。正式包络禁止无单位数据混合。

### 配筋闭合

新增构件级配筋闭合服务，检查：

- 实际配筋面积与需求面积；
- 有效高度和截面反馈；
- 箍筋、构造筋和节点附加筋；
- 锚固、搭接和承压构造；
- 需要扩大截面或重新计算的构件；
- 正式配筋结果与计算包络的当前性。

自动闭合状态和正式构件级钢筋方案分开表达。闭合筛查通过不等于已形成可发行的逐构件钢筋表。

任何成果与当前快照不一致时禁止正式发行。
