# PitGuard V3.35.0：人机交互、系统稳定性与围护设计可靠性深化

## 1. 本轮审查结论

系统已经形成“地质—开挖—围护—支撑—施工阶段计算—配筋—图纸/IFC—质量闸门”的完整骨架，并具备异形平面识别、候选方案优化、支撑节点、围檩连续梁、施工阶段计算、外部大对象存储、隔离任务和正式交付门禁。当前限制集中在四个层面：

1. 操作状态与工程证据混在同一界面，用户容易将“方案生成完成”理解为“设计已通过”。
2. 快速筛查、历史计算和当前完整计算的证据边界不够严格，存在旧轴力被新方案复用的风险。
3. 长期运行的可观测性、瞬时故障恢复和资源退化提示仍有缺口。
4. 构件快速模型仍是候选筛查模型，节点局部、三维二阶、施工误差和土参数不确定性需要更高等级的计算与工程验证。

V3.35.0 优先修复会导致错误决策的链路问题，并建立可继续扩展的证据分级框架。

## 2. 已完成的代码优化

### 2.1 人机交互

- 工作台增加“系统可靠性条”，持续显示 API P95 延迟、服务端错误率、并发请求、任务运行/排队、API 进程内存和监控路径基数。
- 水平支撑深化面板增加 A-D 证据等级、当前计算合同状态、分阶段轴力覆盖率、地质证据状态和正式设计就绪度。
- 原有单一 `hardPass` 仍保留用于候选兼容，但界面和新接口使用三个明确状态：
  - `screeningPass`：候选构件快速筛查无硬失败；
  - `calculationReady`：当前方案完整分阶段轴力覆盖率达到 95% 且筛查通过；
  - `formalDesignReady`：进一步满足节点闭环、地质证据和计算质量包要求。
- HTTP 错误附带 `X-PitGuard-Request-Id`，便于前端报错与后端日志一一对应。
- 主动取消与超时分别提示，避免用户把取消操作误认为服务器故障。

### 2.2 系统稳定性

- GET 请求对网络断连及 429、502、503、504 增加最多一次自动重试；写操作默认不重试，避免重复创建、重复计算或重复导出。
- 运行指标对项目、任务、计算结果等动态 ID 做路径归一化，并设置 512 个聚合路径上限，避免长时间运行产生无界字典。
- 新增当前请求数、最大并发、客户端错误率、慢请求率和最近服务端失败记录。
- `/api/system/readiness` 增加数据库磁盘空间、API 进程内存软限制和任务排队判定，区分 `ready`、`degraded` 与 `not_ready`。
- 软件版本、算法版本、规则集版本和导出协议统一升级为 3.35 系列，消除结果追溯元数据错位。
- 当前混凝土标准名称统一为 `GB/T 50010-2010（2024年局部修订）混凝土结构设计标准`。

### 2.3 支撑设计合理性与可靠性

- 完整计算中的支撑深化评估改为直接消费本次 `stage_results`，不再在结果落库前回查历史计算。
- 独立调用深化设计时只允许采用“当前计算合同”对应的最新结果；合同过期时，历史轴力全部忽略并生成阻断性整改建议。
- 轴力证据优先级固定为：当前分阶段计算包络 > 当前构件存储设计轴力 > 分担宽度快速估算。
- 新增每根支撑轴力来源统计和覆盖率，避免估算轴力被包装成完整计算结果。
- 新增地质证据审计：土层数量、钻孔数量、重度/内摩擦角完整度、低置信度土层、参数来源、地质模型和覆盖审计。
- 正式设计就绪需要同时满足：
  - 快速稳定/偏心组合无硬失败；
  - 当前分阶段轴力覆盖率不低于 95%；
  - 支撑—围檩节点全部闭环；
  - 土层控制参数完整且无低置信度阻断；
  - 地质模型覆盖通过；
  - 计算 assurance 允许正式发行。

## 3. 设计模型的现阶段边界

V3.35.0 的快速构件模型用于候选排序、截面初选和风险定位。以下内容仍需完整计算或专项分析：

- H 型钢采用等效截面时，需要替换为实际型钢表和规范稳定系数曲线。
- 钢支撑二阶效应、初弯曲、残余应力、节点半刚性及局部屈曲尚未形成统一高保真模型。
- 混凝土支撑需要进一步耦合开裂刚度、徐变收缩、施工龄期和节点区受压/受剪。
- 支撑节点、围檩转角、斜撑落点和墙体局部承压需要壳/实体子模型或经过验证的构造计算模块。
- 快速土压力估算使用平均重度和内摩擦角，正式计算应继续使用分层土水压力、支点形成/拆除和开挖阶段路径。
- 不规则基坑仍需用图论传力路径、结构冗余与施工分区联合评价，几何规则不能代替结构分析。
- 正式工业级可靠性需要第三方软件对标、实际工程监测反演和校审签署。

## 4. 后续优化优先级

### P0：错误决策防护

- 将所有计算、配筋、图纸和 IFC 入口统一读取 `formalDesignReady` 与全局发行门禁。
- 方案、几何、土层、施工阶段或构件修改后，立即失效所有依赖结果，并在界面展示失效原因和影响范围。
- 对计算异常、缺失阶段、空结果和求解回退建立禁止“完成”进度状态的统一协议。

### P1：结构模型深化

- 引入实际钢截面库与 GB 50017 轴压稳定曲线，补充整体/局部屈曲、二阶效应和初始缺陷。
- 建立墙—围檩—支撑—立柱空间杆系的刚度协调、节点半刚性和施工阶段激活/钝化模型。
- 对转角、凹角、端头、扩宽区、坡道和出土口建立分区模板与局部再优化器。
- 把“单构件墙对路径”升级为 k-edge/k-node 连通度、关键构件移除和换撑阶段鲁棒性分析。

### P2：岩土与施工可靠性

- 土参数采用区间/分布描述，自动生成不利组合和敏感性排序。
- 引入地下水时变、降水失效、局部承压水和渗流—变形联合工况。
- 建立安装偏差、温度、预加轴力损失、支撑松弛和开挖超挖的施工误差包络。
- 接入监测数据进行参数反演、预警阈值更新和剩余阶段预测。

### P3：工程交付与验证

- 建立典型矩形、长条形、L/U/T/C 形、深厚软土和高水位基准案例库。
- 与至少一种成熟岩土/结构软件逐阶段对标位移、弯矩、剪力、支撑轴力和稳定系数。
- 对施工图执行尺寸链、构件编号、标高、材料表、钢筋表、大样引用和版本一致性自动审查。
- 建立真实项目“设计值—监测值—反演值”闭环，形成适用范围和模型偏差统计。
