# PitGuard V3.65.1：恢复重建、配筋与界面闭环

## 版本性质

原 V3.65.0 临时交付文件已不可用。本版本以仍可校验的 V3.62.0 完整源码包为基线，依据已确认的 V3.63–V3.65 变更目标重新实现并重新测试。版本号提升为 V3.65.1，校验值重新生成，不沿用原 V3.65.0 的哈希。

## 计算与配筋修复

- 围檩斜截面承载力由混凝土项 `0.7*ft*b*h0` 与实际多肢箍筋项 `fyv*Asv*h0/s` 共同组成；设计结果记录混凝土贡献、箍筋贡献、总承载力、肢数、直径、间距和利用率。
- 围檩截面搜索遍历完整实用候选集，在全部通过项中按截面面积、截面比例和箍筋用量择优；不会在搜索结束后回退到原小截面。
- 无直接支撑端点的短回墙，在两端相邻墙面存在连续传力证据时，将角部传力代理纳入局部围檩计算，并只将物理支撑写入工程量和构件轴力。
- 水平钢筋混凝土支撑同时检查总纵筋率、单侧纵筋率、轴压承载力、净距和截面上限。1.6m×1.6m 回归对象采用 `24D32`，总纵筋率约 0.754%，单侧约 0.220%。
- 大截面支撑生成复合箍筋。上述对象为 8 肢，端部 `D14@120`、跨中 `D14@180`，并继续输出分布筋、拉结筋和搭接附加筋。
- 墙筋锚固与搭接采用统一函数；生成、构造校核和下料表不再分别使用 35d、30d、40d 等互相冲突的长度。1m 分析分区不会再被误作整根墙筋可用锚固空间。
- 配筋后台任务在计算证据缺失或失效时先自动补算，并在截面变化后执行最多三轮“补强—复算—重新配筋”。

## 数据、状态和稳定性修复

- 正式计算从内容寻址成果中按需恢复完整地质面与体数据；普通工作区继续使用轻量预览。
- 安全目标取基坑安全等级、企业/地方模板与项目提高值中的最大值。低于底线的保存请求会被提升，并记录安全目标审计。
- 项目设计输入变更在兼容保存分支中同样清除旧计算结果和发行状态。
- 自适应资源策略继续根据可用内存降档，同时尊重 `PITGUARD_API_FULL_PROJECT_LIMIT_MB` 管理员硬上限。
- 旧项目缺失派生墙段时，计算预检按当前闭合轮廓重建墙段，并保留既有加深墙趾、材料和配筋。

## 界面修复

- 配筋主操作改为“一键修复并闭合配筋”；计算证据缺失时按钮仍可启动自动补算。专业人员可选择“仅更新配筋”。
- 支撑五类钢筋预览不受“仅问题项”表格筛选影响。
- 不利工况、钢筋类型、构件角色、墙体分区、深化状态和可施工性校核采用中文工程术语。
- 配筋与钢筋详图长表格支持横向滚动，窄屏下不再逐字挤压。

## 工程边界

软件闭合表示计算证据、构件设计记录、钢筋合同和导出数据一致。正式施工成果仍需项目设计、校核、审核和批准人员根据适用规范、勘察资料、施工组织和企业标准签署。
