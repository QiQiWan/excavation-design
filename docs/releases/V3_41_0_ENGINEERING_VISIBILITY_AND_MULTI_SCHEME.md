# PitGuard V3.41.0 工程可视化、规范对应与多方案设计

## 1. 版本目标

V3.41 在 V3.40 五阶段核心工作台上增加必要的工程解释和可视化，不恢复此前被移除的工业成熟度、监测、基准算例和高级套件面板。默认界面仍围绕工程输入、围护方案、计算验算、配筋深化和成果交付五个步骤。

## 2. 关键计算与规范对应

核心状态接口增加 `standards` 字段，按五个设计阶段返回最多五项相关规范。前端默认只显示规范编号，点击“查看对应关系”后显示：

- 规范名称和属性；
- 当前步骤关注内容；
- 软件已实现范围和边界；
- 官方来源入口。

计算书正文每个步骤只列两项最主要规范，完整的“设计流程—关键计算—规范条文对应矩阵”移至附录，避免项目管理正文再次变成规范目录。

当前核心规范包括 JGJ 120-2012、GB 55003-2021、GB 55008-2021、GB/T 50010-2010（2024年局部修订）、GB 50017-2017、GB 50007-2011 和 GB 50009-2012。系统提供条文对应和实现边界，项目适用版本、地方标准及专项要求仍需专业人员确认。

## 3. 五阶段轻量可视化

本版本使用 SVG 和有界摘要数据实现可视化，未重新引入 Three.js 到核心构建。

### 3.1 工程输入

- 钻孔控制范围；
- 地层面轻量剖面；
- 地质模型覆盖状态；
- 最大外推距离。

完整 IDW 网格继续存储在外部对象，网页只读取 `surfacePreviews`。

### 3.2 围护方案

- 基坑轮廓；
- 围护墙边界；
- 支撑与角撑传力线；
- 临时立柱；
- 当前采用方案或候选方案预览。

### 3.3 计算验算

- 侧向压力沿深度分布；
- 墙体位移、弯矩和剪力包络；
- 支撑轴力平面云图；
- 稳定性安全系数分布。

### 3.4 配筋深化

- 围护墙钢筋笼截面示意；
- 墙体配筋分区数量；
- 支撑配筋组和节点附加筋；
- 配筋检查数量。

示意图不生成虚构钢筋参数。无法从当前配筋方案提取直径、间距或保护层时，界面明确提示以钢筋表为准。

## 4. 稳定性安全系数分布

后端新增稳定性汇总模型，统一提取：

- 嵌固稳定；
- 坑底抗隆起；
- 渗流稳定；
- 承压水突涌；
- 整体稳定；
- 施工阶段中明确给出安全系数和限值的检查。

每项结果包含计算值、规范限值、限值比、状态、规范和检查重点。限值比小于 1.0 判为不通过，1.0～1.10 作为优先复核区间。

可靠性处理：

- 旧数据中的 `999` 等“不控制”哨兵值不进入安全系数分布；
- 普通利用率、降水阶段状态和构件稳定利用率不冒充安全系数；
- 计算结果有系数但缺少适用限值时标记为“需人工复核”，不自动给出通过结论；
- 当前采用方案的完整计算结果可直接回填至多方案表，其他候选仍显示“待完整计算”。

## 5. 多方案备选

候选生成支持 `requireDiverseSchemes=true`。搜索维度包括：

- 可用结构体系族；
- 支撑目标间距；
- 立柱最大服务跨；
- 支撑线位模式；
- 支撑和立柱数量；
- 拓扑硬约束及构造评分。

每个候选保存 `diversityBasis`，用于解释 A/B/C 的差异。排序优先选择不同体系族，再选择不同立柱跨、支撑数量或线位配置，避免只输出三套轻微平移的同类方案。

默认核心任务保留三套候选，只计算当前采用方案。用户点击“完整比选”后，才对 A/B/C 分别执行施工阶段计算并比较最大位移、支撑轴力、围檩内力、稳定性和工程评分。

## 6. 管理者可读交付

### 6.1 计算书

DOCX 主体调整为：

1. 项目管理摘要；
2. 工程条件与输入资料；
3. 设计依据与规范对应；
4. 围护结构多方案比选；
5. 当前采用围护结构；
6. 施工阶段内力与变形；
7. 基于规范的稳定性校验；
8. 配筋设计与构造摘要；
9. 校核结论与复核清单；
10. 成果使用与责任边界。

完整规则矩阵、计算原理和版本哈希移至附录。报告图表使用可用的 CJK 字体，避免稳定性图中的中文标签显示为空框。

### 6.2 协同交付包

交付包增加 `00_release/项目管理摘要.html`，作为项目经理和专业负责人的首读文件，集中展示：

- 发行状态；
- 核心指标；
- A/B/C 方案；
- 稳定性安全系数；
- 管理行动项；
- 优先阅读成果链接。

`30_reports` 只保留人读计算书。计算合同和质量包移动到 `90_audit`，完整项目快照及机器台账保留在 `50_data`。交付索引将文件分为“优先阅读成果”“技术附件”“机器数据与审计文件”，JSON 不再作为主交付内容展示。

## 7. 数据与性能边界

- 可视化读取现有工作区摘要，不复制完整项目；
- 候选预览最多显示 1200 根支撑和 500 根立柱；
- 完整地质网格、逐根钢筋、施工阶段完整数组和导出工件继续外部化；
- A/B/C 完整计算保持显式触发；
- 核心前端构建未重新引入 Three.js。

## 8. 验收重点

1. 五个阶段均显示简明规范对应；
2. 地质、围护、内力变形、支撑轴力和配筋模块均有明确空状态；
3. 稳定性分布不包含哨兵值或普通利用率；
4. 三个候选至少在体系、立柱跨、数量或线位上存在可解释差异；
5. 当前采用方案有计算值，未完整计算候选明确标记；
6. 计算书正文无大段 JSON；
7. 项目管理摘要是交付包首读入口；
8. `30_reports` 不再混入计算合同和质量包 JSON。

## 9. 最终质量验证

本版本在最终发布目录完成以下验证：

- V3.11 规范、钢筋和计算书回归 + V3.41 专项回归：14 项通过；
- V3.36–V3.40 资格、运行时、启动恢复、增量工作区和核心重整回归：26 项通过；
- V3.24 计算合同与交付发行回归：5 项通过；
- 前端完整测试：19 个测试文件、32 项通过；
- TypeScript 编译和 Vite 生产构建通过；
- 核心端到端自检：3.591 s，生成 3 个候选、计算 1 个采用方案、完成 240 项配筋检查；
- 自检项目完整快照 1.91 MB，网页工作区 1.59 MB，外部对象 0.70 MB，总逻辑数据 2.61 MB；
- DOCX 示例计算书逐页渲染检查完成，正文图表和规范标签均可正常显示。

额外修复了隔离基准算例运行器的 `PYTHONPATH` 根目录错误，以及计算结果在深化证据附加后结果哈希未覆盖最终载荷的问题。施工图全量 CAD 包测试仍属于高耗时扩展验证，不纳入核心模式的启动门禁。

## 10. 报告可读性收敛

最终计算书进一步完成以下收敛：

- 图题、坐标轴和图例统一使用中文工程术语；
- 施工阶段以“第 N 阶段（开挖至某标高）”显示，不暴露内部 UUID；
- 支撑轴力图使用构件编号，不显示哈希尾号；
- 重复构件检查按规则、状态和结论合并，正文只显示控制值、影响对象数和处置要求；
- 常见围檩、节点、支撑和地质检查名称转换为项目管理者可读名称；
- 英文内部规则说明转换为中文规范适用范围；
- 逐构件、逐阶段机器记录继续保留在审计归档中，不占用主计算书篇幅。
