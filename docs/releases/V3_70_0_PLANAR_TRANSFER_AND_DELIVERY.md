# PitGuard V3.70.0 异形支撑平面框架与交付闭环

## 1. 版本定位

V3.70.0 将 V3.66.0 的“异形轮廓识别—闭合内环几何—安全阻断”继续推进到可计算、可比较、可追溯的设计链路。此次实现按 V3.67、V3.68、V3.69、V3.70 四个顺序阶段合并交付，每一阶段均以上一阶段的数据合同和门禁为前提。

本版本的核心状态划分为：

1. `geometryClosed`：几何构件闭合且不存在断环；
2. `loadPathClosed`：每个围护墙面均能连接到转接体系；
3. `structuralModelClosed`：平面框架/桁架模型可组装、可求解且残差满足要求；
4. `constructionStageClosed`：施工阶段包络中的转接体系均完成求解；
5. `formalCalculationReady`：前四项满足且转接构件已产生设计结果；
6. `officialIssueReady`：当前拓扑对应的自动深化证据、专业复核和审批均有效。

几何闭合不再直接等同于正式计算或正式出图资格。

## 2. V3.67.0：工程真实性与状态门禁

### 2.1 数据合同

转接梁增加显式角色与拓扑标识：

- `transfer_ring_beam`：闭合环梁；
- `transfer_frame_beam`：中心汇交框架梁；
- `transfer_brace`：环内弦杆/斜杆；
- `partition_beam`：预留给临时分隔体系。

构件同时记录：

- `transferSystemId`；
- `transferZoneId`；
- `startNodeId`、`endNodeId`；
- `loadPathId`；
- `analysisStatus`。

支撑拓扑族增加 `zoned_ring_transfer`、`transfer_frame`、`ring_truss`、`partition_wall` 和 `center_island`，避免使用构件编号前缀代替结构语义。

### 2.2 几何重构与立柱

负缓冲内环生成后执行：

- 短边消除；
- 近共线边合并；
- 闭合性检查；
- 转接节点与墙面覆盖检查。

立柱由“支撑线交点”扩展为“环梁角点、框架节点、长跨中间点和高集中力节点”的候选布置，减少径向撑无交点时立柱漏配。

### 2.3 状态门禁

候选预筛仅可获得 `proxyCalculationReady=true`。只有平面模型求解和施工阶段包络完成后，系统才设置 `formalCalculationReady=true`。正式发行继续要求与当前拓扑哈希一致的专业审批。

## 3. V3.68.0：平面框架求解闭环

### 3.1 单元与自由度

新增 `planar_transfer_frame.py`。每个平面框架节点具有：

\[
\mathbf d_i=[u_i,\;v_i,\;\theta_i]^\mathrm T
\]

转接梁采用二维 Euler–Bernoulli 框架单元，径向支撑采用二维桁架单元。支撑端点会切分转接梁，使集中支撑反力作用于真实节点。

每道支撑层求解：

\[
\mathbf K_{\mathrm{plan}}\mathbf u=\mathbf F
\]

并检查：

- 刚度矩阵可解性；
- 条件数；
- 相对平衡残差；
- 节点位移；
- 转接梁轴力、剪力和弯矩；
- 支撑轴力与构件设计结果映射。

### 3.2 施工阶段

现有施工阶段支撑作用力按阶段映射至平面框架模型，形成转接梁内力与位移包络。转接梁结果进入围檩/梁构件设计链路，自动生成设计结果和配筋合同。

### 3.3 当前力学边界

当前耦合为墙体/围檩计算向平面转接框架传递作用力的一次耦合。平面框架重分配反力尚未迭代回写围护墙竖向模型。当前模型也未显式包含：

- 三维扭转和竖向偏心；
- 节点刚域与经过标定的半刚性连接；
- 支撑—墙—平面框架反力迭代收敛；
- 空间立柱二阶效应。

因此本版本适合设计阶段体系比较和构件初步设计，正式工程仍需专业复核与外部基准校验。

## 4. V3.69.0：多拓扑 A/B/C 与完整计算重排

### 4.1 三类正式候选

正交凹形基坑默认生成三种具有不同构件关系的体系：

| 模板 | 拓扑类别 | 主要构件 |
|---|---|---|
| `compact_elbow_ring` | `closed_ring` | 闭合内环梁 + 径向支撑 |
| `junction_hub_frame` | `junction_hub_frame` | 闭合内环梁 + 中心汇交框架梁 + 径向支撑 |
| `ring_chord_frame` | `ring_chord_frame` | 闭合内环梁 + 环内弦杆/斜杆 + 径向支撑 |

候选指纹覆盖转接构件几何、构件角色、节点和传力路径。体系差异不再只依赖支撑间距。

### 4.2 候选完整计算

候选重建、候选采用和完整比选均显式传递 `transferSystemTemplate`。这修复了候选预览为汇交框架、完整计算却退回默认闭合环梁的状态错误。

完整计算后以以下指标重新排序：

- 最大墙体位移；
- 最大支撑轴力；
- 最大围檩/转接梁弯矩；
- 平面框架位移和残差；
- 支撑、立柱与转接梁数量；
- 施工与材料代理指标；
- Pareto 层级。

## 5. V3.70.0：自动深化证据与正式发行

### 5.1 自动深化包

新增 `concave_transfer_detailing.py`，输出：

- 转接梁构件表；
- 各梁控制 N/M/V；
- 构件设计与配筋完整性；
- 节点承压板和附加钢筋状态；
- 施工阶段闭合状态；
- 当前拓扑哈希；
- 仍需人工复核的边界。

自动深化包用于形成可审查证据，不替代注册结构工程师审签。

### 5.2 拓扑追溯

拓扑哈希覆盖：

- 支撑与转接梁构件角色；
- 节点和传力路径 ID；
- 构件坐标；
- 截面；
- 层号与标高。

计算状态等可变结果不进入拓扑哈希。构件几何或截面变化会使旧审批失效；重新计算本身不会无故改变哈希。

### 5.3 正式门禁

正式发行至少要求：

1. 平面框架施工阶段包络通过；
2. 转接梁设计和配筋证据完整；
3. 节点、安装、换撑和拆撑复核完成；
4. 复核人和证据引用已填写；
5. 审批拓扑哈希与当前模型一致；
6. 项目地质、计算保证和其他正式交付门禁同时通过。

## 6. 前端改进

候选面板新增：

- 闭合环梁、汇交框架、弦杆框架的差异化图例；
- 支撑、立柱、转接构件数量；
- 几何闭合、传力闭合、平面模型、施工阶段和正式出图五级状态；
- 完整比选中的墙体位移、平面框架位移、转接梁弯矩和阶段求解状态；
- 旧版审计数据的兼容显示。

## 7. 评估结果

标准 L 形、16 m 深、三道支撑的合成算例结果如下：

| 指标 | V3.66 旧分区直撑 | V3.70 闭合环 | V3.70 汇交框架 | V3.70 弦杆框架 |
|---|---:|---:|---:|---:|
| 支撑数 | 36 | 72 | 72 | 72 |
| 立柱数 | 14 | 19 | 22 | 22 |
| 转接构件数 | 0 | 18 | 30 | 24 |
| 拓扑 Fail | 9 | 0 | 0 | 0 |
| 几何闭合 | 否 | 是 | 是 | 是 |
| 平面模型可解 | 否 | 是 | 是 | 是 |
| 完整施工阶段框架 | 否 | 通过 | 通过 | 通过 |
| 自动深化证据 | 无 | 通过 | 通过 | 通过 |

完整 A/B/C 计算耗时约 15.45 s；候选生成约 1.15 s。平面框架最大相对残差为 \(4.98\times10^{-15}\)。该数值只说明线性方程求解平衡良好，不代表力学模型已通过外部软件验证。

L/U/T/Z/H 五类标准正交凹形样例的三种拓扑均实现：

- 拓扑 Fail 为 0；
- 墙面覆盖完整；
- 名义平面模型可解；
- `structuralModelClosed=true`。

合成算例未提供钻孔和地层资料，且未录入专业审批，因此三套方案的正式发行门禁均保持阻断。

## 9. 后续优先级

V3.70 之后仍建议按以下顺序继续：

1. 墙—围檩—平面框架反力迭代耦合；
2. 与成熟结构软件建立 10–20 个基准算例；
3. 临时分隔墙、中心岛和留土台真实结构体系；
4. 三维偏心、扭转、刚域和半刚性节点；
5. 转接节点局部承压、加腋、锚固和抗扭钢筋自动深化；
6. 永久柱网、坡道、出土通道和施工阶段联合优化。
