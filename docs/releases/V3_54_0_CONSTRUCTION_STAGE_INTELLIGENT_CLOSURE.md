# PitGuard V3.54.0 施工阶段与智能设计闭环

## 交互主线

1. 在“设计基准”录入坑外设计水位、坑内控制水位和坑边允许超载。
2. 在“工程输入”确认坑顶、坑底标高；最终开挖深度由此唯一确定。
3. 在“围护方案”生成并采用墙、围檩、支撑和立柱。
4. 进入“计算验算 → 施工阶段方案”，系统根据最终开挖深度和支撑层位生成中文推荐方案。
5. 设计人员逐阶段修改开挖标高、激活/退出支撑、水位、超载和换撑条件，通过校验后确认并锁定。
6. 完整计算仅使用已确认方案；计算期间不会静默重生成施工阶段。

## 安全校核与设计反馈

完整计算同时判定规范硬失败和项目安全储备不足。若启用自动闭环，系统会将控制项映射到具体墙体、围檩或支撑，执行有边界的截面补强并重新计算。每轮保留变更对象、原因、补强前后截面和复算结论，达到最大轮次仍未闭合时转为明确人工处置，不会把超限项包装为通过。

墙体嵌固长度支持三种模式：

- 统一墙趾：施工简单，适合地质和环境差异小的工程；
- 分区墙趾：默认推荐，按控制分区平衡安全和经济性；
- 逐墙墙趾：允许不同墙对象采用不同设计长度，同时要求更严格的专项复核和施工交底。

## “待重算”的可操作含义

| 界面结论 | 真实原因 | 处理方法 |
| --- | --- | --- |
| 缺资料 | 计算必需输入不存在 | 按展开清单到指定页面补齐；界面同时说明责任方和是否可在设计阶段提供 |
| 待重算 | 输入已具备，但当前设计快照没有对应追溯记录 | 先确认施工阶段，再点击“计算当前方案”；配筋改变截面时由系统自动复算 |
| 结果未载入 | 阶段成果已分块保存，但当前读取链路未回载 | V3.54 已统一修复配筋、P3 和交付入口；若成果文件真实缺失，需重新计算 |
| 已过期 | 地质、开挖、支撑、墙趾、截面或施工阶段在计算后发生变更 | 界面显示实际变更原因和影响范围，重新确认工况并计算 |
| 专项复核 | 已有自动证据，但超出软件自动结论边界 | 由相应专业设计、校审或专家论证形成正式结论 |

## P3 与交付入口

计算失效、施工阶段证据不完整、硬承载力不足或支撑拓扑不闭合仍阻断 P3。锚固、搭接、节点拥挤、预埋件和碰撞属于 P3 本身要解决的工作，因此作为带入深化的复核项，不再形成循环阻断。完成 P3 后，计算书、IFC、CAD、正式图纸包和联合交付包均从同一最新计算结果回载完整阶段证据。

## 验证范围

V3.54 专项端到端用例已实际执行：建立工程、导入地质、建立基坑与围护体系、生成并确认施工阶段、完整计算、安全补强闭环、配筋写入与复算、P3 深化，以及审查版联合交付包生成。
