# PitGuard V3.87.6：统一计算、优化与闭合

## 1. 本轮问题定位

对用户提供的运行日志进行复核后，确认存在三类相互叠加的问题。

### 1.1 设计控制工况引用旧支撑 ID

支撑拓扑重建后，设计控制工况仍保留旧构件 ID。日志中连续出现 52、104、156 根无效必需支撑，正好对应逐层累计的支撑数量。计算在进入优化器前即被工况校验阻断。

V3.87.6 对常规开挖、支撑安装、底板和最终阶段按开挖标高重新映射当前支撑；换撑、拆撑阶段涉及专项传力路径，继续要求人工确认。

### 1.2 原搜索空间只有截面增强

V3.87.5 的四个候选主要调整墙厚、梁截面和支撑截面。用户项目的最大围护墙位移约 90.866 mm，连续截面调整后仍保持相同位移，说明控制因素已经进入体系刚度和施工阶段层面。

V3.87.6 将搜索扩展为七类候选：

1. 当前体系闭环；
2. 经济分区增强；
3. 整体刚度优先；
4. 控制截面增强；
5. 平面支撑加密；
6. 增设控制支撑层；
7. 支撑层与截面联合增强。

新增支撑层后会同步重建围檩、支撑、立柱、节点和施工阶段支撑引用，再运行完整阶段计算。

### 1.3 进度显示缺少单调约束和候选内部阶段

旧任务仅在候选完成时更新进度，单个候选计算期间长期停留；页面恢复任务或接收旧轮询响应时还可能出现视觉回退。

V3.87.6 增加：

- 后端进度单调递增约束；
- 候选准备、拓扑重建、工况同步、求解完成等子阶段进度；
- 前端同一任务进度取历史最大值；
- 同一页面只允许一个计算任务提交；
- 按钮在提交和计算期间持续禁用。

## 2. 单一入口

计算页仅保留一个主按钮：

`一键计算、优化并闭合`

该任务依次执行：

1. 设计资格检查；
2. 安全的几何、地质覆盖和支撑拓扑修复；
3. 常规设计控制工况支撑引用修复；
4. 七类体系—截面候选计算；
5. 多目标排序和方案采用；
6. 正式复算；
7. 配筋刷新及必要的附加复算；
8. 输出闭合结论。

原“修复阻断并复算”“一键优化并复算”“计算当前方案”和逐条“按当前值递增并复算”按钮已从主界面移除。剩余措施作为诊断信息展示，自动措施统一纳入搜索。

## 3. 闭合结论

系统只输出以下三种结论：

- `closed`：当前方案满足计算闸门；
- `cannot_close`：在当前允许的体系和截面搜索边界内仍无法闭合；
- `needs_manual_input`：换撑、拆撑、测量坐标或专项资料需要人工确认。

无闭合候选时，界面明确显示“在当前自动优化边界内无法计算闭合”，并列出剩余控制项，不再以“最优方案已应用”替代闭合结论。

## 4. 工程边界

自动搜索允许调整：

- 围护墙、围檩和支撑截面；
- 平面支撑间距；
- 最大层数约束内增设一道控制支撑层；
- 常规开挖阶段的支撑激活集合。

自动搜索不修改：

- 土层参数；
- 地下水位；
- 荷载和安全目标；
- 测量坐标；
- 换撑、拆撑专项传力路径。
