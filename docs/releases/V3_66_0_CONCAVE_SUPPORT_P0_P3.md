# PitGuard V3.66.0：异形基坑水平支撑 P0–P3 闭环迭代

## 1. 迭代目标

V3.66.0 针对 L、U、T、Z、H 等正交凹形基坑中出现的“轮廓可识别、直撑可生成、转角传力路径不闭合、候选全部拓扑失败”问题，重构候选来源契约、异形转接体系、方案比选门禁、正式交付审批和前端诊断表达。

本次迭代将四个阶段划分为：

- P0：消除旧候选与当前轮廓混用，统一正式候选、诊断试案和完整比选资格；
- P1：为正交凹形平面生成闭合内环梁与墙—环径向支撑，形成可进入施工阶段计算的连续传力路径；
- P2：建立分区—墙面—凹角—转接环的拓扑图和三种真实几何候选；
- P3：将转接构件写入拓扑哈希，建立节点深化证据、专业复核人和当前拓扑绑定的正式出图闸门。

## 2. P0：候选来源与 UI 状态一致性

### 2.1 候选来源哈希

新增 `support_candidate_contract.py`。候选来源哈希覆盖：

- 基坑轮廓、坑顶/坑底标高、局部坑和障碍物；
- 围护墙分段、厚度、标高和材料；
- 单位制、坐标系和设计设置；
- 算法版本和规则集版本。

每个候选保存 `candidateSourceHash`。采用、完整比选和候选复用均要求候选哈希与当前项目一致。

### 2.2 失效与归档

轮廓、标高、墙体、单位制、坐标系或设计设置变化后：

1. 当前候选从工作区移除；
2. 候选摘要写入 `staleSupportCandidateArchive`；
3. 完整计算比较结果清空；
4. UI 状态切换为 `not_generated`；
5. 旧候选不能采用或进入完整比选。

### 2.3 正式候选和诊断试案分离

候选集增加：

- `candidateState`；
- `formalCandidateCount`；
- `controlledCandidateCount`；
- `staleCandidateCount`；
- `comparisonEligibility`。

完整比选至少需要两个来源有效且通过硬约束的正式候选。受控阻断候选在 UI 中显示为“诊断试案”，采用和完整计算按钮保持禁用。

### 2.4 UI 调整

方案面板完成以下改造：

- 原始英文失效原因和形状枚举本地化；
- 正式方案与诊断试案采用不同标题、按钮和状态；
- 失败根因按类别显示；
- 候选数量为 1、2、3 时自适应布局；
- 预览显示转接区和转接梁；
- 显示“计算资格”和“正式出图资格”两个独立状态；
- 顶部完整比选按钮与面板内部使用同一资格判定。

## 3. P1：正交凹形转接体系

### 3.1 新结构模板

新增三类转接模板：

- `compact_elbow_ring`：紧凑型异形闭合内环梁；
- `balanced_elbow_ring`：均衡型异形闭合内环梁；
- `extended_elbow_ring`：延伸型异形闭合内环梁。

系统对基坑多边形执行内偏置，选择可保持单一连通域的闭合内环。若内偏置分裂、净空不足或多边形无效，方案继续保持受控阻断。

### 3.2 墙—环径向支撑

每道支撑层均执行：

1. 对每个真实围护墙面按目标分仓生成支承点；
2. 将支承点连接至闭合内环；
3. 支撑标记为 `ring_strut` 和 `wall_to_ring`；
4. 禁止支撑止于另一支撑跨中；
5. 对每个墙面、每个支撑层检查覆盖完整性。

### 3.3 闭环审计

闭环审计不再依赖“环梁段数不少于 4”的弱判据。当前判据要求每层：

- 环梁图只有一个连通分量；
- 每个环梁节点度数为 2；
- 梁段数等于节点数；
- 每个真实墙面存在径向支撑；
- 每个凹角纳入转接区控制。

审计结果写入 `layoutSummary.transferSystem`。

## 4. P2：通用拓扑中间模型与 A/B/C 搜索

### 4.1 支撑分区图

新增 `support-zone-graph-v1`，节点包括：

- 外围围檩边界；
- 闭合转接内环；
- 每个围护墙面；
- 每个凹角交汇节点。

边包括：

- 墙面到转接环的径向压杆路径；
- 凹角到转接环的转接控制关系；
- 各墙面在不同支撑层的覆盖记录。

该图为后续中心岛、分隔墙、显式平面框架和施工分区扩展提供统一中间层。

### 4.2 候选变量

异形候选搜索新增：

- 转接模板；
- 环梁偏置尺度；
- 支撑分仓；
- 立柱服务跨；
- 原有位置扰动和锁定约束。

候选几何指纹和结构签名纳入转接梁数量、模板和几何。A/B/C 因此具有真实结构差异。

### 4.3 计算比较门禁

`run_candidate_comparison_for_project` 仅接收：

- 来源哈希有效；
- 硬约束通过；
- `formalSchemeEligible=true`；
- 正式候选数量不少于 2。

旧候选或诊断试案会在启动密集计算前被拒绝。

## 5. P3：计算追溯与正式交付

### 5.1 拓扑哈希

支撑拓扑哈希增加：

- 支撑角色和传力路径类别；
- 支撑端点、层号、标高和截面；
- 环梁/转接梁轴线、截面和标高；
- 转接体系模板、闭环、墙面覆盖和计算资格。

任一转接梁坐标变化都会使旧计算合同和旧深化审批失效。

### 5.2 深化审批证据

新增异形转接深化审批服务和 API：

- `GET /api/projects/{project_id}/design/concave-transfer-detailing`
- `PUT /api/projects/{project_id}/design/concave-transfer-detailing`

正式解除转接节点出图阻断需要：

1. 闭合环梁二维/三维轴力—弯矩—剪力—扭矩复核通过；
2. 支撑—环梁—围檩节点半刚性、承压、锚固及附加钢筋深化通过；
3. 安装、开挖、换撑和拆撑阶段复核通过；
4. 填写专业复核人；
5. 审批记录的拓扑哈希与当前体系一致。

拓扑变化后，既有审批自动失效。

### 5.3 交付门禁

异形转接体系具有两个独立状态：

- `calculationReady`：允许进入施工阶段完整计算；
- `officialIssueReady`：满足证据与审签后才允许正式出图。

系统不会因几何闭环自动宣称达到施工图发行条件。

## 6. 量化评估

评估样例为 60 m × 45 m 外包尺寸、开挖深度 16 m 的 L 形基坑，三道支撑层。

| 指标 | V3.65.1 原分区直撑 | V3.66.0 均衡闭合内环方案 |
|---|---:|---:|
| 支撑数量 | 36 | 72 |
| 立柱数量 | 14 | 1 |
| 转接梁段数 | 0 | 18 |
| 支撑质量状态 | fail | warning |
| 质量评分 | 0 | 84 |
| 拓扑 fail 数 | 9 | 0 |
| 围檩支点间距 fail | 9 | 0 |
| 非法交叉 | 0 | 0 |
| 越界支撑 | 0 | 0 |
| 无支承内部端点 | 0 | 0 |
| 每层墙面覆盖 | 0/6 | 6/6 |
| 环梁闭合 | 否 | 三层全部闭合 |
| 完整计算资格 | 否 | 是 |
| 直接正式出图 | 否 | 否，等待节点深化与审签 |

三种模板均生成硬约束通过的正式候选：

| 排名 | 模板 | 得分 | 支撑 | 转接梁段 | Fail / Warning |
|---|---|---:|---:|---:|---:|
| 1 | 紧凑型闭合内环 | 49.24 | 72 | 18 | 0 / 2 |
| 2 | 均衡型闭合内环 | 47.96 | 72 | 18 | 0 / 2 |
| 3 | 延伸型闭合内环 | 46.38 | 72 | 18 | 0 / 2 |

均衡方案完成 42 个施工阶段计算，耗时约 3.97 s。样例未输入钻孔和地层，因此结果中保留 2 个地质资料缺失 fail；该 fail 与本次支撑拓扑闭环无关。正式出图仍被 `shape_transfer_detailing` 阻断，符合 P3 安全边界。

完整原始评估数据见 `V3_66_0_CONCAVE_SUPPORT_EVALUATION.json`，可通过以下命令复现：

```bash
python scripts/evaluate-v366-concave-support.py --full-calculation
```

## 7. 当前边界

V3.66.0 已完成正交凹形平面的闭合环梁—径向支撑计算代理和正式交付证据门禁。以下能力仍需后续深化：

- 任意斜交凹多边形的通用骨架分区；
- 中心岛、临时分隔墙和刚接双向框架自动生成；
- 环梁扭转、节点半刚性和偏心刚臂的统一高保真平面/空间框架求解；
- 转接节点钢筋三维构造自动生成；
- 带坡道、出土口、永久柱网和复杂施工分区的联合优化。

因此，本版本显著扩大了异形基坑自动设计覆盖范围，同时继续保留专业复核和正式交付闸门。
