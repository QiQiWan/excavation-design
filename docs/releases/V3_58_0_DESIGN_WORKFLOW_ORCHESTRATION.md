# PitGuard V3.58.0 设计—校核—配筋统一编排

## 1. 问题结论

旧流程把“对象已经生成”误当作“阶段已经完成”，并把施工阶段藏在计算内部，形成四类断点：

1. 系统能生成推荐施工阶段，但未修改时保存按钮不可用，推荐值无法成为正式计算合同；
2. 计算完成只代表存在结果，不代表结果对应当前方案、当前施工顺序且全部定量校核闭合；
3. 配筋草案、正式构件设计、配筋写回、截面回算和 P3 深化共用一个“配筋完成”状态；
4. 无直接支撑端点的墙面围檩缺少构件弯剪包络，24 根同类围檩会被误报为没有正式设计结果。

## 2. 新的七阶段工程合同

| 阶段 | 系统自动完成 | 工程师确认 | 完成证据 |
|---|---|---|---|
| 设计基准 | 汇总规范、材料、荷载组合 | 采用值与来源 | 基准已确认且可追溯 |
| 工程模型 | 校验几何、地勘和水位 | 最终开挖标高与资料来源 | 闭合轮廓及地层/钻孔存在 |
| 方案定型 | 生成并比较候选 | 采用方案及经济性取舍 | 正式墙、梁、支撑拓扑存在 |
| 施工路径 | 按坑深和支撑标高生成建议 | 修改并锁定实际开挖/换撑顺序 | 已保存且阶段校验通过 |
| 分析校核与设计迭代 | 计算、定位控制构件、有限补强、复算 | 超出自动边界时选择方案 | 当前计算合同有效且硬失败为 0 |
| 正式配筋与复算 | 恢复梁证据、生成五类配筋、截面回算 | 构造和专项设计复核 | 全部构件有正式设计/配筋记录 |
| P3 深化与成果交付 | 节点、逐根钢筋、碰撞和成果组装 | 注册工程师签审 | P3 与发行门禁通过 |

## 3. 自动闭环边界

自动化允许调整墙厚、墙长、围檩或支撑截面以及配筋，但必须保留已采用拓扑和锁定施工顺序，不得降低荷载、弱化地层参数或绕过专项复核。计算补强和配筋回算都有最大轮次；超过边界后任务以 `needs_intervention` 正常结束，并返回控制对象、原因、可选方案和完成标准。

## 4. 关键修复

- 新增统一 `designWorkflow` 状态合同和 `design_workflow_closure` 编排任务；
- 旧 `core_design` 在已有采用方案时转入统一闭环，除非显式要求重新生成方案；
- 未修改的系统推荐施工阶段也可直接采用并锁定；未确认阶段不能进入正式计算；
- 配筋入口自动恢复过期计算合同，并按冠梁、围檩和环梁分组补算缺失证据；
- 截面改变后自动回到同一施工路径复算，再重新定筋；
- 无直接支撑墙面围檩按构件跨形成弯矩、剪力和挠度包络，但不伪造支撑轴力；
- 待重算和配筋阻断统一显示“为什么、怎么处理、完成标准”和一键恢复动作；
- 首页、登录页及版本清单统一显示 V3.58.0。

## 5. 验证范围

- 七阶段顺序、主动作和人机边界单元测试；
- 推荐施工阶段可直接采用交互测试；
- 围檩转角传力有构件证据且不产生虚假支撑反力测试；
- V3.52、V3.53、V3.55 计算与智能设计闭环回归；
- Web TypeScript 构建与核心工作台、施工阶段、计算恢复、配筋面板测试。

本软件用于方案设计和技术复核辅助；自动闭环不能替代项目负责人、专项设计人和注册工程师的最终判断与签审。
