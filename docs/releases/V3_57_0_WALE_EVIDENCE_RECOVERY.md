# PitGuard V3.57.0：连续围檩证据与配筋恢复

## 现场现象

完整计算显示 `Fail 0`，进入配筋深化后仍固定提示 24 根围檩缺少正式设计结果。自动重算能够完成，但每次仍返回相同的 `WB-L1/L2/L3-S2、S4、S6……` 对象，配筋方案因任务失败无法写入。

## 根因

这 24 根梁由 8 个没有直接支撑端点的墙段和 3 个支撑层组成。原支撑轴力函数在墙段直接支撑列表为空时提前返回，未生成该墙段围檩内力；计算引擎随后又按墙段本地支撑层过滤结果，进一步删除了全周连续围檩在这些墙段上的证据。梁几何存在，但施工阶段内力、截面设计和配筋写回链条中断。

## 修复

- 全周激活层与墙段直接支点分开处理：前者决定围檩是否存在，后者只决定是否产生支撑反力。
- 无直接支点墙段采用闭合周边围檩两端转角传力模型，形成保守的 `M+=qL²/8`、`M-=qL²/12`、`V=qL/2` 与挠度包络。
- 模型不创建虚构支撑反力；无直接支点长度超过项目硬限值时仍产生真实硬失败。
- 每次完整计算先清除旧梁设计结果，再从当前施工阶段重新建立，避免旧拓扑结果掩盖缺失。
- 配筋任务自动刷新计算合同、补齐缺失梁、写入正式配筋并重新执行结构/P3 入口校核。
- 界面按“为什么阻断—工程师怎么处理—处理完成后”展示，并提供一键自动恢复。

## 完成标准

正式配筋写回后，缺失梁数量为 0；每根冠梁/围檩具有施工阶段内力包络、截面设计结果和纵筋、箍筋、侧面构造筋、拉结筋、节点附加筋五类记录。真实承载力、支撑间距或地基稳定性失败仍保持阻断。

## 验证

- 上传日志同形拓扑回放：`S2/S4/S6/S8/S12/S14/S16/S18 × 3`，24 根围檩均形成正式设计结果；
- 反力守恒边界：上述转角传力段不增加任何虚构支撑轴力记录；
- 超长无直接支点围檩：超过项目硬限值时设计结果与规则检查均为阻断；
- 自动恢复任务：重新计算后缺失梁由 24 归零，配筋方案写入并返回恢复摘要；
- 水平支撑图示：默认“仅问题项”下仍显示通过构件的端部/跨中箍筋、侧面构造筋、拉结筋和搭接附加筋。
