# Changelog

## 3.81.0

- 将设计控制工况、施工计划阶段和现场执行快照拆分为独立领域对象，后续业务不得覆盖设计对象；
- 新增设计发行、施工准备、现场阶段放行三套独立门禁和就绪度；
- 设计发行门禁移除专项施工方案、专家论证、现场验收、实际开挖和监测实测值等未来信息；
- 旧施工阶段可幂等迁移为设计控制工况，并同步生成锁定的数值计算案例；
- 新增基准及八类不利设计情景，正式包络只使用真实完成的计算结果；
- 新增施工计划 A–E 符合性分级，超出设计允许域时触发设计复核、复算或变更；
- 新增现场执行快照、偏差事件、停工建议、复算要求和设计回复要求；
- 新增业务责任与阶段门禁前端面板、施工计划/现场快照录入和偏差台账；
- 在线文档新增责任边界章节，版本路线扩展至 V3.81；
- 新增 V3.81 专项回归和业务评估脚本。
- 设计控制工况审批元数据与数值输入解耦：仅批准/冻结不再使现有计算失效，控制边界变化才触发计算及情景包络失效。

## 3.80.0

- 新增施工计划阶段与设计控制工况绑定、允许域比对和 A–E 分级。

## 3.79.0

- 新增设计包络情景生成、正式情景复算就绪台账和控制情景追溯。

## 3.78.0

- 将施工阶段语义重构为设计控制工况，隔离施工计划日期和现场实测数据。

## 3.77.1

- 热修设计发行门禁：施工期证据不再阻断设计成果首次发行。

## 3.77.0

- 新增 L0–L3 分析等级、参数来源、默认/代理/回退识别和正式模式门禁。
- 新增地基反力溯源、位移动员非线性诊断及土体刚度/水位敏感性。
- 新增全局六自由度空间杆系验证和钢管支撑稳定曲线。
- 新增独立参考、空间梁核验及 OpenSees 外部基准验证矩阵。
- 新增危大工程分类、专项方案、专家论证、监测、验收、审签和档案证据链。
- 更新 `/api/documentation` 和前端在线文档。

## 3.72.0

- 将同步计算改为选择性事务工作集，计算失败时回滚围护体系、施工工况和工程证据，避免半成品状态污染项目；
- 新增六阶段执行台账、阶段耗时、瓶颈识别、工程状态与交付状态分离，以及可持久化运行时事件；
- 墙—围檩—转接框架反力迭代增加自适应松弛、绝对/相对残差、两周期振荡和停滞诊断；
- 新增数值健康包，统一汇总条件数、平衡残差、回退、病态阻断、敏感性和反力迭代；
- 新增施工阶段矩阵、关键阶段排序、墙/支撑/围檩/转接梁/立柱包络、节点热点和钢筋清单；
- 新增阻断、预警和人工复核根因台账以及按规则聚合的状态计数；
- 区分结果完整度、工程就绪度和正式交付就绪度，关键结果域失败不能被平均分抵消；
- 修复稳定指标语义混用：安全系数、风险比值和质量指数分别按工程方向统计，分层渗流风险不再被误报为最小安全系数；
- 修复支撑构件稳定被误归入岩土稳定的问题，并将强度、变形、岩土稳定作为独立结果域；
- 外部 OpenSeesPy 不可用时不再伪报通过；新增三项独立 SciPy 参考模型，并按结构内核版本保留既有外部基准证书；
- 前端新增流程阶段条、工程/交付状态、完整度/就绪度、数值健康、关键阶段和阻断根因显示。

- Core 候选优化限制为最多 12 组试算、800 个支撑构件和有界候选对象池。
