# PitGuard V3.28.0 异形基坑识别与水平支撑体系深度迭代报告

## 1. 迭代目标

本版本将异形基坑的水平支撑设计从“统一扫描线+局部补撑”升级为“轮廓识别—局部坐标—工程分区—体系选型—专用生成器—传力门禁—分阶段计算—受控交付”的完整工作流。

核心原则如下：

1. 支撑体系先由平面形状和施工分区决定，几何补丁不能代替结构体系。
2. 普通水平支撑按轴压杆件处理时，两端必须落在围护墙、围檩、环梁或具有明确计算模型的节点上。
3. 凹形、多臂和错台交汇区必须具有独立的转接体系，例如闭合环梁、分隔墙、中心岛或显式平面框架。
4. 非法穿越、支撑中部终止、墙上节点拥挤和无支承围檩跨均属于生成阶段硬约束。
5. 当前求解器能力不足时输出受控阻断，不通过任意斜撑或虚构节点把方案强行标记为可计算。

## 2. 常见基坑平面轮廓分类

### 2.1 规则与准规则凸平面

| 轮廓 | 典型特征 | 推荐体系 | 关键控制 |
|---|---|---|---|
| 狭长矩形 | 长宽比通常大于约2.2 | 短跨直对撑+端部平行角撑 | 端墙围檩跨、温度分区、预加轴力分区 |
| 一般矩形/旋转矩形 | 长短轴明确 | 局部主轴短跨对撑 | 不能依赖全局X/Y判断方向 |
| 近方形 | 两个方向跨度接近 | 闭合内环梁+径向支撑，或经专项计算的双向框架 | 环梁闭合、径向节点独立、出土空间 |
| 梯形 | 一组墙平行或近似平行 | 可见墙面对撑+端部局部斜撑/环梁 | 斜撑与墙面法向、端点独立、零扇形汇聚 |
| 平行四边形 | 局部主轴斜交 | 沿局部短跨的平行斜向对撑 | 支撑族平行、墙上链距排序 |
| 不规则四边形 | 墙面对关系不均匀 | 可见墙对候选+环撑比选 | 每根杆件的可见性和端点法向 |
| 三角形 | 尖角和三边汇聚 | 小型闭合内环+径向撑 | 禁止所有杆件汇聚单点，尖角围檩加强 |
| 一般凸多边形 | 无凹角，墙面方向较多 | 可见墙面对撑或多边形内环+径向撑 | 墙面对匹配、局部跨度和节点构造 |

### 2.2 竖井和曲线平面

| 轮廓 | 推荐体系 | 关键控制 |
|---|---|---|
| 圆形 | 环梁/环撑为主，必要时少量径向撑 | 环向连续、开口局部闭合、环向轴力和弯矩 |
| 椭圆形 | 相似椭圆/多边形内环+非均匀径向撑 | 长轴端和曲率变化区加密 |
| 正多边形竖井 | 多边形内环+边中径向撑 | 转角节点加强、各边围檩支点间距 |
| 带中心岛 | 外围围檩—径向撑—中心岛/内环 | 中心岛尺寸、施工通道、换撑顺序 |

### 2.3 正交凹形与分叉平面

| 轮廓 | 主要分区 | 推荐方案 | 必须设置的转接区 |
|---|---|---|---|
| L形 | 两个矩形走廊 | 两走廊分别短跨对撑 | 凹角肘部环梁、分隔墙或专项框架 |
| U形 | 两翼+底部横廊 | 三分区对撑+中心岛/横向闭合环 | 两个凹角转接区 |
| C形 | 上翼+下翼+腹板 | 三分区对撑，开口侧保留物流 | 腹板主传力带及开口端局部转接 |
| T形 | 横翼+竖翼 | 三臂分仓+中心转接环 | 三臂交汇区 |
| Z形 | 连续错台矩形区 | 分段短跨对撑 | 两处折转过渡框/局部环 |
| H形 | 两侧竖廊+中部横廊 | 多环或双中心岛 | 两个独立交汇转接区 |
| 单缺口/台阶形 | 主区+局部凹口 | 分区墙—墙支撑 | 缺口头部传力构件 |
| 梳齿/多台阶 | 多个狭长分支 | 多分区、分期施工 | 多个转接区和施工分隔 |

### 2.4 一般非正交凹多边形

采用可见性分解和凸子区划分。每个子区生成墙—墙支撑候选；凹角、分叉和几何方向突变位置建立明确的转接子结构。禁止使用任意角度射线补齐、跨凹口斜撑和切向擦墙连接。

### 2.5 组合基坑和局部坑

外轮廓、局部深坑、中心岛、坡道和出土口应分别建模。局部坑不会简单改变外轮廓类型，但会增加局部墙、局部支撑和施工阶段。系统将其标记为 `local_pits_require_stage_and_local_wall_check`，要求独立阶段和局部稳定复核。

## 3. V3.28.0形状识别算法

### 3.1 几何预处理

- 删除重复点和闭合重复点；
- 检查自交、零面积和多连通区域；
- 对诊断用途执行几何修复，并保留正式设计前清理警告；
- 计算面积、周长、凸包、最小旋转外接矩形和局部主轴。

### 3.2 识别特征

系统计算：

- 长跨、短跨和长宽比；
- 圆度；
- 矩形度；
- 凸度；
- 正交边长度比例；
- 边方向簇数量；
- 凹角数量；
- 外包矩形与基坑轮廓的缺口数量、位置和接触边；
- 局部深坑数量和中心岛状态。

### 3.3 精确原型识别

通过缺口签名和凹角拓扑识别 L/U/C/T/Z/H 形、单缺口、分叉、台阶、梳齿和一般凹形。凸平面进一步区分狭长矩形、近方形、梯形、平行四边形、三角形、圆形、椭圆形和多边形竖井。

### 3.4 工程分区

正交凹形轮廓按照局部坐标网格进行矩形分解，形成 `designZones`。每个分区保存：

- 分区编号；
- 局部边界和全局角点；
- 长度、宽度和面积；
- 推荐支撑方向；
- 推荐拓扑族。

## 4. 专用水平支撑生成器

### 4.1 直对撑生成器

适用于狭长矩形、矩形、旋转矩形和部分凸多边形。使用最小旋转外接矩形确定局部长短轴，沿长轴设置支撑站位，沿短跨连接真实墙面。

### 4.2 平行角撑族

端墙或凸角围檩跨超限时，以共同转角为基准，在两侧墙上使用对应链距生成平行角撑。禁止共用墙节点、V形扇撑和截断到其他支撑。

### 4.3 可见墙面对撑

适用于梯形、平行四边形和一般凸多边形。候选必须满足：

- 支撑全线位于基坑内部；
- 两端落在真实墙面；
- 与墙面法向夹角合理；
- 不穿越障碍和其他普通支撑；
- 不形成切向擦墙；
- 围檩支点间距满足硬限值。

### 4.4 内环径向体系

适用于近方形、圆形、椭圆形、多边形竖井及显式中心岛方案。外围各边按照围檩允许跨长布置径向节点，连接闭合内环。圆形和椭圆形使用相似缩放内环，紧凑凸平面使用局部轴矩形环。

### 4.5 异形分区 `zoned_direct` 生成器

本版本新增真实的分区生成器：

1. 读取识别出的矩形走廊/翼缘区；
2. 在每个分区的长方向按目标间距设置站位；
3. 沿分区短跨方向建立无限候选线；
4. 与实际基坑多边形求交；
5. 选择包含分区站位中心的有效线段；
6. 将端点落到真实围护墙；
7. 执行障碍、坑内包含、重复线和交叉检查；
8. 保存分区、里程、局部净跨、布置原因和传力类别。

任何支撑均不得终止在虚拟分区边界或另一根支撑上。

## 5. 异形转接区的工业门禁

分区支撑只能解决各子区的直接墙—墙传力。L/T/U/C/Z/H形的凹角和多臂交汇区还需要转接构件。系统新增：

- `shapeTransferSystemRequired`；
- `shapeTransferSystemComplete`；
- `shapeArchetype`；
- `shapePrimarySystem`。

当识别结果属于 `zoned_preliminary_then_controlled_check`，且没有闭合环梁、中心岛或明确转接体系时：

- 允许生成方案级分区支撑；
- 保持零非法交叉和两端落墙；
- `calculationReady=false`；
- A/B/C只保留一个诊断方案；
- 完整计算和施工版发行阻断；
- 界面给出应切换的体系和转接区要求。

## 6. 优化后的完整工作流

1. 导入或绘制闭合轮廓；
2. 几何清理和坐标归一化；
3. 识别形状原型、局部主轴、凹角和设计分区；
4. 根据形状选择支撑体系族；
5. 生成直撑、墙对斜撑、内环径向撑或分区支撑；
6. 检查端点、可见性、坑内包含、障碍、交叉、墙上节点和围檩跨；
7. 检查异形转接体系是否完整；
8. 生成多个真正不同的可行候选；
9. 按非法穿越、无效端点、墙上汇交、转接完整性、支撑数量和工程性能排序；
10. 对可计算候选执行施工阶段完整计算和独立复核；
11. 采用方案后生成节点、配筋、IFC、图纸和计算书；
12. 施工版发行前校验形状决策、计算合同和跨成果一致性。

## 8. 几何资格测试

资格套件扩展到15类：

- 160×33m长条形；
- 90×18m狭长形；
- 旋转矩形；
- 梯形；
- 凸六边形；
- L/T/U/C/Z/H形；
- 一般凹多边形；
- 近方形；
- 圆形；
- 椭圆形。

资格结果分为：

- `calculation_ready`：当前体系形成可计算传力闭环；
- `controlled_block`：拓扑安全、无虚假传力，但需切换或补充体系；
- `unsafe_failure`：存在非法交叉、坑外杆件、支撑中部终止或节点拥挤。

本版本15类资格算例全部达到 `calculation_ready` 或 `controlled_block`，不存在 `unsafe_failure`。

## 10. 当前能力边界

V3.28.0已完成形状识别、体系决策、专用几何生成和工业门禁。以下能力仍需后续结构求解升级：

- L/T/U/C/Z/H形转接环梁的自动尺寸设计；
- 分隔墙和中心岛的联合优化；
- 多环体系和环梁开口的空间分析；
- 支撑节点平面内弯剪刚度；
- 双向框架、桁架和空间框架求解；
- 多个独立外轮廓和真正带孔多边形；
- 曲线墙的原生圆弧/样条建模；
- 分区开挖施工路径的多目标优化；
- 与独立有限元软件的批量对标。

在这些能力完成前，系统会对相应轮廓输出明确的方案建议和受控阻断，不会将方案级几何误标为可施工成果。
