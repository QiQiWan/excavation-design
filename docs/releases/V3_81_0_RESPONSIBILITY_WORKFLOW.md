# PitGuard V3.77.1—V3.81 设计责任边界、施工计划与现场偏差闭环

## 1. 迭代目标

本轮解决原 V3.77 将设计阶段、施工准备和现场实施信息混入同一完整性门禁的问题。系统保留基坑设计软件的核心定位，设计人员只维护其可获得且依法应负责的数据；施工计划和现场实测通过后续协同对象接入。

## 2. 版本能力

### V3.77.1：三门禁热修

- `designIssueGate` 只检查设计基础资料、设计控制工况、当前计算、控制条件和设计校审；
- `constructionPreparationGate` 检查施工计划、专项方案、监理审查、专家论证、监测方案和应急预案；
- `fieldStageReleaseGate` 检查现场快照、设计允许域和阶段验收；
- 每个缺项记录责任主体、影响门禁和处理动作。

### V3.78：设计控制工况

新增 `DesignControlStage`。旧 `CalculationCase.stages` 可幂等迁移，迁移结果只表达设计计算边界。数值引擎仍使用 `CalculationCase`，但该对象由设计控制工况同步生成并锁定，施工计划日期和现场实测数据不进入计算合同。

### V3.79：设计包络情景

每个设计控制工况自动生成基准、延迟支撑、局部超挖、预加轴力不足/偏高、高水位、临时超载、支撑刚度折减和关键支撑异常九类情景。正式包络只读取真实完成的基准计算和正式情景复算，不用经验放大系数伪造构件结果。未完成情景明确列入 `pendingFormalScenarioCodes`。

### V3.80：施工计划符合性

新增 `ConstructionPlanStage`，绑定设计控制工况。系统比较计划开挖标高、支撑激活、预加轴力、水位和坑边荷载，输出 A—E 级符合性、是否在设计允许域内、是否需要设计复核、复算或禁止实施。

### V3.81：现场偏差闭环

新增 `FieldExecutionSnapshot` 和 `DeviationEvent`。实际开挖、支撑、水位、预加轴力和监测信息只影响现场阶段放行。超出设计允许域时生成关键偏差事件，记录影响工况、构件、设计限值、计划值、实测值、暂停建议、复算和设计回复要求。

## 3. 数据隔离原则

- 设计对象：可由设计人员维护和冻结；
- 施工对象：只引用设计控制工况，不能覆盖设计；
- 现场对象：追加式保存，不覆盖计划或设计；
- 所有对象拥有独立 ID、版本和时间；
- 设计变更生成新设计版本，旧施工计划和现场映射保留审计记录。

## 4. API

基础路径：`/api/projects/{projectId}/workflow`

- `GET /workflow`：三门禁和责任边界总览；
- `POST /migrate-legacy-stages`：旧工况迁移；
- `GET|PUT /design-control-stages`：设计控制工况；
- `POST /design-scenarios/generate`：生成设计情景；
- `PATCH /design-scenarios/approval`：批准、拒绝或停用设计情景；
- `POST /design-scenarios/execute`：在后台隔离 worker 中逐情景正式复算；
- `POST /design-scenarios/envelope`：构建真实结果包络；
- `GET|POST /construction-plan-stages`：施工计划；
- `GET /construction-plan-stages/{id}/compliance`：计划符合性；
- `GET|POST /field-snapshots`：现场快照；
- `GET|PATCH /deviations`：偏差事件台账。

## 5. 迁移

原施工阶段复制为设计控制工况，原数据保留。原正式发行状态重新按设计发行、施工准备和现场放行分别计算。历史项目不会因缺少尚未发生的现场数据降低设计发行就绪度。

## 6. 验收边界

当前版本已经完成业务对象、门禁、API、前端工作台、在线文档和专项回归。已批准的非基准情景可通过 `design_scenario_envelope` 后台重任务逐一隔离复算，完整结果外置保存，项目快照只保留摘要和包络。未执行、失败或因数量上限跳过的情景持续列入待办，不会被标记为已计算。

## 7. V3.81.0 最终业务门禁加固

- 设计控制工况在进入设计发行门禁前必须处于 `approved` 或 `frozen` 状态；
- 设计校审必须同时满足当前快照审批有效、职责分离有效和注册结构工程师资格/数字签名有效；
- 施工计划达到 C 级时必须完成增量复算，D/E 级必须调整计划或发起设计变更；
- 施工计划未批准时不能通过施工准备门禁；
- 现场快照只有 `verified` 状态可以作为阶段放行证据，临时数据仍可用于预警但不能放行；
- 设计情景支持显式批准/拒绝，包络服务兼容 V3.77 正式不利工况复算摘要和 V3.81 情景执行摘要；
- 包络只统计实际完成的结果，已批准但未复算的情景持续列入待办。

### 审批元数据与数值输入分离

批准或冻结设计控制工况属于业务审批动作。若开挖标高、支撑激活、水位、荷载、预加轴力范围和允许偏差等控制内容未变化，系统保留既有计算结果及已完成情景包络，不再触发无意义复算。只有数值或控制边界变化时，才同步锁定计算案例并失效相关计算与情景结果。
