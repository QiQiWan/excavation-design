# PitGuard V3.87.2 设计核心完整性与稳定性加固

## 1. 迭代定位

V3.87.2 是 V3.87.1 的完整性加固补丁。结构计算内核、规则集和导出模式保持 V3.87.0 口径；本轮处理前端装载、候选几何契约、参数正式资格、项目读取副作用、候选重复搜索、工作区证据压缩和运行诊断等跨层问题。

本轮审查输入包括：

- V3.87.1 源码；
- L 形基坑三种转接体系；
- 候选搜索、项目存储、任务生命周期、钢筋合同和 worker 内存诊断日志；
- 设计核心、V3.81 工作流、V3.77 合规、V3.70 转接框架及历史异形支撑回归测试。

## 3. 关键实现

### 3.1 候选预览契约 V3

候选预览现在同时保存：

- 普通支撑、转接梁、转接区、障碍物和立柱；
- 来源数量和实际渲染数量；
- 每一类对象的截断标志；
- 无效点和无效构件数量；
- 项目是否要求转接体系、转接几何是否实际存在；
- `complete / warning / incomplete` 完整性状态。

前端收到不完整预览时显示明确警告，不再以空数组或坐标 0 静默替代。

### 3.2 正式参数资格

参数用于正式设计必须同时满足：

1. 来源属于勘察、建设单位、规范、项目批准、企业标准或可追溯推导；
2. 存在来源引用；
3. 已由责任人确认；
4. 服务端允许用于正式设计；
5. 推导值的置信度达到中或高。

软件建议值、默认值、人工估算值仍可用于草案和敏感性分析，不能直接解除正式门禁。

### 3.3 候选物理去重

几何指纹基于实际传力模型：

- 普通支撑的每个线段与角色；
- 转接梁折线的每个子段与角色；
- 临时立柱位置；
- 坐标按统一容差量化。

同一物理拓扑在候选池建立前仅保留评分更优项，降低后续排序和存储负担。

### 3.4 有界工作区证据

工作区保留交互所需摘要：

- 候选完整计算的决策分、最大位移、最大轴力、Fail/Warning 和推荐原因；
- 计算阶段状态及耗时；
- 条件数、残差、回退和阻断计数；
- 结果域完整度；
- 关键阶段、阻断、警告和人工复核台账。

逐阶段矩阵、逐构件包络、节点数组和完整图表数据仍通过外部对象按需读取。

## 6. 保留边界

1. 本补丁没有改变结构计算内核和配筋算法公式；
2. 候选预览仍采用有界上限，超大型模型会显式标记截断；
3. 前置物理去重减少重复候选进入池，但试算几何本身仍需生成后才能计算指纹；
4. 早期项目中已经保存的不完整配筋结果不会被静默改写，应重新运行完整计算与配筋；
5. 多项目并发、100 次连续耐久和完整前端生产构建仍属于生产发布前强制验证项；
6. 当前发行资格保持 `engineering_preview`。
