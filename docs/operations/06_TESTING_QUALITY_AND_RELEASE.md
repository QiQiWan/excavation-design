# 测试、质量门禁与发布

## 1. 测试分层

后端测试分为公式与数据模型单元测试、几何属性测试、公开基准算例、API 集成测试、IFC/CAD/DOCX 导出测试和完整工作流测试。前端测试覆盖状态管理和核心交互，生产构建用于检查 TypeScript、资源分包与静态产物。

计算与绘图测试会加载 NumPy、Matplotlib、Shapely 和文档导出库。连续在同一 Python 进程中运行全部重型测试时，第三方原生库的内存驻留可能导致耗时明显增长。因此发布门禁提供两种模式：

- `bash scripts/test-backend.sh fast`：日常快速门禁，在单个解释器中覆盖 V3.2 配筋诊断/交互、V3.1 配筋/CAD、V3.0 几何/任务及代表性兼容接口。
- `bash scripts/test-backend.sh full-isolated`：逐测试节点启动独立进程，隔离原生库状态，用于正式发布。
- Windows 对应命令：`powershell -ExecutionPolicy Bypass -File scripts/test-backend.ps1 -Mode fast` 或 `-Mode full-isolated`。
- 前端：`bash scripts/test-frontend.sh`。
- 一体化：`bash scripts/test.sh fast`。

数值测试默认设置 `PITGUARD_NUMERIC_THREADS=1`，避免任务线程与 BLAS 线程形成嵌套并发。需要调整时应结合并发项目数和矩阵规模重新压测。

## 2. 发布门禁

发布前至少完成以下检查：

1. `python -m compileall -q app tests` 无错误。
2. V3.0 几何/任务集成测试和 V3.1 配筋/CAD 集成测试全部通过；重型计算、墙长闭环和方案台账按独立进程复核。
3. 项目建立、轮廓更新、完整计算、任务持久化、候选比选、几何一致性和主要导出路径通过。
4. 前端测试和生产构建通过。
5. 项目列表只返回摘要，响应体不包含完整计算历史。
6. 云图使用高程坐标，可切换阶段、包络和有符号/绝对值模式。
7. IFC、CAD、主模型和计算模型的墙段拓扑可通过几何哈希对账。
8. 交付制品记录文件大小和 SHA-256。
9. 正式计算记录软件、算法、规则集和导出模式版本。

## 3. 性能基线

V3.0.0 性能收束样本中，82 个项目的完整 JSON 列表约 136.02 MB；摘要列表约 30.76 KB，初始列表载荷降低约 99.98%。前端入口包由约 917 KB 降至约 11 KB，工作台和 Three.js 改为按需加载。Three.js 独立资源包仍约 524 KB，属于后续可继续压缩的专项。

典型项目在数值线程限制为 1 的情况下连续执行 6 次完整计算，总耗时约 24.3 秒，单次约 2.7—5.9 秒。该结果仅用于软件回归，不构成工程精度验证。

## 4. 交付包清理

交付包不得包含开发数据库、历史导出结果、`node_modules`、前端 `dist`、运行日志、`__pycache__`、`.pytest_cache`、`.pyc` 或 TypeScript 构建缓存。根目录仅保留 README、启动脚本、源码目录和必要工程配置；说明文档统一位于 `docs/`。

正式压缩前应生成文件清单和 SHA-256。解压后可通过当前版本 `docs/releases/V3_1_0_FILE_MANIFEST.sha256` 校验源码交付内容。

## 5. 工程验证边界

自动化测试证明软件行为与当前实现约定一致，无法替代以下验证：成熟有限元软件对标、规范条文逐条适用性审查、复杂地下水专项分析、施工阶段参数核定、现场监测反演和注册工程师审签。相关边界必须在计算书与交付台账中保留。
