# V1.8.0 P0-P5 综合工程化迭代说明

## P0 围檩设计继续工程化

V1.8 将 V1.7 的围檩本体设计扩展为多工况包络机制。`WaleBeamEnvelopeResult` 汇总同一围檩在不同施工阶段的正弯矩、负弯矩、剪力和挠度包络，`WaleBeamDesignResult` 增加挠度限值、截面优化历史、节点局部承压扩散尺寸和围檩-地连墙连接说明。

当前仍属于工程筛查级：截面设计采用矩形钢筋混凝土子集，后续需要进一步补裂缝、锚固、施工缝、预埋件、穿墙筋和节点详图。

## P1 支撑施工阶段装拆模型

支撑对象新增生命周期字段：预加轴力阶段、拆撑阶段、生命周期说明和预加协议状态。计算结果新增 `JGJ120-SUPPORT-LIFECYCLE-PATH-SUBSET`，与既有预加轴力、温度、间隙闭合和施工偏心效应共同形成支撑施工阶段筛查。

后续应将地下室底板/楼板作为换撑构件显式建模，并按楼板强度达到时间和拆撑顺序更新支撑边界条件。

## P2 CAD 编辑器工程图层

前端基坑编辑器新增 DXF 导出、支护轴线偏距、地下室外墙线偏距、图层保存和命令行扩展。偏距线当前采用基于轮廓质心的示意偏移，用于方案交互；生产级应改为多段线几何偏移和自交修复。

## P3 三维 Viewer 审查化

三维 Viewer 新增审查定位面板，列出 fail/warning/manual_review 项，点击后可在属性面板查看规则、对象、状态和说明。该能力与已有支撑轴力云图、节点颜色、剖切、测距、最大轴力支撑和最大弯矩墙段定位配合使用。

## P4 计算书图表数据

计算结果新增 `reportDiagramData`，包含围檩包络和墙体内力样本。DOCX 计算书新增 V1.8 摘要、墙-围檩-支撑耦合摘要和围檩包络表。下一步可用这些数据生成正式弯矩图、剪力图、挠度图和轴力云图图片。

## P5 核心计算扩展接口

阶段结果新增 `coupledSystemResult`，集中记录墙体弹性地基梁结果、围檩连续梁结果和支撑弹性反力结果。当前是耦合摘要接口，正式生产级应推进到统一矩阵求解：墙体竖向梁、围檩水平连续梁和支撑杆件共同组装全局刚度矩阵。

## 后续优先级

1. 生产级墙-围檩-支撑全局联立刚度模型。
2. 计算书图表自动绘制。
3. CAD 多段线布尔偏移与 DXF 图层语义识别。
4. 三维 Viewer 的构件选择联动和构件树。
5. 地下水专项、坑底隆起和整体稳定的更完整规范实现。
6. 监测反分析和施工阶段数字孪生接口。
