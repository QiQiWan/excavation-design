# V1.4.0 水平支撑拓扑化设计算法说明

## 1. 设计目标

V1.4.0 将原先“按外包矩形画几根支撑线”的简化逻辑，升级为面向工程平面拓扑的水平支撑自动布置算法。目标是让自动生成结果更接近常规基坑内支撑设计的平面风格：

1. 主对撑优先沿短跨方向布置，降低单根支撑计算长度。
2. 沿长向按目标间距分仓，避免支撑过密或过疏。
3. 凸直角区域自动生成角撑，改善角部约束和整体扭转刚度。
4. 凹形基坑通过“扫描线-多边形求交”拆分支撑，避免支撑穿过坑外空区。
5. 根据主对撑跨长自动生成临时立柱候选点。
6. 支撑对象保留 spanLength、baySpacing、supportRole、layoutNote 等属性，供 IFC、计算书和前端显示使用。
7. 支撑轴力估算采用同层 role-weighted split，避免角撑与主对撑获得完全相同轴力的早期原型错误。

## 2. 算法输入

输入对象为 `ExcavationModel`：

- `outline.points`：基坑平面闭合多边形。
- `topElevation`：坑顶标高。
- `bottomElevation`：坑底标高。
- `depth`：开挖深度。
- `support_elevations()`：根据开挖深度生成支撑道数和标高。

## 3. 主对撑方向识别

首先计算基坑轮廓外包范围：

```text
span_x = max_x - min_x
span_y = max_y - min_y
```

若 `span_x >= span_y`，认为基坑长向为 X 方向，主对撑沿 Y 方向跨越短边；否则主对撑沿 X 方向跨越短边。

该规则对应常规内支撑设计习惯：优先缩短支撑跨长，提高支撑刚度并降低轴压稳定风险。

## 4. 长向分仓

设目标主支撑间距为：

```text
TARGET_MAIN_SUPPORT_SPACING_M = 18.0 m
```

主对撑根数按下式初选：

```text
count = ceil(long_span / target_spacing) - 1
```

并限制在 `[1, 8]` 之间。随后沿长向均分生成扫描位置：

```text
coord_i = long_min + (i + 1) * long_span / (count + 1)
```

如果扫描线刚好穿过多边形顶点或边界，系统会自动微移坐标，避免几何求交时出现奇偶配对错误。

## 5. 凹形基坑的扫描线求交

对于每条扫描线，算法不再直接连接外包矩形两端，而是与真实基坑多边形求交。

若长向为 X，生成竖向扫描线 `x = coord`，计算其与多边形所有边的交点，得到一组 y 坐标，排序后按奇偶配对得到有效区间：

```text
[y1, y2], [y3, y4], ...
```

每个有效区间单独生成一根主对撑。这样对于 L 形、凹形、带缺口的基坑，支撑不会穿越坑外空区。

若有效区间长度小于 `MIN_MAIN_STRUT_SPAN_M = 5.0m`，自动过滤。

## 6. 凸直角角撑生成

角撑只在凸直角生成，不在凹角生成。

算法步骤：

1. 计算多边形有向面积，判断轮廓方向。
2. 遍历每个顶点，判断该点是否为凸角。
3. 计算相邻两边夹角，若角度位于 70°～115°，认为接近直角。
4. 当基坑长宽比大于 1.35 或长边大于 36m 时，启用角撑。
5. 在相邻两边上分别取偏移点，连接形成角撑。
6. 检查角撑 1/4、1/2、3/4 位置是否在基坑多边形内，若不在则不生成。

这样可以避免凹角、洞口和不规则缺口处错误跨越布撑。

## 7. 临时立柱布置

临时立柱由主对撑跨长控制。对于每根主对撑：

```text
n_columns = ceil(strut_span / COLUMN_MAX_UNBRACED_SPAN_M) - 1
```

其中：

```text
COLUMN_MAX_UNBRACED_SPAN_M = 18.0 m
```

若主对撑长 30m，则在跨中布置 1 个临时立柱；若主对撑长 40m 左右，则按三分点布置 2 个立柱。

不同支撑层同一平面位置的立柱自动去重，并在 `ColumnElement.supportCodes` 中记录其服务的支撑构件编号。

## 8. 支撑轴力分配

V1.4.0 将同层支撑轴力由“均分”改为按角色加权分配：

```text
main_strut      weight = 1.00
corner_diagonal weight = 0.35
manual          weight = 1.00
```

主对撑承担主要墙体侧向荷载，角撑承担角部约束和扭转控制作用。该处理仍是初步估算方法，但比所有同层支撑完全等轴力更符合工程直觉。

## 9. 输出字段

`SupportElement` 新增字段：

```text
spanLength      支撑几何跨长
baySpacing      主对撑沿长向分仓间距
startFaceCode   起点设计面编号，预留
endFaceCode     终点设计面编号，预留
layoutNote      布置逻辑说明
supportRole     main_strut / corner_diagonal / manual
```

`ColumnElement` 新增字段：

```text
supportCodes    该立柱服务的支撑编号列表
```

## 10. 当前边界

V1.4.0 已解决短跨布撑、角撑、凹形避空、立柱候选点和角色加权轴力分配。但仍属于方案级自动建议，后续还需要继续实现：

1. 环撑体系。
2. 支撑与地下室柱网、坡道、出土口避让。
3. 支撑节点局部承压和节点配筋。
4. 立柱桩设计。
5. 分区开挖与换撑路径。
6. 支撑预加轴力。
7. 更真实的二维/三维支撑体系刚度分析。
8. 按每个设计墙面建立支撑关联，而不是按全局同层支撑估算。
