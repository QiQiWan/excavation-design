# V2.0.7 支撑线约束优化器与操作员效率界面

## 目标

V2.0.7 将支撑优化从 V2.0.6 的“参数枚举候选比选”推进为“支撑线位置变量约束优化”。优化器不再只枚举目标分仓和立柱服务跨，而是将主支撑扫描线位置作为变量，尝试对支撑线进行对称外扩、对称内收和障碍逃逸微调，并对每个候选方案进行硬约束和软目标评分。

## 硬约束

- 支撑不得无节点交叉。
- 支撑不得穿越出土口、坡道、保护区等障碍。
- 支撑端点必须吸附到围檩、墙面或环梁节点。
- 临时立柱不得落入障碍区。
- 换撑路径不得中断。

硬约束不满足的候选方案仍会保留在诊断列表中，但排序时显著降权，不会优先作为写回方案。

## 软目标

- 支撑间距接近 3-6m。
- 支撑体系尽量对称。
- 支撑跨长尽量短。
- 立柱数量不过多。
- 轴力峰值代理项尽量低。
- 出土路径尽量连续。

## 输出

- 输出 3-5 个候选方案。
- 每个候选方案包含变量策略、线位调整记录、硬约束状态、软目标分项、IFC/计算书导出就绪状态。
- 前端结果页提供候选方案对比表和六项软目标条形“雷达式”对比。

## UI 优化

- 工作台顶部增加操作员摘要面板：流程进度、支撑布置评分、候选数量、出图闸门。
- 主流程增加“操作员下一步”提示条，避免用户在多按钮页面中迷失。
- 常用路径保留主按钮；高级命令继续收纳在侧拉框。
