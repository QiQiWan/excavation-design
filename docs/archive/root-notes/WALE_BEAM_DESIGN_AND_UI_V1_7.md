# V1.7.0 围檩本体设计与工程交互深化说明

## 1. 围檩本体设计

V1.7.0 在 V1.6.0 “围檩连续梁—弹性支撑节点反力”模型基础上继续输出围檩本体设计结果。每个墙面、每个支撑层的围檩被建模为一维 Euler-Bernoulli 连续梁，墙面压力带积分得到的线荷载作用在围檩上，支撑端部按 EA/L、法向投影和角色折减作为弹性支座。

新增数据对象：

```text
WaleBeamInternalForceResult
WaleBeamDesignResult
```

`WaleBeamInternalForceResult` 记录围檩里程点、剪力、弯矩、挠度、最大弯矩、最大剪力和最大挠度。`WaleBeamDesignResult` 记录设计弯矩、设计剪力、所需钢筋面积、实配钢筋面积、弯矩承载力、剪力承载力、主筋、箍筋和节点区附加筋协调说明。

## 2. 围檩截面和配筋

系统先按当前围檩截面计算正截面受弯和斜截面受剪子集；如果不能通过，会自动尝试工程常用大截面范围内的扩截面设计，并同步更新 BIM 中的围檩截面名称。该过程不删除校核项，也不把无法通过的工程伪装成通过；若达到自动范围仍不足，会以 warning 保留并要求专项复核。

配筋输出包括：

```text
围檩上/下缘主筋
围檩箍筋
节点区附加抗裂筋
```

节点区附加筋与支撑节点承压板、围檩主筋连续通过、U 形筋和加密箍筋协调。

## 3. 支撑施工效应

支撑设计轴力增加施工阶段效应：

```text
N_eff = N_wale + 0.5*N_preload + N_temperature + N_gap
M_e = N*e0
```

其中 `N_wale` 来自围檩节点反力，`N_preload` 为预加轴力，`N_temperature` 为温度约束筛查效应，`N_gap` 为节点间隙闭合效应，`M_e` 为施工偏心产生的附加弯矩提示。当前参数是方案阶段默认值，正式工程应按支撑施工方案、温度场、预加轴力记录和监测数据复核。

## 7. 后续边界

V1.7.0 仍属于工程辅助设计原型。后续应继续引入围檩连续梁的真实荷载组合、多跨支座非线性、支撑预加轴力施工闭环、温度时程、支撑拆换撑有限元工况、地下室楼板换撑刚度和监测反分析。
