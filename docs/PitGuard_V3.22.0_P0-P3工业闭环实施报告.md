# PitGuard V3.22.0：P0-P3 工业闭环实施与验证报告

## 1. 版本定位

V3.22.0 将原有设计辅助链路扩展为可计算、可检查、可追溯、可恢复、可监测的 P0-P3 软件闭环。该版本完成工业部署前的软件内部基线，不能替代具体工程的勘察资料、专项审查、第三方计算复核、设计单位质量体系或注册工程师签发。

## 2. P0：计算可信度、资格测试与正式闸门

### 2.1 通用几何资格测试

新增旋转矩形、梯形、L 形、U 形和近方形五类资格算例，自动检查：

- 支撑中心线位于基坑轮廓内；
- 同层非法平面穿越为零；
- 围檩支点间距和强度拓扑预检可闭合；
- 凹形和近方形场景仅在必要时启用受控 T/Y 框架节点；
- 一般矩形优先保持直撑体系，避免为形式多样性增加无效节点。

### 2.2 计算合同

计算结果绑定支撑拓扑哈希、算法版本和规则集版本。工程输入、支撑拓扑、版本恢复或监测反演参数发生变化后，旧计算自动失效并要求重新计算。正式闸门增加：

- 控制结果有限性检查；
- 当前拓扑与计算结果一致性检查；
- 公式、条文、方法、输入和结果路径追溯覆盖率检查；
- 计算、三维模型和图纸几何一致性检查；
- P0 通用几何资格测试结果检查。

### 2.3 无勘察资料的草案筛查边界

仅进行支撑交叉、越界和拓扑诊断时，系统允许创建带 `SCREENING-UNVERIFIED` 标识的保守单层土占位模型。该模式的地质覆盖检查保持失败状态，并设置 `screeningFallbackUsed=true`，因此不能通过正式计算、施工图或监测阈值闸门。录入真实钻孔和土层参数后必须重新计算。

## 3. P1：洁净支撑主目标与施工深化

### 3.1 支撑拓扑分层目标

候选方案按以下优先级排序：

1. 安全、稳定、端点、障碍物、围檩支点间距等硬约束；
2. 非法同层穿越数量；
3. 平面交叉复杂度；
4. 高分支拥挤节点数量；
5. 内部 T/Y/X 汇交节点数量；
6. 辅助杆件和支撑总量；
7. 轴力、跨长、对称性、立柱数量和施工通道等综合性能。

支撑修复结果同时输出两类评分：

- `scoreBefore/scoreAfter`：拓扑修复优先级评分，非法穿越具有最高优先级；
- `rawQualityScoreBefore/rawQualityScoreAfter`：原始工程质量评分，用于保留全部失败和警告信息。

该设计避免“成功消除非法交叉，但因新增低级警告导致修复评分下降”的误判，同时不会隐藏原始工程问题。

### 3.2 A/B/C 完整比选与深化阻断

工业闭环自动补齐排名前 3 个候选方案的独立完整计算，比较最大支撑轴力、位移、围檩内力、稳定性和拓扑复杂度。以下成果缺失时 P1 保持阻断：

- A/B/C 完整计算证据；
- 地连墙或排桩逐根钢筋几何；
- 钢筋笼分段、吊点、接头、套筒和 BBS；
- 围檩、支撑和节点配筋；
- 端板、承压板、加劲肋、锚栓和焊缝等节点硬件；
- 预埋件与钢筋碰撞筛查；
- 当前计算快照对应的深化成果。

## 4. P2：项目数据、任务、权限、备份与可观测性

### 4.1 项目修订与审计

SQLite 存储启用 WAL、忙等待和事务写入。每次保存形成不可变修订快照、SHA-256 内容哈希和审计事件。`expectedRevision` 实现乐观并发控制，过期写入返回 HTTP 409。系统支持修订查询、审计查询和指定版本恢复；恢复后旧计算自动失效。

### 4.2 后台任务

任务记录持久化原始载荷、尝试次数、父任务、心跳、日志和结果哈希。失败或服务中断任务可按原始输入重试，并保留重试链。服务启动时会把遗留的 queued/running 任务标记为 interrupted，避免显示为永久运行。

### 4.3 可选 API Key 与角色控制

配置 `PITGUARD_API_KEYS` 后自动启用 API Key RBAC，可通过 `X-PitGuard-Key` 或 Bearer Token 访问。角色包括 viewer、designer、checker、reviewer、approver 和 admin。只读访问、设计写入、审查和系统备份采用不同权限等级。未配置密钥时保留本地开发兼容模式，生产就绪检查会给出安全警告。

### 4.4 在线备份与系统就绪度

提供 SQLite 在线一致性备份，备份完成后执行 `PRAGMA integrity_check`、项目/修订/审计数量统计和 SHA-256 校验。系统指标输出 HTTP 状态码、请求延迟 p50/p95/p99、任务状态、任务心跳和备份配置。生产部署仍需增加异机备份、定期恢复演练、TLS、密钥轮换和数据库高可用。

## 5. P3：项目阈值、监测反馈与数字孪生快照

监测闭环支持墙体位移、地表沉降、支撑轴力和地下水位，统一量纲后按对象建立时序。系统计算变化速率、可配置时段趋势外推、计算值比值或偏差，并输出 normal、watch、warning、alarm、manual_review 五级状态。

项目可选择：

- 系统默认筛查阈值；
- 经项目监测方案确认的项目级阈值。

前端可配置墙体位移、沉降、支撑轴力比、水位偏差和趋势外推时长，并在监测控制台显示当前值、外推值、控制等级和建议动作。默认值明确标记为非强制筛查阈值。参数反演采用“预览—工程师批准—应用—旧结果失效—重新计算”的控制链。

## 6. 工业闭环执行路径

同步接口和后台任务均支持完整闭环：

1. 检查当前计算合同；
2. 对过期或缺失计算重新执行分阶段计算；
3. 补齐前三个候选方案的完整计算；
4. 刷新正式报告闸门与几何一致性；
5. 生成施工深化成果；
6. 运行通用几何资格测试；
7. 汇总 P0-P3 完成率、阻断项、警告和整改动作；
8. 将结果写入当前项目修订和审计链。

## 8. 验证结果

本轮完成的关键验证包括：

- 后端 P0-P3、洁净支撑、设计流程、钢筋笼、专家墙配筋、角部地质和非交叉拓扑等 43 项连续测试通过；
- 支撑交叉修复、正式闸门和两个公开算例/配筋深化单项测试通过；
- 前端 9 个测试文件、13 项测试通过；
- TypeScript 编译和 Vite 生产构建通过；
- Python `compileall` 通过；
- API Key 权限、HTTP 401/403、管理员在线备份、备份完整性和 SHA-256 校验通过。

历史全量测试在单一进程连续执行时会因公开算例子进程和大型 CAD/配筋模型累积占用而超时。项目已提供 `scripts/test-backend.sh full-isolated` 和 PowerShell 对应命令，以测试节点隔离方式执行完整门禁。该问题属于测试运行资源治理，关键 P0-P3 回归已采用隔离方式验证。

## 9. 尚需真实工程和部署环境完成的工作

软件内部 P0-P3 已闭环。接近法定工业应用仍需外部工作：

- 建立 30–50 个真实项目和标准算例库；
- 与设计院手算、理正/启明星、PLAXIS、MIDAS 等进行逐项对标；
- 完成规范适用性和参数来源审查；
- 完成节点非线性、焊接工艺、吊装专项和施工组织复核；
- 建立 PostgreSQL/对象存储/任务队列的高可用生产架构；
- 开展渗透测试、备份恢复演练、灾备和密钥轮换；
- 将项目监测方案阈值、监测设备质量和现场异常处置流程纳入审批；
- 由设计单位质量体系和注册专业工程师完成最终签发。
