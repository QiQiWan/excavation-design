# 工程方法与边界

## 设计基准

候选生成和计算前必须确认项目工程等级、基坑安全等级、场地复杂程度、周边环境等级、设计阶段、规范体系、荷载组合以及材料和地基参数。系统保存项目级设计取值和来源，通用默认值只用于形成待确认草案，不替代勘察报告、地方标准和专项设计条件。

## 围护结构

围护墙按基坑边段生成，平面分幅与竖向墙趾分开管理。墙长、厚度、嵌固深度和局部加强由地层、开挖深度、稳定性和构造条件共同控制。

## 水平支撑

系统支持短跨对撑、墙—墙角撑混合、分区对撑和闭合环撑等已实现体系。支撑中部 T/Y 交点不作为隐式支座。未实现完整受力模型的中心岛或显式框架体系只能作为工程师定义的专项体系，不能自动降级为普通直撑。

## 计算与安全系数

计算采用施工阶段模型，当前版本聚焦规范算法和统一杆系/墙体计算。计算结果区分当前有效结果和历史归档。任何围护几何、支撑拓扑、截面或配筋修改都会使下游结果失效。

强度、刚度和稳定性统一显示设计值、限值、利用率和安全系数。已有数值模型的项目输出真实计算值；资料不足或模块尚未覆盖的规范必检项保持人工复核，不使用代理值填充正式结论。

## 配筋

围护墙按控制区段和内外侧弯矩包络配筋；围檩按连续梁弯剪包络配筋；混凝土支撑按轴力、偏心和构造要求配筋。配筋草案、可施工性检查和施工图发行资格分层管理。自动结果不替代专业工程师对裂缝、节点、接头、钢筋笼吊装和施工阶段的复核。

## 三维可视化

地质、围护结构、受力云图和钢筋模型用于工程检查和沟通。逐根钢筋采用分层抽样，完整加工数量、接头、弯钩、套筒和吊装构造以经审签的施工图、钢筋表和项目专项构造为准。

## 自动化边界

系统可以自动完成数据完整性、几何、已实现规范子集和构件承载力检查。复杂邻近环境、特殊地下水、未建模施工工法、超出构件目录的节点、地方标准和正式发行签审仍需人工处理。

## P0–P2 工程分析扩展

V3.46 的工程空间模型可以考虑墙体和围檩开裂刚度、节点半刚性、交汇刚域、初始缺陷以及兼作永久结构时的长期效应。所有系数均写入计算模型合同，便于复核和重算。未经试验、构造分析或项目经验标定的半刚性参数属于项目假设。

不利情景模块基于当前施工阶段结果进行透明筛查，用于识别降水失效、超挖、局部渗流、承压水头抬升和长期效应的控制风险。筛查结果不能替代按不利工况重新组装荷载后的正式计算。

局部节点检查用于支撑—围檩节点的承压和爆裂风险筛查。复杂多向节点、预埋件、钢—混连接和高轴力牛腿应采用经验证的壳/实体子模型和项目构造设计。

型钢截面目录提供可版本化的数据接口。内置条目用于软件验证和候选推荐，不构成项目采购或正式设计依据。

## 正式不利工况与节点深化边界

V3.47 的正式不利工况任务会修改项目副本中的水位、开挖标高、渗透系数、承压水头、预加轴力/温差/安装偏差或长期效应参数，并重新生成施工阶段计算。复杂三维渗流、非线性土体和降水井群仍需专项软件。

企业节点模板用于节点初选和施工图草案。高利用率、多向汇交、复杂钢—混组合、预埋件密集和碰撞冲突节点会生成局部子模型输入并保持专项复核状态。逐根钢筋和碰撞结果属于空间协调证据，不能替代施工组织、吊装分段和现场试装。
