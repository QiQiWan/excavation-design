# 核心设计流程

## 1. 设计基准

确认工程等级、基坑安全等级、场地复杂程度、周边环境等级、设计阶段、规范体系、荷载组合、地基承载力以及混凝土和钢筋参数。设计基准未确认时，候选生成、完整计算和正式交付保持阻断。

## 2. 工程输入

必需数据为钻孔或统一地层、闭合基坑轮廓、坑顶标高和坑底标高。三维地质网格用于展示和空间查询，不作为生成概念方案的强制前置条件；正式计算仍需完成地质覆盖和参数来源检查。页面同步显示钻孔、轻量地层剖面、覆盖状态和本步骤规范依据。

## 3. 围护方案

系统先识别轮廓特征，再选择可计算的支撑体系。每次最多生成三个具有体系、支撑间距、立柱服务跨或线位差异的候选。候选必须满足轮廓内布置、端点落墙或明确节点、普通支撑零非法穿越、围檩跨长控制和障碍避让。用户采用一个候选后进入计算。

## 4. 计算验算

默认只计算当前采用方案。计算内容包括土水压力、墙体内力和位移、围檩弯剪、支撑轴力，以及强度、刚度和稳定性。页面给出内力变形包络、支撑轴力云图和安全系数矩阵。A/B/C 三套完整计算只有明确需要时才运行。

## 5. 配筋深化

配筋直接读取当前有效计算包络。核心界面提供保守、均衡和经济三种目标。配筋草案、可施工性检查和施工图资格分层管理。逐根钢筋、加工表和碰撞数据只在深化或导出时生成，不写入网页工作区。

## 6. 成果交付

核心交付包含项目管理摘要、明文计算书、施工图、IFC 协调模型、钢筋深化包和审查成果包。正式施工发行仍需完成项目级规范适用性、节点构造、地质参数和专业审签复核。

## P0–P2 增强规则

- 旧项目缺少 V3.46 字段时只补充待确认草案，不自动确认工程等级和安全目标；
- 作用分组、荷载组合和分析模型参数发生变化时，下游计算与配筋结果失效；
- 候选比选先满足硬约束，再采用 Pareto 排序比较位移、内力、材料和施工复杂度；
- 计算页面同时显示规范目录和证据状态，未计算或未实现项目不能显示为通过；
- 不利情景筛查进入控制区间时，应生成专项复算任务；
- 配筋完成后必须继续执行拥挤、锚固、搭接、机械连接和节点构造检查。

## V3.47 P3 工程深化

- 计算验算完成后，可显式运行正式不利工况专项复算；各场景独立重建施工阶段并保存控制安全系数。
- 配筋方案生成后，可显式运行企业节点与钢筋空间深化，完成节点模板匹配、锚固、套筒、预埋件、碰撞和局部子模型。
- 任务完成后工作台保持当前步骤，仅显示下一步提示；设计人员应先检查当前结果。
- 设计基准首页显示各选项对荷载、刚度、稳定性、配筋、节点和图纸的直接影响。
