# 测试、发行与已知边界

## 最低回归要求

- 项目创建、打开、删除；
- 钻孔导入、地质模型和轮廓保存；
- 三类以上轮廓候选生成，并验证 A/B/C 的体系或构造差异；
- 当前方案完整计算、内力变形包络和稳定性安全系数分布；
- 结构化规范检查值的计算追踪；
- 配筋方案生成；
- 管理者可读 DOCX、项目管理摘要、CAD、IFC 和审查包导出；
- 大型项目工作区加载和外部对象恢复。

## 发行门禁

正式成果至少需要当前计算合同、支撑拓扑、地质参数、构件检查和配筋状态一致。自动通过表示已实现规则未发现超限，不代表所有项目条件均已覆盖。

## 已知边界

- 特殊支撑体系需要专项模型；
- 复杂渗流、邻近结构和三维土体效应需要外部有限元或专项分析；
- DXF 深度依赖 `ezdxf`；
- IFC 兼容性受目标查看器实现影响；
- 专业签审和地方审图要求需要项目配置。

- 安全系数只有在计算值与适用限值均可追溯时才自动判定；缺少限值时保持人工复核。
- 项目管理摘要用于快速沟通，不替代完整计算书、施工图和专业审签。

## V3.46 P0–P2 发行检查

- 旧项目设计基准迁移不得自动确认；
- 规范校核目录必须完整显示证据状态；
- 作用分组启停需要反映在计算合同中；
- A/B/C 至少形成 Pareto 排名和工程差异说明；
- 工程空间模型必须保存开裂刚度、节点半刚性、刚域和初始缺陷假设；
- 不利工况筛查必须明确“筛查”边界，不能写成正式复算结果；
- 配筋可施工性必须覆盖拥挤、锚固、搭接和机械连接；
- 型钢目录必须记录版本，项目使用截面需单独确认；
- 运行诊断和项目 P0–P2 审计能够独立执行。

## V3.53 施工阶段证据发行检查

- 施工阶段输入必须通过标高顺序、坑底覆盖、支撑引用和拓扑一致性校验；
- 用户锁定工况不得被完整计算或兼容入口静默重建；
- 外置计算结果必须保留实际数、期望数、分块数和完整性摘要；
- 验算、计算追溯和配筋入口必须按同一最新结果 ID 读取阶段证据；
- `partial`、`artifact_missing`、结果过期或计算合同不一致均保持配筋阻断；
- 设计资料缺失、计算输出未生成和系统成果不可读必须使用不同原因码与处理动作。
