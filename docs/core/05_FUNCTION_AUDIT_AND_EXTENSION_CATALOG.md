# 功能审计与扩展目录

## V3.39 审计基线

审计发现：177 个 Python 文件、59 个服务模块、22 个路由文件、170 个 API 端点、44 个 React 组件和 138 份 Markdown 文档。多个版本叠加形成了重复的资格、成熟度、问题中心、深化、监测和导出入口。

## 核心模块

核心运行只依赖项目、钻孔、地质、轮廓、设计、计算、配筋、任务和导出模块。默认页面只调用项目工作区、核心状态、候选预览和任务接口。

## 扩展模块

以下能力保留为按需扩展，不参与核心首屏和常规链路：

- 工业成熟度和 P0–P3 评估；
- 高级监测、校准和数字孪生；
- 公开论文基准算例；
- 多套绘图规则优化；
- 多配置 IFC 和完整制造数据；
- 专家墙长、墙趾和协同优化；
- 全量问题中心和模块台账。

## 删除或合并的交互

- 八阶段渐进式配置合并为五个工程步骤；
- 系统可靠性、数据工作区和标准矩阵不再占据主页面；
- 资格、问题、成熟度和高级套件不再并发加载；
- A/B/C 默认只做几何候选，完整计算只执行采用方案；
- 多种成果入口收敛为计算书、施工图、IFC 和审查包四项。
