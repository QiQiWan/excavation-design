# V3.21 工业化成熟度审查与整洁支撑拓扑优化

## 1. 审查结论

PitGuard V3.21 已具备较完整的基坑围护设计辅助链路：地质数据、基坑几何、围护墙、围檩、水平支撑、施工阶段计算、稳定性、配筋、钢筋笼、IFC、CAD、PDF、计算书、候选方案和校审状态均已有代码实现。按代码覆盖范围判断，系统已经超过一般概念验证，处于“高级工程原型/设计辅助平台”阶段。

当前仍不宜将其定义为可独立承担正式施工图签发责任的工业设计软件。主要原因集中在计算模型验证、规范条文闭环、节点与钢筋深化、图纸发行质量、软件工程治理和真实项目回归六个方面。代码中仍存在 simplified FEM、screening、proxy、manual_review、placeholder 等明确边界标记，这些边界必须在正式发行闸门中继续保留。

本次审查采用五个维度的内部成熟度估计。分值用于确定迭代优先级，不代表第三方认证：

| 维度 | 当前估计 | 主要判断 |
|---|---:|---|
| 功能覆盖 | 84/100 | 主流程较全，少量专业模块仍为筛查模型 |
| 工程计算可信度 | 61/100 | 有分阶段、墙梁、围檩和耦合计算，但多处采用简化或代理模型 |
| 施工图与深化 | 66/100 | 图种、钢筋笼和节点已形成，仍缺少设计院级构造规则库和图面自动质检深度 |
| 软件工业化 | 48/100 | 缺少完整 CI、权限、多用户、审计、灾备、性能基线和安全治理 |
| 项目验证与认证 | 39/100 | 尚未形成足量真实项目双算、第三方校核、规范版本认证和签发责任体系 |

综合成熟度建议按 55～62/100 管理。近期目标应是“可用于方案设计、内部校核和成果草拟”，中期目标是“可控条件下的设计院试点”，正式工业发行需完成 P0 和 P1 全部事项。

## 2. 本次支撑拓扑优化

### 2.1 原有问题

原系统已经设置 `supportNoCrossing` 硬约束，非法同层跨中穿越必须为 0。该逻辑只能识别明显 X 型穿越，仍有三个不足：

1. 合法 T/Y 节点不会被统计，候选方案可能没有非法穿越，但内部汇交节点很多。
2. 多杆集中于同一点时，节点钢筋、承压板、立柱和施工顺序会明显复杂，原评分没有单独惩罚。
3. 候选排序主要依赖综合分和拓扑族多样性，整洁度没有形成稳定的首要排序规则。

### 2.2 新增指标

质量审查新增以下指标：

- `supportCrossingCount`：非法同层跨中穿越数，硬约束必须为 0。
- `internalJunctionCount`：坑内 T/Y/X 汇交节点数。
- `highDegreeJunctionCount`：分支度不小于 4 的高度拥挤节点数。
- `maxJunctionBranchDegree`：单节点最大几何分支数。
- `projectedCrossLevelIntersectionCount`：不同支撑层在平面投影中的交叠数，用于图纸和安装协调。
- `sameLevelPlanIntersectionPointCount`：同层非法穿越点与合法内部汇交点总数。
- `planIntersectionComplexity`：交叉与节点复杂度综合指标。

复杂度采用以下分层表达：

```text
C_plan = 100 N_cross + N_junction + 2 N_high + 0.20 N_projected
```

其中非法穿越的惩罚远高于合法节点，确保安全和传力路径优先。合法节点只用于同等可行方案之间的整洁度比较。

### 2.3 候选排序优先级

候选方案采用以下词典序排序：

1. 硬约束是否全部通过；
2. 非法穿越数；
3. 平面交叉复杂度；
4. 高度汇交节点数；
5. 内部汇交节点数；
6. 辅助支撑数量；
7. 支撑总数；
8. 综合工程评分；
9. 几何差异、状态、立柱数量等次级指标。

该顺序保证推荐方案优先保持清晰的直接传力路径，同时不允许通过减少支撑来突破 3～6 m 分仓、围檩支点硬上限、坑内包含、障碍净空、端点有效性或换撑连续性。

### 2.4 默认优化预设

新增 `clean_support_layout` 预设，并设为前端默认：

| 目标 | 默认权重 |
|---|---:|
| 非法穿越 | 80 |
| 汇交节点复杂度 | 80 |
| 障碍冲突 | 34 |
| 间距偏差 | 20 |
| 跨长 | 18 |
| 对称性 | 18 |
| 其他目标 | 保留原工程权重 |

权重用于综合分，词典序清洁度优先级不允许被滑块完全覆盖。设计人员仍可调整偏好，但无法将非法穿越变为可接受方案。

## 3. 距离工业标准的主要差距

### P0：正式设计前必须完成

#### 3.1 计算内核验证与替换

- 墙体分析仍含弹性地基梁简化 FEM、经验水平地基系数和固定支撑刚度。
- 全局耦合模块仍明确使用 spatial-frame proxy 和 condensed matrix。
- 墙内力、角部传力、配筋分区和长期效应存在 fallback 或 proxy。
- 立柱基础、节点局部承压、预加轴力、温度和安装误差多为筛查模型。

要求：建立统一有限元/有限差分求解内核，提供单元定义、边界条件、施工阶段激活、渗流-应力耦合、材料非线性、收敛准则和数值稳定性日志；与至少两套成熟软件及手算基准进行双算。

#### 3.2 规范条文闭环

- 规则模块覆盖多个规范，但部分条文为子集实现、工程筛查或需人工确认。
- 缺少规范版本失效管理、条文适用条件、地区补充规定和项目采用版本冻结。
- 计算书中的公式、参数、来源、取值和条文还需要实现逐项可追溯。

要求：每个规则形成“条文—适用条件—输入—公式—输出—判定—测试案例—版本”八联表；项目创建时冻结规则集哈希，修订后强制提示重新计算。

#### 3.3 真实工程验证库

- 当前测试数量较多，但主要是软件回归和样例工程。
- 缺少连续墙、排桩、SMW、内支撑、锚杆、异形坑、承压水和软土等足量真实案例。

要求：建立不少于 30 个脱敏项目基准，其中至少 10 个包含完整设计院计算书、施工图和监测数据；关键内力、位移、稳定系数、配筋和工程量给出误差门槛。

#### 3.4 正式施工图闸门

- CAD/PDF/IFC 已能输出，仍缺少设计院图框、字体、线型、打印样式、关联尺寸、节点索引、剖切符号、修订云线和跨图一致性自动审查。
- 钢筋笼、围檩、支撑和节点的构造规则仍有 manual_review 项。

要求：建立图纸对象模型和图面规则引擎；所有尺寸来源于模型对象，禁止独立文字覆盖；形成图号、版本、校审、发行状态和变更影响分析。

### P1：设计院试点前完成

#### 3.5 节点与钢筋深化

- 支撑端节点、T/Y 节点、围檩转角、钢筋锚固、接头错开、套筒、预埋件和施工缝仍需更完整的参数化构造族。
- 钢筋碰撞目前偏向包络筛查，缺少真实弯折半径、保护层、净距和加工容差。

要求：按节点类型建立参数化构造模板，联动轴力、弯矩、剪力、局部承压和施工安装；BBS、IFC 钢筋和 CAD 大样必须使用同一根钢筋对象。

#### 3.6 优化器升级

- 当前候选采用离散枚举和规则修复，搜索空间有限。
- 支撑线、墙段、立柱、出土通道和换撑时序尚未形成统一混合整数优化模型。

要求：采用分层优化：拓扑生成、连续线位优化、截面优化、完整计算重排序；增加 Pareto 前沿、约束敏感性和不可行原因解释。

#### 3.7 数据互操作

要求补充：

- DXF/DWG 几何与图层语义导入；
- AGS 或标准化勘察数据接口；
- PLAXIS、MIDAS GTS、FLAC3D 或通用网格/荷载接口；
- IFC IDS/BCF 问题交换；
- 项目坐标、测量坐标和模型局部坐标转换；
- 单位、精度和舍入规则统一。

### P2：规模化部署前完成

#### 3.8 软件工程与安全

- 当前 SQLite 和本地任务模式适合单机试用，不适合多组织并发。
- 缺少用户、组织、项目权限、电子签名、操作审计、密钥管理和数据分级。
- 缺少服务健康、指标、链路追踪、错误聚合、备份恢复和容量规划。

要求：引入 PostgreSQL、对象存储、队列、RBAC、不可篡改审计日志、备份演练、SAST/依赖扫描和部署基线。

#### 3.9 前端可维护性与性能

- `ProjectWorkspace` 仍承担过多职责。
- Three.js 主包超过 500 kB，构建已产生大块警告。

要求：按工作流拆分页面和状态域，实施路由级懒加载、三维组件按需加载、Web Worker 计算、虚拟列表和大模型分层加载。

### P3：产品化增强

- 监测数据自动反演和预警闭环；
- 施工进度、资源和风险联动；
- 设计知识库和企业标准模板；
- 云端项目协同、问题单和审查意见闭环；
- 方案成本、碳排和工期多目标优化。

## 4. 建议的工业化验收指标

### 4.1 支撑拓扑

- 非环形同层非法穿越数 = 0；
- 高度汇交节点数原则上 = 0，存在时必须有专项节点计算和大样；
- 内部 T/Y 节点数在满足围檩和跨长约束下取最小；
- 每个节点有立柱/承托、构造类型、计算结果和图纸索引；
- 候选推荐结果在重复运行中保持确定性。

### 4.2 计算

- 基准案例内力和位移误差满足预设门槛；
- 所有施工阶段有收敛日志和荷载平衡审计；
- 结果与当前几何哈希、规则集哈希、求解器版本绑定；
- 任何 proxy/fallback 结果禁止通过正式发行闸门。

### 4.3 图纸与模型

- 模型、计算书、CAD、PDF、IFC、BBS 数量和编号一致；
- 图纸尺寸、标高和钢筋标注均来自模型；
- 关键图种完整率 100%；
- 无未关闭 fail，warning 和 manual_review 均有责任人、意见和签字状态。

### 4.4 软件质量

- 后端和前端全量测试通过；
- 关键算法具有性质测试、随机几何测试和黄金文件测试；
- 生产构建无高危依赖和版本漂移；
- 支持数据备份、恢复、升级和回滚。
