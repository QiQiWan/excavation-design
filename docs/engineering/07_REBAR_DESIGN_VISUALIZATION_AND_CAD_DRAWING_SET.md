# 配筋设计、三维审查与 CAD 成套出图

## 1. 功能定位

V3.1.0 将钢筋模块从构件级统一配筋扩展为“计算包络—配筋分区—构造检查—三维审查—施工图纸—材料表”的一致性链路。系统输出可用于方案设计、深化设计辅助和可编辑施工图底稿，正式施工图签发仍需完成项目级裂缝、抗震、接头、吊装、预埋件及专业审签。

核心数据只维护一套 `rebarDesignScheme`。三维钢筋、CAD 图纸、钢筋表和构件回写均从该方案派生，减少不同成果之间的配筋规格漂移。

## 2. 配筋方案生成

### 2.1 输入与模式

接口：

- `GET /api/projects/{project_id}/rebar/design-scheme?mode=balanced`
- `POST /api/projects/{project_id}/rebar/apply-design-scheme`
- `GET /api/projects/{project_id}/rebar/detailing?mode=balanced`

模式包括：

- `conservative`：提高需求放大系数、优先采用较密间距，适合初步保守校核。
- `balanced`：默认模式，兼顾承载、构造、钢筋拥挤和材料量。
- `economic`：在承载与最小构造约束满足时放宽间距，适合候选方案比较，不自动绕过失败项。

方案优先读取最新施工阶段的墙体弯矩、剪力、位移和支撑轴力包络。局部弯矩结果缺失时，系统以构件控制值或剪力—位移代理进行分区，并将 `envelopeSource` 标记为 `shear_displacement_proxy`，同时要求复算。

### 2.2 地下连续墙分区

每幅墙根据墙顶、开挖面、支撑节点、墙趾及控制包络自动形成以下区域：

- `crown_zone`：墙顶及冠梁连接区；
- `support_node_zone`：支撑或围檩作用标高附近的局部加强区；
- `excavation_transition_zone`：开挖面上下的内力转换区；
- `toe_zone`：墙趾及嵌固控制区；
- `field_zone`：其余一般区段。

每个分区分别保存：

- 顶、底标高和区段高度；
- 坑内侧与坑外侧主筋；
- 计算需求面积和实际提供面积；
- 水平分布筋和拉结筋；
- 控制弯矩、剪力、位移及来源；
- 承载状态、推荐最小墙厚和施工性说明；
- 关联图号，如 `R-02`、`D-04`、`D-06`。

墙体竖向主筋采用每延米配筋面积表达：

\[
A_{s,\mathrm{prov}}=\frac{1000}{s}\frac{\pi d^2}{4}
\]

其中，`d` 为钢筋直径，`s` 为间距。程序从可施工直径和间距组合中选择满足需求的方案。需求超过预设单层钢筋组合能力时，状态保留为 `fail`，并给出增大墙厚、采用经核验的双层/多排布置或机械连接体系等复核建议。

### 2.3 钢筋混凝土支撑分区

钢筋混凝土支撑根据设计轴力、截面和跨度生成：

- 纵向受力钢筋；
- 两端加密区箍筋；
- 跨中区箍筋；
- 侧面拉结/架立钢筋；
- 错开搭接及局部附加钢筋。

端部加密区间距始终不大于跨中区间距。系统同时输出纵筋拥挤率、容量余量、端部区长度和关联图号 `R-04`、`D-01`、`D-03`、`D-07`。

### 2.4 冠梁、围檩、环梁与节点

梁和节点方案包含：

- 梁纵向主筋与箍筋；
- 支撑—围檩节点 U 形附加筋；
- 节点约束箍筋；
- 支撑端部抗劈裂钢筋网；
- 角撑转角区局部加强；
- 支撑—立柱交叉处承压和定位构造。

节点方案与 D-01～D-07 大样图绑定。节点几何、预埋件、套筒型号和焊缝要求仍需结合项目施工工法确认。

### 2.5 方案应用与可追溯性

“生成方案”仅进行预览。“应用方案”会：

1. 将完整分区方案写入 `retainingSystem.rebarDesignScheme`；
2. 将各墙面控制分区回写为构件级控制钢筋组；
3. 将钢筋混凝土支撑的纵筋、端部箍筋、跨中箍筋和附加筋回写至构件；
4. 保留分区级细节供三维显示、CAD 和表格使用；
5. 保存模式、检查状态、包络来源和图纸引用。

构件级回写用于兼容既有 IFC 和计算书接口；分区方案是 V3.1.0 配筋成果的主数据源。

## 3. 配筋审查状态

方案摘要包括：

- 墙体分区数量；
- 支撑、梁和节点方案数量；
- `pass`、`warning`、`fail` 数量；
- 包络异常数量；
- 需要升级截面或配筋体系的分区数量；
- 墙竖向钢筋和支撑纵筋的相对材料量指标。

状态原则：

- `pass`：当前组合满足程序实现的承载和构造子集；
- `warning/manual_review`：需要项目级构造、裂缝、抗震、接头或施工复核；
- `fail`：需求超过当前组合能力、输入或计算包络异常，禁止通过“经济模式”自动降级为通过。

## 4. 三维配筋审查

V3.1.0 的钢筋三维查看器支持：

- 按钢筋类型、校核状态或宿主构件着色；
- 按墙、支撑、围檩/冠梁和节点过滤；
- 按宿主构件隔离；
- X、Y、标高三个方向的剖切；
- 宿主混凝土透明度调节；
- 钢筋显示半径放大，便于远景检查；
- 支撑端部锚固、错开搭接、闭合箍筋和附加筋标注；
- 选中钢筋显示分区、侧别、包络来源和关联图纸；
- 按宿主轮询采样，并优先保留失败、警告和复核对象，避免大量钢筋集中在首个构件。

应用分区方案后，墙体钢筋具有 `zoneId`、`zoneType`、`face`、区段标高和 `drawingRefs`，可从三维对象追溯到 `R-02-Wxx` 单墙立面及节点大样。

浏览器显示仍采用抽样几何。`estimatedFullCount` 表示估算完整数量，抽样条数不得直接用于下料统计；正式数量以钢筋表和逐根几何 CSV 为准。

## 5. CAD 成套图纸结构

接口：

- `GET /api/projects/{project_id}/export/drawings-manifest`
- `GET /api/projects/{project_id}/export/drawings-cad?scope=full&rebar_mode=balanced`

`scope` 可取：

- `full`：完整图纸包；
- `general`：总说明、总平面、分层平面和剖面；
- `rebar`：配筋总图、分区立面和钢筋表；
- `details`：节点大样。

目录结构：

```text
00_general/     图纸目录、设计总说明、图例
10_plans/       围护与支撑总平面、各层支撑平面
20_sections/    开挖剖面与施工阶段
30_rebar/       墙、支撑、围檩配筋总图及单墙立面
40_details/     典型节点和构造大样
90_schedules/   钢筋、材料、分区、搭接、吊装和质量检查表
```

### 5.1 全局图形

- `G-00`：图纸目录、设计总说明与图例；
- `S-00`：基坑围护与支撑总平面图；
- `S-02-Lxx`：逐层支撑平面图；
- `S-03`：典型开挖剖面与施工阶段图；
- `M-01`：监测点布置总图；
- `R-01`：地下连续墙配筋总图；
- `R-04`：钢筋混凝土支撑配筋总图；
- `R-05`：冠梁、围檩及环梁配筋总图。

总平面表达墙幅编号、支撑层、围檩、立柱和设计轴力。分层平面避免多层支撑叠加造成读图困难。

### 5.2 分区立面

- `R-02`：全部墙幅配筋分区汇总立面；
- `R-02-Wxx`：每幅墙独立分区立面。

单墙立面标注：墙顶、坑底、墙趾、支撑节点区、开挖转换区、一般区、坑内外侧钢筋、水平分布筋、区段状态和图纸引用。该结构适合后续按图幅拆分和企业模板排版。

### 5.3 节点大样

- `D-00`：典型节点组合索引；
- `D-01`：支撑—围檩节点；
- `D-02`：角撑节点及转角加强；
- `D-03`：支撑—立柱交叉节点；
- `D-04`：地连墙支撑区局部加强；
- `D-05`：临时立柱及立柱桩；
- `D-06`：地连墙墙幅接头及钢筋笼连接；
- `D-07`：钢筋混凝土支撑端部锚固与错开搭接。

图纸标题栏根据实际图元边界自动放置，降低深基坑负高程立面与标题栏重叠的风险。

### 5.4 表格与一致性文件

`90_schedules/` 包含：

- 配筋表与材料表；
- 钢筋下料和逐根几何；
- 钢筋笼分节、搭接区和吊装计划；
- 保护层、弯折和冲突检查；
- 墙体分区配筋表；
- 支撑端部/跨中配筋表；
- 交付一致性矩阵；
- 企业模板清单。

`drawing_set_manifest.json` 保存计划图纸、实际生成文件、作用域、配筋模式和模型绑定；`rebar_design_scheme.json` 保存生成图纸时采用的完整配筋方案。V2.x 脚本使用的若干根目录 CSV 文件继续保留兼容副本。

## 6. 仍需继续深化的事项

以下内容尚需项目级参数或更完整的有限元/施工信息，系统保留为后续迭代项：

1. 按裂缝宽度、长期组合和耐久性重新控制墙体配筋；
2. 抗震等级、节点核心区和强节点弱构件规则；
3. 机械连接套筒、焊接接头、墙幅接头型式和止水构造选型；
4. 钢筋笼吊点、吊装变形和分节重量专项验算；
5. 钢筋与型钢、预埋件、止水钢板和监测元件的三维碰撞；
6. CAD 纸空间、自动视口、企业 CTB/STB、DWG 原生输出和批量 PDF 出图；
7. 施工阶段变更后配筋分区自动差异追踪和变更云线；
8. 与注册工程师审签、图纸修订号和电子签章流程联动。

## V3.2.0 诊断与出图闸门补充

V3.2.0 在分区配筋数据之上增加计算有效性和传力体系审计。配筋状态不得仅依据钢筋面积判断，还必须检查施工阶段是否引用当前支撑、端墙是否具有直接传力路径、支撑设计轴力是否已经完成极限状态组合、构件截面是否超出单筋或单层钢筋适用范围，以及节点承压板是否满足目标利用率。

界面按四步流程组织，默认隐藏已通过的批量数据。图纸输出分为 `review` 与 `construction`：存在计算失效、拓扑失败或配筋阻断项时，只允许输出带审查标识的设计辅助成果；阻断项清零后可输出施工图复核包。两类成果均需完成企业审签后方可用于正式施工。

主次支撑网格交点统一绑定支撑、立柱、节点和 D-08 大样图号。CAD、三维、配筋表和计算结果均采用同一 `supportRole=secondary_strut` 数据，不允许在出图阶段临时补画无计算对象的支撑。
