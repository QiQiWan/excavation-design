# 异形基坑计算恢复与智能出图

## 1. 问题背景

L 形、T 形和带凹口基坑不能沿用矩形基坑的单一主方向支撑假设。凹角相邻的回墙若没有支撑端点，墙体计算会退化为非预期悬臂工况，常见表现包括：

- 回墙最大位移异常放大；
- 墙体弯矩、剪力和配筋需求集中；
- 同一条文在多个施工阶段重复报错；
- 用户只看到 `fail`，难以判断是支撑拓扑、施工阶段还是截面设计问题。

V3.5.0 将该问题拆分为几何拓扑、施工工况、墙体计算、规范检查和图纸表达五个可追溯环节。

## 2. 凹角回墙支撑诊断

系统按闭合多边形有向转角识别凹角，并检查凹角相邻墙面是否存在直接支撑端点。发现回墙缺少直接传力路径时：

1. 沿墙面坑内法向发射局部支撑线；
2. 与首个对向墙面求交；
3. 排除穿越坑外空区、障碍物和既有无节点交叉的候选；
4. 在所有支撑层复制有效平面线；
5. 与主对撑交点生成共享节点和临时立柱；
6. 重新计算分担宽度、围檩节点和支撑拓扑哈希。

该修复采用增量方式，保留人工布置和既有优化支撑。

## 3. 施工工况同步

支撑增补或候选方案采用后，构件 ID 与拓扑哈希会变化。计算前置检查比较：

- 当前支撑拓扑哈希；
- 计算案例拓扑哈希；
- 每个施工阶段引用的支撑 ID；
- 无有效支撑的阶段数量。

存在失效引用时，系统按支撑层、安装顺序、换撑和拆撑状态重建阶段激活关系。同步过程记录在 `supportTopologySynchronization` 中。

## 4. 计算根因诊断

项目级检查台账对跨阶段重复问题进行归并，阶段结果仍保留逐阶段明细。诊断输出包括：

- 根因编码与说明；
- 受影响墙段；
- 各支撑层直接支撑数量；
- 墙体位移、弯矩和剪力包络；
- 自动修复和工况同步记录；
- 修复前后指标；
- 推荐的下一步操作。

当前根因分类覆盖：

- `UNRESTRAINED_CONCAVE_RETURN_WALL`；
- `STALE_STAGE_SUPPORT_REFERENCES`；
- `WALL_SHEAR_CAPACITY`；
- `WALL_REBAR_OR_FLEXURE`；
- `WALL_DEFORMATION`。

## 5. 配筋构造修复

自动钢筋组合增加 75 mm 最小净距约束。达到单排钢筋能力边界时，系统进入双筋、截面增大或支撑拓扑调整分支，不再采用与自身构造规则冲突的 D40@100 兜底组合。

## 6. 智能出图

Drawing Intelligence Engine 在 V3.4 规则引擎之上增加工程知识推理。输入包括：

- 基坑凹角数量；
- 墙面和支撑层数量；
- 次对撑与临时立柱；
- 计算根因和警告；
- 当前图纸规则覆盖。

输出包括：

- 图纸建议及原因；
- 建议图纸规则 ID；
- 置信度和覆盖状态；
- 覆盖度、可读性、追溯性、施工完整性和一致性评分。

异形基坑且已生成局部次对撑时，规则自动触发 D-09《异形凹角回墙局部支撑与围檩转折大样》。CAD 包同时输出：

- `90_schedules/calculation_diagnostics.json`；
- `90_schedules/calculation_diagnostics.csv`；
- `90_schedules/drawing_intelligence.json`。

## 7. 工程边界

局部次对撑由几何和线弹性设计筛查生成。以下内容仍需专业复核：

- 围檩转折区扭转和局部承压；
- 主次支撑交叉节点；
- 临时立柱与立柱桩；
- 支撑安装、换撑和拆撑顺序；
- 出土通道及机械施工净空；
- 高利用率节点的非线性有限元分析。
