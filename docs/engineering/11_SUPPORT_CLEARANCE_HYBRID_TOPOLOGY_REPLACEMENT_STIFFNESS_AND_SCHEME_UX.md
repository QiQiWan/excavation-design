# 支撑净距、混合拓扑、换撑刚度与整案决策技术说明

## 1. 为什么支撑轴线不能与墙线重合

围护墙线表示墙体或墙面控制边界，水平支撑轴线通常位于坑内，并通过围檩、承压板、节点核心区或短刚臂传力。将支撑轴线画在墙线上会造成三类问题：几何碰撞、节点语义丢失以及围檩里程计算混乱。

V3.6 同时保存两套点：

- `wallConnection`：墙面/围檩的真实连接位置；
- `start/end`：坑内偏移后的支撑中心线端点。

围檩节点、墙面分担宽度和全局矩阵使用 `wallConnection`；支撑实体、三维显示和净距检查使用偏移后的中心线。

## 2. 支撑拓扑选择

### 2.1 传统直对撑

优点是构造清晰、轴向传力直接；缺点是异形坑中可能形成超长杆件、占用出土通道，并使凹角回墙受力路径不均匀。

### 2.2 斜撑与短对撑混合

长墙凸角采用短斜撑形成邻墙之间的局部三角传力，靠角超长直对撑可以删除；中部仍保留必要直对撑，避免长墙中段无支点。该体系适合长墙、明显转角和通道受限项目。

### 2.3 双向网格

在主对撑正交方向增加次对撑，并在交点设置临时立柱或刚性节点。该体系可直接约束回墙，但节点数量、交叉协调和施工组织更复杂。

## 3. 全案决策评分

候选搜索阶段的评分用于快速筛选，完整计算阶段增加第二次决策评分。评分使用候选集内归一化，较小值更优的指标包括：

\[
N_i(x)=\frac{x_{max}-x_i}{x_{max}-x_{min}}
\]

综合得分由结构响应、施工性和预筛知识组成，并扣除失败、警告和人工复核惩罚。任何存在硬失败的方案都不会被标记为推荐。由于采用候选后拓扑和构件编号会变化，正式采用动作必须重建施工阶段并复算。

## 4. 换撑刚度表达

“0”可能代表未激活、参数缺失、构件失效或真正接近零刚度，工程含义完全不同。V3.6 将状态与数值分离：

| 状态 | 数值 | 工程含义 |
|---|---|---|
| `not_active` | `null` | 当前阶段尚未形成楼板换撑 |
| `active` | 正数 | 已装配等效换撑刚度 |
| `missing` | `null` | 应激活，但有效宽度/厚度/弹模等缺失 |
| `invalid` | `null` | 参数非正、非有限或传力长度无效 |

当前等效模型适合施工阶段筛查。楼板开洞、框架梁柱、连接滑移和楼板开裂会改变实际刚度，正式设计应使用板壳—杆系或实体局部模型校准。

## 5. 界面信息分层

默认决策层只显示：

1. 当前计算状态；
2. A/B/C 完整方案三维卡片；
3. 支撑/立柱、跨度、轴力、位移、Fail/Warning；
4. 完整计算推荐和整体采用按钮。

以下内容默认折叠：

- 内力包络和墙体云图；
- 分墙设计长度冗余；
- 评分分解和方案差异动画；
- 全局矩阵及换撑刚度逐墙台账；
- 全量规范检查和围檩节点反力。

这使设计人员先完成方案决策，再进入专业审查。
